# DOM и события

<span class="text-gray">DOM (Document Object Model) — это представление HTML в браузере в виде дерева объектов. События — механизм, через который пользовательские взаимодействия и системные изменения распространяются по этому дереву. Для senior-разработчиков: делегирование событий, passive listeners, очистка через AbortController и понимание тайминга микрозадач относительно рендеринга — частые темы на собеседованиях.</span>

---

## DOM Tree

<b class="text-on-semi-light">Типы узлов</b>

<span class="text-gray">Всё в DOM — это узел (node). Основные типы: **Element**, **Text**, **Comment**, **Document**, **DocumentFragment**.</span>

---

<b class="text-on-semi-light">Обход и выборка элементов (Traversal & Selection)</b>

| Метод | Возвращает | Live/Static | Скорость |
|---|---|---|---|
| `getElementById()` | Element | — | Самый быстрый |
| `querySelector()` | Element | — | Быстрый |
| `querySelectorAll()` | NodeList | **Static** | Быстрый |
| `getElementsByClassName()` | HTMLCollection | **Live** | Быстрый |
| `getElementsByTagName()` | HTMLCollection | **Live** | Быстрый |

<span class="text-gray">**Live** коллекция автоматически обновляется при изменении DOM — это может вызвать баги в циклах. **Static** — это снимок (snapshot) на момент вызова.</span>

```js
const live = document.getElementsByClassName('item'); // live
const static_ = document.querySelectorAll('.item');   // static snapshot

document.body.appendChild(document.createElement('div')).className = 'item';
console.log(live.length);    // увеличился!
console.log(static_.length); // не изменился
```

> Live-коллекции удобны для отслеживания изменений, но опасны в `for`-циклах: удаление элемента сдвигает индексы.

## DOM Manipulation

<b class="text-on-semi-light">Создание и вставка элементов</b>

```js
// Один элемент
const el = document.createElement('div');
el.textContent = 'Hello';
el.classList.add('card');
parent.appendChild(el);

// Batch вставка через DocumentFragment (одна reflow)
const fragment = document.createDocumentFragment();
for (let i = 0; i < 1000; i++) {
  const li = document.createElement('li');
  li.textContent = `Item ${i}`;
  fragment.appendChild(li);
}
list.appendChild(fragment); // одна вставка в DOM

// insertAdjacentHTML — быстрее innerHTML для добавления
el.insertAdjacentHTML('beforeend', '<span>new</span>');
```

---

<b class="text-on-semi-light">innerHTML vs DOM API</b>

<span class="text-gray">`innerHTML` парсит HTML-строку. Быстрее для массовой замены, но создаёт риск XSS если пользовательский ввод не экранирован. DOM API — безопаснее, больше контроля, лучше для работы с отдельными элементами.</span>

```js
// ⚠️ XSS уязвимость
el.innerHTML = userInput; // если userInput = '<img onerror="alert(1)" src=x>'

// ✅ Безопасно
el.textContent = userInput; // escapes HTML
```

---

<b class="text-on-semi-light">Virtual DOM (кратко)</b>

<span class="text-gray">Фреймворки (React, Vue) используют виртуальный DOM — JS-представление реального DOM. При изменении состояния они сравнивают (diff) виртуальное дерево и применяют минимальные изменения к реальному DOM. Это позволяет избежать ручной манипуляции и layout thrashing.</span>

> Подробнее о Virtual DOM: [React](/content/09.react) и [Vue](/content/10.vue).

## Событийная модель (senior-critical)

<b class="text-on-semi-light">Три фазы события</b>

```text
         document
          │  ↑
     Capture │  │ Bubble
          ↓  │
         body
          │  ↑
          ↓  │
        parent
          │  ↑
          ↓  │
     ──▶ target ◀──
```

1. **Capture phase** (сверху вниз): событие спускается от `document` к `target`
2. **Target phase**: событие на самом элементе
3. **Bubble phase** (снизу вверх): событие поднимается от `target` к `document`

<span class="text-gray">По умолчанию `addEventListener` слушает на фазе **bubble**. Чтобы перехватить событие на фазе capture, нужно передать `{ capture: true }` или `true` третьим аргументом.</span>

---

<b class="text-on-semi-light">addEventListener options</b>

```js
element.addEventListener('click', handler, {
  capture: false,  // слушать на фазе capture? (default: false = bubble)
  passive: true,   // не вызовет preventDefault? (browser can optimize scroll)
  once: true,      // автоматическая отписка после первого вызова
  signal: controller.signal, // отмена через AbortController
});
```

---

<b class="text-on-semi-light">stopPropagation vs stopImmediatePropagation vs preventDefault</b>

| Метод | Что делает | Пример |
|---|---|---|
| `preventDefault()` | Отменяет дефолтное действие браузера | Отмена submit формы, отмена перехода по ссылке |
| `stopPropagation()` | Останавливает всплытие/погружение | Клик внутри модала не закрывает его |
| `stopImmediatePropagation()` | + останавливает другие обработчики на том же элементе | Гарантировать, что только один обработчик сработает |

> `stopPropagation()` не отменяет дефолтное действие браузера — для этого нужен `preventDefault()`.

## Event Delegation

<span class="text-gray">Вместо навешивания обработчиков на N элементов, вешаем ОДИН на их родителя. События всплывают (bubble), и родитель их перехватывает.</span>

```js
// ❌ Плохо: 1000 обработчиков
document.querySelectorAll('.item').forEach(item => {
  item.addEventListener('click', handleClick);
});

// ✅ Хорошо: 1 обработчик с делегированием
document.querySelector('.list').addEventListener('click', (e) => {
  const item = e.target.closest('.item');
  if (!item) return;
  handleClick(item);
});
```

<b class="text-on-semi-light">Преимущества</b>

- Меньше обработчиков → меньше памяти
- Работает с динамически добавленными элементами
- Проще очистка (cleanup)

<b class="text-on-semi-light">Когда НЕ использовать</b>

<span class="text-gray">`focus`/`blur` не всплывают по умолчанию — используйте `focusin`/`focusout`. Также делегирование не подходит, если нужна фаза capture.</span>

<b class="text-on-semi-light">Как это делают фреймворки</b>

<span class="text-gray">React: один event listener на root-элементе + система синтетических событий (SyntheticEvent). Vue: использует нативное делегирование для `v-on`.</span>

## Passive Listeners (senior interview topic)

<span class="text-gray">По умолчанию браузер должен дождаться завершения обработчика `touchstart`/`touchmove`, чтобы узнать, будет ли вызван `preventDefault()`. Это блокирует плавный скролл.</span>

`{ passive: true }` гарантирует, что обработчик не вызовет `preventDefault()` → браузер может скроллить сразу, не дожидаясь.

```js
// ⚠️ Scroll jank: browser waits for handler
window.addEventListener('touchmove', handler);

// ✅ Smooth scroll: browser doesn't wait
window.addEventListener('touchmove', handler, { passive: true });

// Chrome devtools warning если passive не установлен на scroll/touch events
```

<span class="text-gray">Альтернатива в CSS: `touch-action: manipulation` — отключает double-tap-to-zoom и pinch, позволяя браузеру скроллить сразу.</span>

> В Chrome `touchstart` и `touchmove` обработчики на `document`/`window`/`body` по умолчанию passive. Если нужен `preventDefault()`, явно укажите `{ passive: false }`.

## AbortController (must-know for seniors)

<span class="text-gray">Универсальный механизм отмены. Работает с fetch, event listeners и любым API, принимающим AbortSignal.</span>

---

<b class="text-on-semi-light">Отмена fetch</b>

```js
const controller = new AbortController();

fetch('/api/data', { signal: controller.signal })
  .then(res => res.json())
  .catch(err => {
    if (err.name === 'AbortError') console.log('Cancelled');
  });

// Отмена через 5 секунд
setTimeout(() => controller.abort(), 5000);

// Или встроенный timeout
fetch('/api/data', { signal: AbortSignal.timeout(5000) });
```

---

<b class="text-on-semi-light">Удаление event listeners</b>

```js
const controller = new AbortController();

window.addEventListener('resize', handleResize, { signal: controller.signal });
window.addEventListener('scroll', handleScroll, { signal: controller.signal });
document.addEventListener('keydown', handleKey, { signal: controller.signal });

// Очистка ВСЕХ обработчиков одним вызовом
controller.abort();
```

---

<b class="text-on-semi-light">Паттерны очистки во фреймворках</b>

```js
// React
useEffect(() => {
  const controller = new AbortController();

  fetch('/api', { signal: controller.signal }).then(/*...*/);
  window.addEventListener('resize', onResize, { signal: controller.signal });

  return () => controller.abort(); // cleanup everything
}, []);
```

```js
// Vue
onMounted(() => {
  const controller = new AbortController();

  fetch('/api', { signal: controller.signal }).then(/*...*/);
  window.addEventListener('resize', onResize, { signal: controller.signal });

  onUnmounted(() => controller.abort());
});
```

---

<b class="text-on-semi-light">AbortSignal.any()</b>

<span class="text-gray">Комбинирование нескольких сигналов:</span>

```js
const timeout = AbortSignal.timeout(5000);
const manual = new AbortController();

fetch('/api', {
  signal: AbortSignal.any([timeout, manual.signal])
});
// Отменится если ЛИБО timeout истечёт, ЛИБО будет вызван manual.abort()
```

> AbortController — замена устаревшим паттернам с флагами `isCancelled` и `removeEventListener`. Один `abort()` очищает всё.

## Custom Events

```js
// Создание
const event = new CustomEvent('item:selected', {
  detail: { id: 42, name: 'Product' },
  bubbles: true,     // всплывает по DOM
  composed: true,    // пересекает Shadow DOM boundary
});

// Отправка
element.dispatchEvent(event);

// Прослушивание
document.addEventListener('item:selected', (e) => {
  console.log(e.detail.id); // 42
});
```

<span class="text-gray">Применение: коммуникация между компонентами без фреймворка, API пользовательских элементов (Custom Elements), развязка модулей.</span>

<b class="text-on-semi-light">Типизированные Custom Events (TypeScript)</b>

```ts
// Объявляем тип для события
interface ItemSelectedDetail {
  id: number;
  name: string;
}

// Типизированная обёртка
function dispatch<T>(el: Element, name: string, detail: T) {
  el.dispatchEvent(new CustomEvent(name, { detail, bubbles: true }));
}

dispatch<ItemSelectedDetail>(element, 'item:selected', { id: 42, name: 'Product' });
```

## MutationObserver

<span class="text-gray">Асинхронно наблюдает за изменениями DOM. Callback-и группируются (batched) в микрозадачу — эффективнее polling.</span>

```js
const observer = new MutationObserver((mutations) => {
  for (const mutation of mutations) {
    if (mutation.type === 'childList') {
      console.log('Children changed:', mutation.addedNodes);
    }
    if (mutation.type === 'attributes') {
      console.log('Attribute changed:', mutation.attributeName);
    }
  }
});

observer.observe(targetNode, {
  childList: true,     // отслеживать добавление/удаление потомков
  attributes: true,    // отслеживать изменения атрибутов
  subtree: true,       // наблюдать за всем поддеревом
  characterData: true, // отслеживать изменения текстового содержимого
});

// Очистка
observer.disconnect();
```

<span class="text-gray">Применение: обнаружение DOM-изменений от сторонних скриптов, инструменты accessibility, полифиллинг фич.</span>

<b class="text-on-semi-light">IntersectionObserver (бонус)</b>

<span class="text-gray">Отслеживает видимость элемента во viewport. Полезен для lazy-loading изображений, бесконечного скролла, аналитики показов.</span>

```js
const io = new IntersectionObserver((entries) => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      loadImage(entry.target);
      io.unobserve(entry.target); // перестать наблюдать после загрузки
    }
  });
}, { threshold: 0.1 }); // сработает когда 10% элемента видно

document.querySelectorAll('img[data-src]').forEach(img => io.observe(img));
```

## Event Loop и рендеринг

<b class="text-on-semi-light">Где рендеринг вписывается в event loop</b>

```text
┌─────────────────────────────────────────────┐
│ 1. Execute one macrotask (setTimeout, I/O)  │
│ 2. Execute ALL microtasks                   │
│    (Promise, queueMicrotask)                │
│ 3. If it's time to render (~16ms):          │
│    a. requestAnimationFrame callbacks       │
│    b. Style calculation                     │
│    c. Layout                                │
│    d. Paint                                 │
│    e. Composite                             │
│ 4. requestIdleCallback (if time remains)    │
│ 5. Go to 1                                  │
└─────────────────────────────────────────────┘
```

---

<b class="text-on-semi-light">requestAnimationFrame vs setTimeout</b>

```js
// ⚠️ setTimeout(fn, 0): runs in NEXT macrotask, may miss frame
setTimeout(() => {
  element.style.transform = 'translateX(100px)';
}, 0);

// ✅ requestAnimationFrame: runs BEFORE next paint
requestAnimationFrame(() => {
  element.style.transform = 'translateX(100px)';
});
```

<span class="text-gray">`requestAnimationFrame` синхронизирован с частотой обновления дисплея. Монитор 60Hz → 60 вызовов/сек. Монитор 120Hz → 120 вызовов/сек.</span>

> Подробнее об event loop: [Потоки и параллелизм](/content/computer-science/threads).

---

<b class="text-on-semi-light">Типичная задача на собеседовании: порядок вывода</b>

```js
console.log('1: script');

setTimeout(() => console.log('2: setTimeout'), 0);

Promise.resolve().then(() => console.log('3: microtask'));

requestAnimationFrame(() => console.log('4: rAF'));

console.log('5: script end');

// Вывод:
// 1: script
// 5: script end
// 3: microtask         ← microtask queue (перед рендером)
// 4: rAF               ← перед paint
// 2: setTimeout        ← следующий macrotask
```

> Микрозадачи выполняются **до** рендеринга, `requestAnimationFrame` — **перед** paint, а `setTimeout` — в следующем цикле macrotask.
