# SEO для фронтенд-разработчика

SEO (Search Engine Optimization) — это не только задача маркетологов. Большинство факторов, влияющих на индексацию и ранжирование, контролируются именно фронтенд-разработчиком: выбор архитектуры рендеринга (SPA vs SSR), семантическая разметка, мета-теги, structured data, производительность и доступность. На senior-интервью вопросы уровня «почему SPA плохо индексируется?» или «как настроить canonical для пагинации?» встречаются регулярно.

---

## Как работают поисковые системы

Процесс состоит из трёх этапов:

<b class="text-on-semi-light">Crawling (сканирование)</b>

<span class="text-gray">Бот (crawler, spider) обходит веб по ссылкам: начинает с известных URL, скачивает HTML, находит ссылки, добавляет их в очередь. Googlebot — это полноценный Chromium: он исполняет JavaScript, ждёт рендеринга и индексирует итоговый DOM. Но ресурсы на рендеринг ограничены — бот не будет ждать 10 секунд, пока SPA загрузит данные через API.</span>

<b class="text-on-semi-light">Indexing (индексация)</b>

<span class="text-gray">После получения контента поисковик анализирует текст, мета-теги, структуру, ссылки и сохраняет страницу в индекс. Если контент дублируется, низкого качества или заблокирован — страница может не попасть в индекс.</span>

<b class="text-on-semi-light">Ranking (ранжирование)</b>

<span class="text-gray">При поисковом запросе система выбирает из индекса наиболее релевантные страницы и ранжирует их. Факторы: релевантность контента, ссылочная масса, поведенческие факторы, Core Web Vitals, mobile-friendliness, HTTPS.</span>

> Googlebot рендерит JavaScript, но с задержкой. Первый проход (crawl) может получить только пустой `<div id="app">`. Рендеринг (second wave) происходит позже, когда у Google есть свободные ресурсы. SSR/SSG устраняют эту проблему полностью.

## Рендеринг и индексация

Выбор архитектуры рендеринга — **главный SEO-фактор**, который контролирует фронтенд-разработчик.

| Архитектура | Индексация | Проблемы |
|---|---|---|
| **SSG** (Static Site Generation) | Отличная — HTML готов | Контент фиксирован до пересборки |
| **SSR** (Server-Side Rendering) | Отличная — HTML с сервера | Нагрузка на сервер |
| **ISR** (Incremental Static Regen) | Отличная — статика + обновления | Может показать stale контент |
| **SPA** (Client-Side Rendering) | Проблемная — JS-рендеринг | Googlebot рендерит с задержкой, другие боты не рендерят вообще |

<b class="text-on-semi-light">Почему CSR (SPA) плохо для SEO</b>

1. **Первый HTML пустой** — бот видит `<div id="app"></div>` до выполнения JS.
2. **JavaScript rendering queue** — Google рендерит JS отдельным потоком с задержкой от часов до дней.
3. **Другие поисковики** (Bing, Yandex) хуже рендерят JS или не рендерят вообще.
4. **Мета-теги устанавливаются через JS** — бот может не дождаться их появления.
5. **Ссылки через `onClick` + `router.push`** — бот не видит их как `<a href>`.

<b class="text-on-semi-light">Решения</b>

- **SSR/SSG** — HTML доступен сразу. Nuxt, Next.js, Astro.
- **Pre-rendering** — генерация HTML для ботов (prerender.io, rendertron). Рабочий, но хрупкий подход.
- **Hydration** — сервер отдаёт HTML, клиент «оживляет» его. Стандартный подход для Nuxt/Next.

> Подробнее об архитектурах рендеринга — [Что такое веб](/content/web/what-is-web).

## Мета-теги

### Базовые мета-теги

```html
<head>
  <!-- Title: 50-60 символов, уникальный для каждой страницы -->
  <title>Купить кроссовки Nike Air Max — Магазин Shoes.com</title>

  <!-- Description: 150-160 символов, содержит ключевые слова -->
  <meta name="description" content="Кроссовки Nike Air Max с бесплатной доставкой. Более 200 моделей в наличии. Гарантия оригинальности.">

  <!-- Canonical: указывает основной URL (избегает дублей) -->
  <link rel="canonical" href="https://shoes.com/nike-air-max">

  <!-- Robots: управление индексацией -->
  <meta name="robots" content="index, follow">
  <!-- noindex — не индексировать, nofollow — не переходить по ссылкам -->

  <!-- Viewport: обязателен для mobile-friendly -->
  <meta name="viewport" content="width=device-width, initial-scale=1">

  <!-- Language -->
  <html lang="ru">
</head>
```

### Open Graph (социальные сети)

```html
<meta property="og:title" content="Купить кроссовки Nike Air Max">
<meta property="og:description" content="Более 200 моделей с бесплатной доставкой">
<meta property="og:image" content="https://shoes.com/images/og-preview.jpg">
<meta property="og:url" content="https://shoes.com/nike-air-max">
<meta property="og:type" content="product">
<meta property="og:locale" content="ru_RU">
```

### Twitter Cards

```html
<meta name="twitter:card" content="summary_large_image">
<meta name="twitter:title" content="Купить кроссовки Nike Air Max">
<meta name="twitter:description" content="Более 200 моделей с бесплатной доставкой">
<meta name="twitter:image" content="https://shoes.com/images/twitter-preview.jpg">
```

<span class="text-gray">Open Graph и Twitter Cards не влияют на ранжирование напрямую, но определяют, как ссылка выглядит при шеринге — а это влияет на CTR и трафик.</span>

## Structured Data (JSON-LD)

Structured data помогает поисковику понять **смысл** контента: это товар, статья, рецепт, FAQ, событие? Результат — rich snippets (расширенные сниппеты) в выдаче: звёзды рейтинга, цена, FAQ-аккордеон, breadcrumbs.

```html
<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Nike Air Max 90",
  "image": "https://shoes.com/images/air-max-90.jpg",
  "description": "Классические кроссовки Nike Air Max 90",
  "brand": { "@type": "Brand", "name": "Nike" },
  "offers": {
    "@type": "Offer",
    "price": "12990",
    "priceCurrency": "RUB",
    "availability": "https://schema.org/InStock",
    "url": "https://shoes.com/nike-air-max-90"
  },
  "aggregateRating": {
    "@type": "AggregateRating",
    "ratingValue": "4.7",
    "reviewCount": "238"
  }
}
</script>
```

Популярные типы: `Article`, `Product`, `FAQPage`, `BreadcrumbList`, `Organization`, `HowTo`, `Event`.

> Используй [Rich Results Test](https://search.google.com/test/rich-results) от Google для проверки structured data.

## Semantic HTML и SEO

Поисковики используют семантику HTML для понимания структуры страницы.

<b class="text-on-semi-light">Заголовки</b>

<span class="text-gray">Один `<h1>` на страницу — основной заголовок. `<h2>`–`<h6>` — иерархия подзаголовков. Не пропускай уровни (h1 → h3 без h2). Не используй заголовки для стилизации — используй CSS.</span>

<b class="text-on-semi-light">Landmark-элементы</b>

<span class="text-gray">`<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>` — помогают боту понять структуру: где навигация, где основной контент, где sidebar.</span>

<b class="text-on-semi-light">Ссылки</b>

<span class="text-gray">Используй `<a href="/page">` для навигации, а не `<div onClick>`. Бот обходит страницы по `<a href>` — без них страница будет «сиротой» (orphan page), не найденной при crawl. Текст ссылки (anchor text) важен для ранжирования целевой страницы.</span>

<b class="text-on-semi-light">Изображения</b>

<span class="text-gray">`alt` — обязателен. Описывай содержание: `alt="Красные кроссовки Nike Air Max 90"`, а не `alt="photo"` или пустой `alt`. `alt` используется для индексации изображений и доступности.</span>

> Подробнее о семантическом HTML — [HTML](/content/html).

## URL, навигация и технические файлы

<b class="text-on-semi-light">URL structure</b>

<span class="text-gray">Чистые, читаемые URL лучше индексируются: `/catalog/nike-air-max-90` лучше, чем `/product?id=12345&ref=main`. Используй дефисы, не подчёркивания. Не меняй URL без редиректов (301).</span>

<b class="text-on-semi-light">Canonical URL</b>

<span class="text-gray">Если один контент доступен по нескольким URL (с параметрами, с www и без, http и https) — укажи canonical, чтобы поисковик не считал их дублями.</span>

```html
<!-- Страница /products?page=2&sort=price -->
<link rel="canonical" href="https://shoes.com/products?page=2">
```

<b class="text-on-semi-light">robots.txt</b>

```text
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /api/
Sitemap: https://shoes.com/sitemap.xml
```

<b class="text-on-semi-light">sitemap.xml</b>

<span class="text-gray">XML-файл со списком всех страниц сайта. Помогает боту найти страницы, до которых сложно добраться по ссылкам. Генерируется автоматически фреймворками (Nuxt: `@nuxtjs/sitemap`, Next.js: `next-sitemap`).</span>

```xml
<?xml version="1.0" encoding="UTF-8"?>
<urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <url>
    <loc>https://shoes.com/nike-air-max-90</loc>
    <lastmod>2025-03-15</lastmod>
    <priority>0.8</priority>
  </url>
</urlset>
```

<b class="text-on-semi-light">Breadcrumbs</b>

<span class="text-gray">Хлебные крошки помогают и пользователю, и поисковику понять иерархию сайта. Google может показать breadcrumbs в сниппете. Используй structured data `BreadcrumbList`.</span>

## Интернационализация (i18n)

Если сайт доступен на нескольких языках, нужно сообщить поисковику о языковых версиях.

```html
<!-- Указать язык страницы -->
<html lang="ru">

<!-- Альтернативные языковые версии -->
<link rel="alternate" hreflang="en" href="https://shoes.com/en/nike-air-max">
<link rel="alternate" hreflang="ru" href="https://shoes.com/ru/nike-air-max">
<link rel="alternate" hreflang="x-default" href="https://shoes.com/nike-air-max">
```

<span class="text-gray">`x-default` — страница по умолчанию для пользователей, чей язык не указан в альтернативах. `hreflang` должен быть **взаимным**: если страница A ссылается на B, то B должна ссылаться на A.</span>

## Производительность и ранжирование

С 2021 года **Core Web Vitals** (LCP, INP, CLS) — фактор ранжирования в Google. Медленный сайт ранжируется ниже при прочих равных. Также учитывается **mobile-friendliness** и **HTTPS**.

<span class="text-gray">Не стоит гнаться за идеальными метриками ради SEO — content relevance по-прежнему важнее. Но при прочих равных между двумя страницами Google выберет более быструю.</span>

> Подробнее о метриках — [Core Web Vitals](/content/web/core-web-vitals).

## Частые ошибки фронтендеров

| Ошибка | Почему это плохо | Как исправить |
|---|---|---|
| SPA без SSR | Бот видит пустой HTML | Nuxt/Next.js с SSR или SSG |
| `<div onClick>` вместо `<a href>` | Бот не переходит по ссылке | Использовать `<a>` или `<NuxtLink>` |
| Lazy-loading всего контента | Бот может не дождаться загрузки | Lazy-load только below-the-fold |
| Дублирование контента без canonical | Несколько URL с одним контентом делят вес | Указать `<link rel="canonical">` |
| Одинаковые title/description на всех страницах | Поисковик не различает страницы | Уникальные мета-теги на каждой странице |
| Скрытый текст через `display:none` | Может быть расценён как спам | Использовать `content-visibility: auto` |
| Бесконечная пагинация без URL | Контент ниже 1-й страницы не индексируется | `?page=N` или `rel="next/prev"` |
| Отсутствие `alt` на изображениях | Потеря image search трафика | Описательный `alt` на каждом `<img>` |

## SEO-чеклист для фронтенда

1. SSR или SSG для страниц, которые должны индексироваться.
2. Уникальные `<title>` и `<meta description>` на каждой странице.
3. `<link rel="canonical">` на каждой странице.
4. Один `<h1>`, логичная иерархия `<h2>`–`<h6>`.
5. Семантические элементы (`<main>`, `<article>`, `<nav>`).
6. Навигация через `<a href>`, не через JS click handlers.
7. `alt` на всех `<img>`.
8. `sitemap.xml` и `robots.txt`.
9. Structured data (JSON-LD) где применимо.
10. `hreflang` для мультиязычных сайтов.
11. HTTPS.
12. Mobile-friendly (`<meta name="viewport">`).
