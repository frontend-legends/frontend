# Web

**Web** — это раздел о веб-платформе как среде выполнения. Здесь разбираем, как устроен браузер изнутри, как страница превращается из HTML в пиксели, как взаимодействовать с DOM, какие API предоставляет платформа, как измерять и отлаживать производительность, и куда движется веб.

---

## Зачем это нужно senior-разработчику?

- Без знания Critical Rendering Path невозможно осознанно оптимизировать производительность.
- Вопрос «что происходит, когда ты вводишь URL?» — один из самых частых на senior-интервью.
- Core Web Vitals влияют на SEO и задаются как метрики в каждой крупной компании.
- V8-оптимизации, event delegation, AbortController, Service Workers — всё это отличает senior от middle.

---

## Что здесь есть

- Основы веба и архитектуры рендеринга (SPA, SSR, SSG).
- Архитектура браузера, движки и V8 internals.
- Rendering pipeline: парсинг, DOM, CSSOM, layout, paint, compositing.
- DOM и событийная модель: delegation, passive listeners, AbortController.
- Browser APIs: IntersectionObserver, Performance API, Streams и другие.
- Core Web Vitals: LCP, INP, CLS — диагностика и оптимизация.
- DevTools: профилирование, поиск утечек памяти, debugging.
- PWA и Service Workers: кэширование, offline, push.
- Будущее веба: WASM, Web Components, WebGPU, новые API.
