# 💻 Computer Science

Computer Science — это фундамент, на котором строится всё остальное.  
Для **Frontend Engineer / Software Engineer** понимание основ CS — не академическая теория, а навык, который напрямую проверяется на собеседованиях и используется в реальной работе.  

---

## 🎯 Зачем это важно?

- На интервью почти всегда встречаются вопросы по **time complexity**, **data structures** и **memory**  
- Без этих знаний невозможно уверенно решать **алгоритмические задачи**  
- Это база для понимания **performance**, **architecture** и **system design**  

---

## 📋 Что внутри этой главы?

- `01.01 Time Complexity` — Big-O, анализ алгоритмов  
- `01.02 Space Complexity` — работа с памятью, trade-offs  
- `01.03 Recursion` — стек вызовов, примеры, хвостовая рекурсия  
- `01.04 Iteration vs Recursion` — сравнительный анализ  
- `01.05 Hashing` — hash-функции, коллизии, практические примеры  
- `01.06 Memory Management` — heap, stack, garbage collection  
- `01.07 Complexity in Practice` — как объяснять сложность кода на интервью

## ⚡ Best Practices

::tip
- При решении задачи всегда **проговаривай сложность кода**  
- Учись объяснять решения **словами и на примерах**, а не только формулами  
- Не уходи слишком глубоко в теорию — важно уметь применять  
::

---

## 🚩 Частые ошибки

::warning
- Зубрить Big-O без практики → на интервью всё равно теряешься  
- Игнорировать **space complexity** → а она часто важнее  
- Путать стек вызовов и структуру данных "стек"  
::
