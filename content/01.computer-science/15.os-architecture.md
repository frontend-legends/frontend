# Архитектура операционных систем

## Введение

Операционная система (ОС) — это базовый слой программного обеспечения, управляющий ресурсами компьютера и обеспечивающий взаимодействие между пользователем, приложениями и аппаратным обеспечением. Архитектура ОС определяет, как организованы её компоненты, какие функции они выполняют и каким образом взаимодействуют между собой.

Знание архитектуры ОС важно программистам и инженерам, так как оно помогает понимать:

- как работает управление памятью;
- как планируются задачи;
- каким образом выполняются системные вызовы;
- где могут возникнуть узкие места в производительности.

---

## Основные функции операционной системы

- **Управление процессами**: создание, планирование, завершение процессов.
- **Управление памятью**: распределение, виртуализация, защита.
- **Файловая система**: хранение, организация, доступ.
- **Управление устройствами**: драйверы, взаимодействие с периферией.
- **Безопасность и защита**: контроль прав доступа, защита данных.
- **Интерфейс с пользователем**: командная строка, графическая оболочка, API для приложений.

## Основные архитектурные модели ОС

### Монолитное ядро

- Все основные службы ОС работают в пространстве ядра.
- Высокая производительность.
- Минус: сложность сопровождения, ошибки могут обрушить всю систему.
- Примеры: **Linux**, старые версии **UNIX**.

### Микроядро

- Ядро минимальное: управление памятью, межпроцессное взаимодействие, базовое планирование.
- Остальные сервисы работают в пространстве пользователя как отдельные процессы.
- Плюсы: надёжность, гибкость.
- Минусы: накладные расходы на IPC.
- Примеры: **Minix**, **QNX**, эксперименты Microsoft с **Windows NT**.

### Гибридные ядра

- Комбинируют монолитный и микроядерный подход.
- Некоторые сервисы работают в ядре, некоторые в пространстве пользователя.
- Примеры: **Windows NT/XP/10**, **XNU (macOS, iOS)**.

### Экзоядро (Exokernel)

- Ядро максимально простое: предоставляет только минимальный доступ к железу.
- Все абстракции (например, файловая система) реализуются библиотеками в пространстве пользователя.
- Пример: проекты **Exokernel** из MIT.

## Управление процессами

- **Процесс** — экземпляр программы в выполнении.
- ОС отвечает за:
  - запуск и завершение;
  - планирование (какому процессу дать CPU);
  - переключение контекста.

### Планировщики

- **Round Robin**: каждый процесс получает равный квант времени.
- **Priority Scheduling**: процессы с высоким приоритетом обслуживаются чаще.
- **Multilevel Queue**: разделение процессов по классам (системные, пользовательские).
- **Completely Fair Scheduler (CFS)**: современный планировщик Linux, балансирующий нагрузку.

Пример (Linux, вывод процессов):

```bash
ps -aux
```

## Управление памятью

- **Физическая память**: реальные ячейки RAM.
- **Виртуальная память**: отображение адресов процессов на физические страницы.
- **Механизмы**:
  - сегментация (устаревшая, в x86 уже почти не используется);
  - страничная организация памяти;
  - подкачка (swap).

Пример (Linux, проверка использования памяти):

```bash
free -h
top
```

---

## Файловые системы

- ОС предоставляет абстракцию «файл».
- Функции:
  - иерархическая структура (каталоги);
  - управление доступом (права);
  - надёжность (журналирование).

Примеры файловых систем:

- **ext4** (Linux)
- **NTFS** (Windows)
- **APFS** (macOS)

Пример (Linux, список файлов с правами доступа):

```bash
ls -l
```

---

## Ввод-вывод и драйверы

- ОС скрывает детали работы устройств через драйверы.
- Существует два уровня:
  - **пользовательский ввод-вывод** (системные вызовы, например `read`, `write`);
  - **низкоуровневый драйверный код** (работа с регистрами устройства).

Пример (Linux, просмотр устройств):

```bash
lsblk
```

---

## Безопасность и защита

- Механизмы безопасности:
  - разграничение прав доступа (пользователи, группы);
  - контроль привилегий (user mode vs kernel mode);
  - шифрование и контроль целостности.

- Пример (Linux, смена прав доступа):

```bash
chmod 644 file.txt
```

---

## Взаимодействие с пользователем

- **CLI** (Command Line Interface): shell-интерфейс, например Bash, PowerShell.
- **GUI** (Graphical User Interface): рабочие столы, окна, иконки.
- **API**: системные вызовы, библиотеки.

Пример (Windows, API вызов):

```c
#include <windows.h>

int main() {
    MessageBox(NULL, "Hello, World!", "Demo", MB_OK);
    return 0;
}
```

## Современные тенденции в архитектуре ОС

- Контейнеризация (**Docker**, **Kubernetes**).
- Микросервисы и облегчённые ядра для IoT.
- Виртуализация (Hyper-V, KVM, VMware).
- Облачные ОС и распределённые ресурсы.
