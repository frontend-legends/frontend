# Потоки и параллелизм

## Введение

- Что такое поток (thread) и чем он отличается от процесса.
- Основные цели: эффективное использование ресурсов, ускорение выполнения задач.
- Где это встречается в реальной жизни: веб-серверы, игры, обработка данных, UI-приложения.

## Многозадачность

- Различие между конкурентностью (concurrency) и параллелизмом (parallelism).
- Кооперативная и вытесняющая многозадачность.
- Примеры: однопоточность с асинхронностью (JavaScript) vs. реальный параллелизм (Java, C++, Go).

## Потоки

- Поток как единица выполнения внутри процесса.
- Общая память процесса и проблемы, связанные с разделяемым состоянием.
- Создание потоков в разных языках.

### Python

```python
import threading

def worker():
    print("Hello from thread!")

thread = threading.Thread(target=worker)
thread.start()
thread.join()
```

### Java

```java
class MyThread extends Thread {
    public void run() {
        System.out.println("Hello from thread!");
    }
}

public class Main {
    public static void main(String[] args) throws InterruptedException {
        Thread t = new MyThread();
        t.start();
        t.join();
    }
}
```

### C++

```cpp
#include <iostream>
#include <thread>

void worker() {
    std::cout << "Hello from thread!" << std::endl;
}

int main() {
    std::thread t(worker);
    t.join();
    return 0;
}
```

### Go

```go
package main

import (
    "fmt"
    "time"
)

func worker() {
    fmt.Println("Hello from goroutine!")
}

func main() {
    go worker()
    time.Sleep(time.Second)
}
```

## Проблемы многопоточности

- Гонки данных (race conditions).
- Взаимные блокировки (deadlocks).
- Пример гонки данных на Python:

```python
import threading

counter = 0

def increment():
    global counter
    for _ in range(100000):
        counter += 1

threads = [threading.Thread(target=increment) for _ in range(2)]
for t in threads: t.start()
for t in threads: t.join()
print(counter)  # результат может быть меньше 200000
```

### Java Deadlock пример

```java
class A {}
class B {}

public class DeadlockDemo {
    private final A a = new A();
    private final B b = new B();

    public void method1() {
        synchronized(a) {
            try { Thread.sleep(100); } catch (InterruptedException e) {}
            synchronized(b) {
                System.out.println("method1");
            }
        }
    }

    public void method2() {
        synchronized(b) {
            try { Thread.sleep(100); } catch (InterruptedException e) {}
            synchronized(a) {
                System.out.println("method2");
            }
        }
    }

    public static void main(String[] args) {
        DeadlockDemo demo = new DeadlockDemo();
        new Thread(demo::method1).start();
        new Thread(demo::method2).start();
    }
}
```

## Механизмы синхронизации

- Мьютексы (mutex), семафоры, блокировки.

### Python Lock

```python
import threading

lock = threading.Lock()
counter = 0

def increment():
    global counter
    for _ in range(100000):
        with lock:
            counter += 1
```

### C++ Mutex

```cpp
#include <iostream>
#include <thread>
#include <mutex>

std::mutex m;
int counter = 0;

void increment() {
    for (int i = 0; i < 100000; ++i) {
        std::lock_guard<std::mutex> lock(m);
        counter++;
    }
}

int main() {
    std::thread t1(increment);
    std::thread t2(increment);
    t1.join();
    t2.join();
    std::cout << counter << std::endl;
}
```

### Go Channels (альтернатива блокировкам)

```go
package main

import "fmt"

func main() {
    ch := make(chan int)
    go func() {
        ch <- 42
    }()
    val := <-ch
    fmt.Println(val)
}
```

## Асинхронное программирование

- Отличие потоков от асинхронных задач.
- Event loop.

### JavaScript

```javascript
function task(name) {
  return new Promise((resolve) => {
    console.log(`Start ${name}`);
    setTimeout(() => {
      console.log(`End ${name}`);
      resolve();
    }, 1000);
  });
}

Promise.all([task("A"), task("B")]);
```

### Python asyncio

```python
import asyncio

async def task(name):
    print(f"Start {name}")
    await asyncio.sleep(1)
    print(f"End {name}")

asyncio.run(asyncio.gather(task("A"), task("B")))
```

## Параллельные вычисления

- Использование нескольких ядер CPU.
- SIMD и GPU-параллелизм.
- Библиотеки: OpenMP, CUDA.

### C++ OpenMP

```cpp
#include <omp.h>
#include <iostream>

int main() {
    #pragma omp parallel for
    for (int i = 0; i < 8; i++) {
        std::cout << "Thread " << omp_get_thread_num() << " i=" << i << std::endl;
    }
}
```
