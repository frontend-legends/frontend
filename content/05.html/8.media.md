# Медиа

HTML предоставляет богатый набор элементов для работы с изображениями, видео, аудио и графикой. На senior-уровне ключевые темы — это **responsive images** (srcset, `<picture>`), **оптимизация** (lazy loading, modern formats, CLS prevention) и **безопасность iframe**.

---

## `<img>`

```html
<img
  src="photo.jpg"
  alt="Описание изображения"
  width="800"
  height="600"
  loading="lazy"
  decoding="async"
  fetchpriority="high"
>
```

### Ключевые атрибуты

| Атрибут | Назначение | Значения |
|---|---|---|
| `src` | URL изображения | Абсолютный или относительный путь |
| `alt` | Альтернативный текст (a11y + SEO) | Описание содержимого. `alt=""` для декоративных |
| `width` / `height` | Внутренние размеры | Числа (без `px`). **Предотвращают CLS** |
| `loading` | Стратегия загрузки | `lazy` (отложенная) / `eager` (немедленная, по умолчанию) |
| `decoding` | Стратегия декодирования | `async` (не блокирует рендеринг) / `sync` / `auto` |
| `fetchpriority` | Приоритет загрузки | `high` / `low` / `auto` |
| `srcset` | Набор вариантов изображения | Для responsive images |
| `sizes` | Размеры viewport → ширина изображения | Для responsive images |

<b class="text-on-semi-light">Предотвращение CLS:</b> Всегда указывай `width` и `height`. Браузер зарезервирует место до загрузки изображения, предотвращая сдвиг контента (Cumulative Layout Shift).

```html
<!-- Без width/height: контент прыгает при загрузке -->
<img src="photo.jpg" alt="...">

<!-- С width/height: место зарезервировано -->
<img src="photo.jpg" alt="..." width="800" height="600">
```

<span class="text-gray">`loading="lazy"` не применяй к hero image (above the fold) — оно должно загрузиться сразу. Для hero используй `fetchpriority="high"` + `loading="eager"` (или не указывай `loading`).</span>

## Responsive images

Две задачи, два подхода:

### 1. Resolution switching (srcset + sizes)

Один и тот же контент в разных разрешениях. Браузер сам выбирает оптимальный вариант.

```html
<img
  srcset="photo-400.jpg 400w,
          photo-800.jpg 800w,
          photo-1200.jpg 1200w"
  sizes="(max-width: 600px) 100vw,
         (max-width: 1200px) 50vw,
         33vw"
  src="photo-800.jpg"
  alt="Фотография"
>
```

<b class="text-on-semi-light">Как работает:</b>

1. `sizes` сообщает браузеру, **какую ширину** займёт изображение при данном viewport
2. `srcset` перечисляет доступные варианты с их **реальной шириной** (`w` дескриптор)
3. Браузер выбирает оптимальный, учитывая viewport, DPR и сетевые условия

<span class="text-gray">`src` — fallback для браузеров без поддержки `srcset` (сейчас таких почти нет). Дескриптор `2x`/`3x` — альтернатива `w` для фиксированных размеров: `srcset="logo.png 1x, logo@2x.png 2x"`.</span>

### 2. Art direction (`<picture>`)

Разный контент для разных viewport-ов (кадрирование, ориентация) или разные форматы.

```html
<!-- Art direction: разный кроп для мобильных -->
<picture>
  <source media="(max-width: 600px)" srcset="photo-mobile.jpg">
  <source media="(max-width: 1200px)" srcset="photo-tablet.jpg">
  <img src="photo-desktop.jpg" alt="Фотография">
</picture>

<!-- Формат-фолбэк: AVIF → WebP → JPEG -->
<picture>
  <source srcset="photo.avif" type="image/avif">
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="Фотография">
</picture>
```

### Когда что использовать

| Задача | Подход | Почему |
|---|---|---|
| Одна картинка в разных разрешениях | `srcset` + `sizes` | Браузер сам оптимизирует выбор |
| Разное кадрирование для мобильных | `<picture>` + `media` | Нужен контроль над выбором |
| Современные форматы с fallback | `<picture>` + `type` | Браузер пропустит неподдерживаемый формат |
| Фиксированный размер, только DPR | `srcset` + `x` дескриптор | Простейший случай (логотипы, иконки) |

## Современные форматы изображений

| Формат | Сжатие | Прозрачность | Анимация | Размер vs JPEG | Поддержка |
|---|---|---|---|---|---|
| **JPEG** | С потерями | Нет | Нет | Baseline | Все |
| **PNG** | Без потерь | Да | Нет | 2–5x больше | Все |
| **GIF** | Без потерь (256 цветов) | Да (1 бит) | Да | Зависит | Все |
| **WebP** | Оба | Да | Да | ~25–35% меньше | 97%+ |
| **AVIF** | С потерями | Да | Да | ~50% меньше | 93%+ |

<b class="text-on-semi-light">Рекомендация:</b> AVIF → WebP → JPEG/PNG через `<picture>`. AVIF даёт лучшее качество при минимальном размере, но медленнее кодируется.

## `<video>` и `<audio>`

```html
<video
  controls
  width="640"
  height="360"
  poster="preview.jpg"
  preload="metadata"
  muted
>
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <track src="subtitles-ru.vtt" kind="subtitles" srclang="ru" label="Русские" default>
  <p>Ваш браузер не поддерживает видео.</p>
</video>

<audio controls preload="none">
  <source src="podcast.mp3" type="audio/mpeg">
  <source src="podcast.ogg" type="audio/ogg">
</audio>
```

| Атрибут | Назначение |
|---|---|
| `controls` | Показать нативные элементы управления |
| `autoplay` | Автовоспроизведение (требует `muted` в большинстве браузеров) |
| `muted` | Без звука по умолчанию |
| `loop` | Зацикленное воспроизведение |
| `poster` | Превью-изображение (только video) |
| `preload` | `none` / `metadata` / `auto` — что загружать заранее |
| `playsinline` | Воспроизведение inline на iOS (без полноэкранного режима) |

<span class="text-gray">`<track>` добавляет субтитры в формате WebVTT. `kind` может быть `subtitles`, `captions` (для глухих — включает звуковые описания), `descriptions`, `chapters`.</span>

## SVG

**SVG (Scalable Vector Graphics)** — векторная графика, описанная XML. Масштабируется без потери качества, стилизуется через CSS, анимируется, доступна для скринридеров.

### Способы вставки

| Способ | Стилизация CSS | JS-доступ | Кэширование | Использование |
|---|---|---|---|---|
| Inline `<svg>` | Полная | Полный | Нет (в HTML) | Иконки, интерактивная графика |
| `<img src="icon.svg">` | Нет | Нет | Да | Декоративные изображения |
| CSS `background-image` | Нет | Нет | Да | Паттерны, декор |
| `<object>` / `<embed>` | Частичная | Через contentDocument | Да | Редко используется |

### Доступность SVG

```html
<!-- Inline SVG с a11y -->
<svg role="img" aria-label="Иконка корзины" viewBox="0 0 24 24">
  <title>Корзина</title>
  <path d="M7 18c-1.1 0-2 .9-2 2s.9 2 2 2..."/>
</svg>

<!-- Декоративная SVG (скрыта от скринридера) -->
<svg aria-hidden="true" focusable="false">
  <path d="..."/>
</svg>
```

<b class="text-on-semi-light">Ключевое правило:</b> Смысловые SVG — `role="img"` + `aria-label` (или `<title>`). Декоративные — `aria-hidden="true"`.

## `<canvas>`

`<canvas>` — пиксельная графика, рисуемая через JavaScript. В отличие от SVG, содержимое canvas — это **bitmap**, не DOM-элементы.

```html
<canvas id="chart" width="400" height="300">
  Ваш браузер не поддерживает canvas
</canvas>
```

```js
const canvas = document.getElementById('chart');
const ctx = canvas.getContext('2d');

ctx.fillStyle = '#1a1a2e';
ctx.fillRect(0, 0, 400, 300);

ctx.fillStyle = '#e94560';
ctx.beginPath();
ctx.arc(200, 150, 80, 0, Math.PI * 2);
ctx.fill();
```

### SVG vs Canvas

| Критерий | SVG | Canvas |
|---|---|---|
| Модель | Retained (DOM-элементы) | Immediate (пиксели) |
| Масштабирование | Без потерь | С потерями (растр) |
| Стилизация | CSS | Только JS API |
| Доступность | Да (DOM, ARIA) | Нет (нужен fallback content) |
| Много объектов (>1000) | Медленно (DOM-нагрузка) | Быстро |
| Интерактивность | Нативные события на элементах | Hit-testing вручную |
| Применение | Иконки, диаграммы, UI | Игры, визуализации, image processing |

## `<iframe>`

`<iframe>` встраивает внешний документ. Главные вопросы на senior-уровне — **безопасность**.

```html
<iframe
  src="https://widget.example.com"
  width="600"
  height="400"
  loading="lazy"
  sandbox="allow-scripts allow-forms"
  allow="camera; microphone"
  referrerpolicy="no-referrer"
  title="Виджет обратной связи"
></iframe>
```

### `sandbox`

Без значения — **максимальные ограничения**: нет JS, нет форм, нет навигации, нет попапов.

| Разрешение | Что даёт |
|---|---|
| `allow-scripts` | Выполнение JavaScript |
| `allow-forms` | Отправка форм |
| `allow-same-origin` | Обращение к parent origin (осторожно с `allow-scripts`!) |
| `allow-popups` | `window.open()`, `target="_blank"` |
| `allow-modals` | `alert()`, `confirm()`, `prompt()` |
| `allow-top-navigation` | Перенаправление parent window |

<b class="text-on-semi-light">Опасная комбинация:</b> `sandbox="allow-scripts allow-same-origin"` позволяет iframe удалить собственный `sandbox` атрибут. Не используй вместе для недоверенного контента.

### `postMessage` коммуникация

```js
// Parent → iframe
const iframe = document.querySelector('iframe');
iframe.contentWindow.postMessage({ action: 'init' }, 'https://widget.example.com');

// Iframe → parent
window.parent.postMessage({ result: 'ok' }, 'https://parent.example.com');

// Получение сообщений (всегда проверяй origin!)
window.addEventListener('message', (e) => {
  if (e.origin !== 'https://widget.example.com') return; // security check
  console.log(e.data);
});
```

### CSP и iframe

```html
<!-- CSP: запрещает встраивание страницы в чужие iframe -->
<meta http-equiv="Content-Security-Policy" content="frame-ancestors 'self'">

<!-- CSP: разрешает загрузку iframe только с определённых доменов -->
<meta http-equiv="Content-Security-Policy" content="frame-src https://trusted.example.com">
```

<span class="text-gray">`X-Frame-Options: DENY` / `SAMEORIGIN` — устаревший аналог `frame-ancestors`. CSP `frame-ancestors` предпочтительнее.</span>

## Частые вопросы на собесе

**В чём разница между `srcset` и `<picture>`?**
`srcset` + `sizes` — браузер **сам выбирает** оптимальное разрешение (resolution switching). `<picture>` — **разработчик контролирует** выбор: art direction (разный кроп) или format fallback (AVIF → WebP → JPEG).

**Как предотвратить CLS от изображений?**
Указать `width` и `height` на `<img>`. Браузер вычислит aspect ratio и зарезервирует место. Или использовать CSS `aspect-ratio`.

**Когда использовать lazy loading?**
Для изображений ниже fold (не видны при загрузке). **Не для** hero image, логотипа, LCP-элемента — им нужна немедленная загрузка (`fetchpriority="high"`).

**SVG или Canvas?**
SVG — для иконок, диаграмм, UI-элементов (масштабируемость, доступность, CSS-стилизация). Canvas — для тысяч объектов, игр, image processing (производительность пиксельной отрисовки).

**Как обезопасить iframe?**
`sandbox` (минимальные права), `allow` (permissions policy), CSP `frame-ancestors` (защита от clickjacking), проверка `origin` в `postMessage`. Никогда `allow-scripts` + `allow-same-origin` для недоверенного контента.
