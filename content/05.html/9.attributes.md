# Web Components

Web Components — это набор **нативных** браузерных технологий для создания переиспользуемых, инкапсулированных компонентов без фреймворков. Три столпа: **Custom Elements**, **Shadow DOM** и **HTML Templates**. На senior-собесе Web Components спрашивают в контексте дизайн-систем, микрофронтендов и понимания разницы с Virtual DOM.

---

## Что такое Web Components

Это **стандарт платформы**, а не библиотека. Компоненты работают в любом фреймворке (React, Vue, Angular) и без него.

| Технология | Что делает |
|---|---|
| **Custom Elements** | Позволяет создавать свои HTML-теги с кастомным поведением |
| **Shadow DOM** | Инкапсулирует стили и разметку — они не «протекают» наружу |
| **HTML Templates** | `<template>` и `<slot>` — инертная разметка и точки подстановки контента |

```html
<!-- Использование кастомного элемента — как обычный HTML-тег -->
<user-card name="Alex" role="Frontend Engineer"></user-card>
```

## Custom Elements

### Создание

```js
class UserCard extends HTMLElement {
  // Наблюдаемые атрибуты — только они вызывают attributeChangedCallback
  static observedAttributes = ['name', 'role'];

  constructor() {
    super();
    // Инициализация. НЕ трогать атрибуты и DOM здесь.
  }

  connectedCallback() {
    // Элемент добавлен в DOM — безопасно работать с DOM и атрибутами
    this.render();
  }

  disconnectedCallback() {
    // Элемент удалён из DOM — cleanup (отписки, таймеры)
  }

  attributeChangedCallback(name, oldValue, newValue) {
    // Наблюдаемый атрибут изменился
    if (oldValue !== newValue) this.render();
  }

  render() {
    const name = this.getAttribute('name') || 'Unknown';
    const role = this.getAttribute('role') || '';
    this.innerHTML = `
      <div class="card">
        <h3>${name}</h3>
        <p>${role}</p>
      </div>
    `;
  }
}

// Регистрация — имя ОБЯЗАНО содержать дефис
customElements.define('user-card', UserCard);
```

### Lifecycle callbacks

| Callback | Когда вызывается | Типичное использование |
|---|---|---|
| `constructor()` | Создание экземпляра | Инициализация полей, Shadow DOM. **Не** менять атрибуты/children |
| `connectedCallback()` | Элемент добавлен в DOM | Рендеринг, подписки, fetch данных |
| `disconnectedCallback()` | Элемент удалён из DOM | Отписки, cleanup таймеров и обработчиков |
| `attributeChangedCallback(name, old, new)` | Изменён наблюдаемый атрибут | Перерисовка по новым данным |
| `adoptedCallback()` | Элемент перемещён в новый document | Редко используется (iframe) |

<b class="text-on-semi-light">Правила именования:</b> Имя тега **обязано содержать дефис** (`my-component`, `app-header`). Это предотвращает конфликты с будущими HTML-тегами. `usercard` — невалидно, `user-card` — ок.

### Autonomous vs Customized Built-in

```js
// Autonomous: полностью кастомный элемент
class MyButton extends HTMLElement { /* ... */ }
customElements.define('my-button', MyButton);
// <my-button>Click</my-button>

// Customized built-in: расширяет нативный элемент
class FancyButton extends HTMLButtonElement { /* ... */ }
customElements.define('fancy-button', FancyButton, { extends: 'button' });
// <button is="fancy-button">Click</button>
```

<span class="text-gray">Customized built-in elements **не поддерживаются Safari** (и Apple отказалась реализовывать). На практике используют только autonomous custom elements.</span>

## Shadow DOM

Shadow DOM создаёт **изолированное DOM-дерево** внутри элемента. Стили внутри Shadow DOM не влияют на внешний документ, и наоборот.

### Создание

```js
class MyComponent extends HTMLElement {
  constructor() {
    super();
    // attachShadow создаёт shadow root
    const shadow = this.attachShadow({ mode: 'open' });

    shadow.innerHTML = `
      <style>
        /* Эти стили не вытекают наружу */
        p { color: red; font-weight: bold; }
      </style>
      <p>Я внутри Shadow DOM</p>
    `;
  }
}

customElements.define('my-component', MyComponent);
```

### Open vs Closed

| Mode | `element.shadowRoot` | Доступ извне |
|---|---|---|
| `open` | Возвращает shadow root | Можно читать и изменять |
| `closed` | `null` | Нельзя (почти — через constructor ref можно обойти) |

<span class="text-gray">На практике почти всегда `open`. `closed` даёт ложное чувство безопасности — это не security boundary. Браузерные built-in элементы (`<video>`, `<input type="date">`) используют closed shadow DOM.</span>

### Light DOM vs Shadow DOM

```html
<my-card>
  <!-- Это Light DOM — контент, переданный снаружи -->
  <span slot="title">Заголовок</span>
  <p>Описание</p>
</my-card>
```

**Light DOM** — контент между тегами компонента (потомки в обычном DOM). **Shadow DOM** — внутренняя «теневая» разметка, определённая в компоненте. `<slot>` соединяет их.

## `<template>` и `<slot>`

### `<template>`

`<template>` — **инертный** HTML: не рендерится, не выполняет скрипты, не загружает изображения. Идеально для шаблонов, которые клонируются по необходимости.

```html
<template id="card-template">
  <div class="card">
    <h3><slot name="title">Default Title</slot></h3>
    <p><slot>Default content</slot></p>
  </div>
</template>
```

```js
class MyCard extends HTMLElement {
  constructor() {
    super();
    const shadow = this.attachShadow({ mode: 'open' });
    const template = document.getElementById('card-template');
    shadow.appendChild(template.content.cloneNode(true));
  }
}

customElements.define('my-card', MyCard);
```

### `<slot>`

Точка подстановки — определяет, **где** Light DOM контент появится внутри Shadow DOM.

```js
// Компонент
class AlertBox extends HTMLElement {
  constructor() {
    super();
    this.attachShadow({ mode: 'open' }).innerHTML = `
      <style>
        .alert { padding: 16px; border: 2px solid; border-radius: 8px; }
      </style>
      <div class="alert">
        <strong><slot name="title">Внимание</slot></strong>
        <div><slot>Текст по умолчанию</slot></div>
      </div>
    `;
  }
}
customElements.define('alert-box', AlertBox);
```

```html
<!-- Использование -->
<alert-box>
  <span slot="title">Ошибка!</span>
  Не удалось сохранить данные.
</alert-box>

<!-- С дефолтами -->
<alert-box></alert-box>
<!-- Покажет: "Внимание" + "Текст по умолчанию" -->
```

| Тип слота | Синтаксис | Что подставляется |
|---|---|---|
| Default (unnamed) | `<slot>` | Все Light DOM children без `slot="..."` |
| Named | `<slot name="title">` | `<* slot="title">` из Light DOM |

## CSS и Shadow DOM

### Инкапсуляция

Стили внутри Shadow DOM **не протекают** наружу, внешние стили **не проникают** внутрь. Исключения:

- **Наследуемые свойства** (`color`, `font-family`, `line-height`) наследуются через shadow boundary
- **CSS Custom Properties** (`--var`) проникают через boundary

```css
/* Внешний CSS — НЕ повлияет на Shadow DOM */
p { color: blue; }

/* Но custom properties — проникают */
:root { --primary: #e94560; }
```

```js
// Внутри Shadow DOM
this.shadowRoot.innerHTML = `
  <style>
    p { color: var(--primary, red); } /* Получит --primary от :root */
  </style>
  <p>Текст</p>
`;
```

### Специальные селекторы

| Селектор | Где работает | Что выбирает |
|---|---|---|
| `:host` | Внутри Shadow DOM | Сам компонент (host element) |
| `:host(.active)` | Внутри Shadow DOM | Host с классом `.active` |
| `:host-context(.dark)` | Внутри Shadow DOM | Host, если один из предков имеет `.dark` |
| `::slotted(span)` | Внутри Shadow DOM | Прямые потомки, распределённые в `<slot>` |
| `::part(name)` | Снаружи | Элемент внутри Shadow DOM с `part="name"` |

### `::part()` — стилизация извне

```js
// Компонент выставляет part
this.shadowRoot.innerHTML = `
  <button part="btn">Click me</button>
`;
```

```css
/* Внешний CSS может стилизовать part */
my-component::part(btn) {
  background: var(--primary);
  color: white;
}
```

<b class="text-on-semi-light">Паттерн дизайн-системы:</b> Компонент определяет `part` на ключевых элементах и CSS custom properties для темизации. Потребители стилизуют через `::part()` и `--variables`. Это контролируемый API стилизации.

## Практический пример: Counter

```js
class ClickCounter extends HTMLElement {
  #count = 0;

  constructor() {
    super();
    this.attachShadow({ mode: 'open' });
  }

  connectedCallback() {
    this.render();
    this.shadowRoot.querySelector('button').addEventListener('click', () => {
      this.#count++;
      this.render();
      this.dispatchEvent(new CustomEvent('count-changed', {
        detail: { count: this.#count },
        bubbles: true,
        composed: true, // пробивается через shadow boundary
      }));
    });
  }

  render() {
    this.shadowRoot.innerHTML = `
      <style>
        :host {
          display: inline-flex;
          align-items: center;
          gap: 8px;
          font-family: inherit;
        }
        button {
          padding: 8px 16px;
          border: 2px solid var(--primary, #333);
          border-radius: 4px;
          background: transparent;
          color: inherit;
          cursor: pointer;
          font: inherit;
        }
        button:hover { background: var(--primary, #333); color: white; }
        span { min-width: 2ch; text-align: center; }
      </style>
      <button part="btn">+</button>
      <span part="count">${this.#count}</span>
    `;
  }
}

customElements.define('click-counter', ClickCounter);
```

```html
<!-- Использование -->
<click-counter></click-counter>

<script>
  document.querySelector('click-counter')
    .addEventListener('count-changed', e => {
      console.log('Count:', e.detail.count);
    });
</script>
```

<span class="text-gray">`composed: true` в CustomEvent позволяет событию пробиться через Shadow DOM boundary и всплыть к родительским элементам. Без этого событие останется внутри Shadow DOM.</span>

## На фронте: Web Components vs Framework Components

| Критерий | Web Components | React / Vue |
|---|---|---|
| Зависимости | Нет (нативный стандарт) | Фреймворк + bundler |
| Инкапсуляция стилей | Shadow DOM (нативная) | CSS Modules, Scoped CSS, CSS-in-JS |
| Реактивность | Вручную (attributeChangedCallback) | Встроенная (useState, ref) |
| SSR | Ограниченный (Declarative Shadow DOM) | Полноценный |
| Совместимость | Работает везде | Только внутри фреймворка |
| DX (Developer Experience) | Ниже (много boilerplate) | Выше (JSX, SFC, hot reload) |

<b class="text-on-semi-light">Когда использовать Web Components:</b>

- **Дизайн-системы** — компоненты, которые должны работать в разных фреймворках (React, Vue, Angular, vanilla)
- **Микрофронтенды** — изоляция стилей и логики между командами
- **Виджеты для третьих сторон** — embed-компоненты, которые не должны конфликтовать со стилями хоста
- **Расширения браузера** — инкапсуляция от стилей страницы

<b class="text-on-semi-light">Когда не стоит:</b>

- Внутри одного React/Vue приложения — фреймворковые компоненты удобнее и реактивнее
- Когда нужен SSR — WC с Shadow DOM плохо серверятся (Declarative Shadow DOM ещё не повсеместен)

### Библиотеки поверх Web Components

- **Lit** (Google) — минималистичная библиотека, убирает boilerplate, добавляет реактивность
- **Stencil** (Ionic) — компилятор Web Components с JSX, TypeScript, lazy loading
- **FAST** (Microsoft) — фреймворк для дизайн-систем на Web Components

## Частые вопросы на собесе

**Чем Shadow DOM отличается от Virtual DOM?**
Shadow DOM — **нативная инкапсуляция** (изоляция стилей и DOM). Virtual DOM — **абстракция для оптимизации** рендеринга (diffing в React). Shadow DOM реально существует в браузере, Virtual DOM — это JS-объект в памяти.

**Когда использовать Web Components?**
Дизайн-системы для нескольких фреймворков, микрофронтенды, embed-виджеты. Не для внутренних компонентов одного React/Vue приложения — там фреймворковые компоненты удобнее.

**Как стилизовать Web Component снаружи?**
Три способа: CSS custom properties (проникают через shadow boundary), `::part()` (стилизация по part-name), наследуемые CSS-свойства (color, font). Внешние классы и селекторы не работают.

**Что такое `composed: true` в CustomEvent?**
Позволяет событию пересечь Shadow DOM boundary и всплыть к родительским элементам. Без этого событие останется внутри Shadow DOM и не будет доступно извне.

**Почему имя Custom Element должно содержать дефис?**
Стандарт требует дефис (`my-component`), чтобы предотвратить конфликты с текущими и будущими HTML-тегами (все нативные теги однословные: `div`, `span`, `button`).
