# Атрибуты

HTML-атрибуты определяют свойства и поведение элементов. На senior-уровне ключевое — понимать разницу между **attribute и property** в DOM, знать gotchas булевых атрибутов и разбираться в атрибутах, связанных с безопасностью.

---

## Глобальные атрибуты

Атрибуты, которые можно использовать на **любом** HTML-элементе.

| Атрибут | Назначение | Пример |
|---|---|---|
| `id` | Уникальный идентификатор элемента | `id="main-title"` |
| `class` | CSS-классы (через пробел) | `class="btn btn-primary"` |
| `style` | Inline CSS-стили | `style="color: red"` |
| `title` | Всплывающая подсказка (tooltip) | `title="Подробнее"` |
| `tabindex` | Порядок фокуса с клавиатуры | `tabindex="0"` / `tabindex="-1"` |
| `hidden` | Скрывает элемент (аналог `display: none`) | `<p hidden>скрыто</p>` |
| `contenteditable` | Делает элемент редактируемым | `contenteditable="true"` |
| `draggable` | Разрешает drag & drop | `draggable="true"` |
| `dir` | Направление текста | `dir="rtl"` (арабский, иврит) |
| `lang` | Язык содержимого | `lang="en"` |
| `inert` | Отключает взаимодействие и a11y для поддерева | `<div inert>...</div>` |
| `popover` | Нативный Popover API | `popover="auto"` |

<b class="text-on-semi-light">`inert`</b> — относительно новый атрибут. Делает элемент и всех его потомков **неинтерактивными**: не фокусируются, не кликаются, невидимы для скринридеров. Идеален для модалок — всё кроме модалки помечается как `inert`.

```html
<!-- Всё кроме модалки неактивно -->
<main inert>
  <p>Этот контент неинтерактивен, пока модалка открыта</p>
</main>
<dialog open>
  <p>Модальное окно</p>
</dialog>
```

## `data-*` атрибуты

Кастомные атрибуты для хранения данных, привязанных к элементу. Любой атрибут с префиксом `data-` валиден.

```html
<button data-action="delete" data-item-id="42" data-confirm="true">
  Удалить
</button>
```

### Доступ из JavaScript

```js
const btn = document.querySelector('button');

// Через dataset (camelCase)
btn.dataset.action;    // "delete"
btn.dataset.itemId;    // "42" (data-item-id → itemId)
btn.dataset.confirm;   // "true" (всегда строка!)

// Через getAttribute
btn.getAttribute('data-action'); // "delete"
```

<b class="text-on-semi-light">Важно:</b> Значения `dataset` — **всегда строки**. `data-confirm="true"` → `"true"`, не `true`. `data-count="5"` → `"5"`, не `5`.

### Доступ из CSS

```css
/* Через attribute selector */
[data-action="delete"] { color: red; }

/* Вывод значения через attr() */
button::after { content: attr(data-action); }
```

<span class="text-gray">В React/Vue `data-*` атрибуты используются для тестирования: `data-testid="submit-btn"` — стабильный селектор для e2e тестов, не зависящий от CSS-классов.</span>

## Булевые атрибуты

Булевые атрибуты **не имеют значения** в обычном смысле. Их **наличие** означает `true`, **отсутствие** — `false`.

```html
<!-- Все три варианта эквивалентны — элемент disabled -->
<input disabled>
<input disabled="">
<input disabled="disabled">

<!-- disabled="false" — ЭТО ВСЁ ЕЩЁ DISABLED! -->
<input disabled="false">  <!-- disabled = true, потому что атрибут присутствует -->
```

<b class="text-on-semi-light">Ключевой gotcha:</b> `disabled="false"` **не отключает** атрибут. Парсер видит наличие атрибута → элемент disabled. Чтобы убрать — нужно удалить атрибут целиком.

### Основные булевые атрибуты

| Атрибут | Элементы | Назначение |
|---|---|---|
| `disabled` | `<input>`, `<button>`, `<select>`, `<textarea>`, `<fieldset>` | Отключает элемент |
| `checked` | `<input type="checkbox/radio">` | Предвыбранный |
| `selected` | `<option>` | Предвыбранный вариант |
| `readonly` | `<input>`, `<textarea>` | Только для чтения (отправляется в форме, в отличие от `disabled`) |
| `required` | `<input>`, `<select>`, `<textarea>` | Обязательное поле |
| `open` | `<details>`, `<dialog>` | Раскрыт/открыт |
| `hidden` | Любой элемент | Скрыт |
| `novalidate` | `<form>` | Отключает HTML5-валидацию |
| `autofocus` | `<input>`, `<button>`, `<textarea>`, `<select>` | Автофокус при загрузке |

### В JavaScript

```js
// Правильно: используй свойства DOM, не setAttribute
input.disabled = true;   // включить
input.disabled = false;  // выключить

// Или через атрибуты
input.setAttribute('disabled', '');  // включить
input.removeAttribute('disabled');   // выключить

// НЕ правильно:
input.setAttribute('disabled', 'false'); // ВСЁ ЕЩЁ disabled!
```

## Attribute vs Property (DOM)

Это **самый частый** вопрос по HTML на senior-собеседованиях.

### Определения

- **Attribute** — то, что написано в HTML-разметке. Хранится в DOM как `element.attributes`. Доступ: `getAttribute()` / `setAttribute()`.
- **Property** — свойство JavaScript-объекта (DOM-узла). Доступ: `element.property`.

<b class="text-on-semi-light">Ключевая идея:</b> Атрибуты определяют **начальное** состояние элемента. Свойства отражают **текущее** состояние.

### Когда они расходятся

```html
<input type="text" value="начальное">
```

```js
const input = document.querySelector('input');

// Начальное состояние: attribute = property
input.getAttribute('value'); // "начальное"
input.value;                  // "начальное"

// Пользователь вводит "новое"
input.getAttribute('value'); // "начальное" ← attribute НЕ меняется!
input.value;                  // "новое"     ← property отражает текущее

// Разница для checked:
const checkbox = document.querySelector('input[type="checkbox"]');
checkbox.getAttribute('checked'); // null или "" (начальное)
checkbox.checked;                  // true/false (текущее)
```

### Таблица расхождений

| Атрибут/Свойство | Attribute (`getAttribute`) | Property | Когда расходятся |
|---|---|---|---|
| `value` | Начальное значение | Текущее значение | После ввода пользователем |
| `checked` | Начальное состояние | Текущее состояние | После клика |
| `href` | Как записано в HTML | Полный URL | Если в HTML относительный путь |
| `class` | Строка классов | — | Property называется `className` (строка) или `classList` (объект) |
| `style` | Строка inline стилей | `CSSStyleDeclaration` объект | Разный тип данных |

### Синхронизация

Некоторые атрибуты и свойства **синхронизированы в одну сторону**, другие — в обе:

- `id` — двусторонняя синхронизация: изменение property обновляет attribute и наоборот
- `value` — **только attribute → property** (при инициализации). После этого они живут независимо
- `checked` — аналогично `value`

```js
// id: двусторонняя
element.id = 'new-id';
element.getAttribute('id'); // "new-id" ← обновился

// value: не синхронизирован после инициализации
input.value = 'новое';
input.getAttribute('value'); // по-прежнему "начальное"
```

<b class="text-on-semi-light">На фронте:</b> В React `defaultValue` задаёт attribute (начальное значение), а `value` — контролируемый property (текущее). Vue `v-model` работает с property (`value`), а `v-bind:value` — с attribute.

## Event-атрибуты

```html
<!-- Inline event handler (устаревший подход) -->
<button onclick="alert('Нажата')">Нажми</button>
```

<b class="text-on-semi-light">Почему не используются в современном коде:</b>

1. **Separation of concerns** — смешивание HTML и JS
2. **CSP** — `script-src` без `'unsafe-inline'` заблокирует inline handlers
3. **Ограничения** — нельзя использовать `addEventListener` (multiple listeners, options)
4. **Скоуп** — inline handler имеет доступ к `document` и `element`, но не к модульным переменным

```js
// Современный подход
element.addEventListener('click', handleClick);
element.addEventListener('click', trackAnalytics); // несколько обработчиков
```

## Атрибуты безопасности

### `rel="noopener noreferrer"`

```html
<a href="https://external.com" target="_blank" rel="noopener noreferrer">
  Внешняя ссылка
</a>
```

- `noopener` — запрещает новому окну доступ к `window.opener` (защита от tabnabbing)
- `noreferrer` — не отправляет заголовок `Referer` (приватность)

<span class="text-gray">Современные браузеры добавляют `noopener` для `target="_blank"` автоматически. Но явное указание — хорошая практика для совместимости.</span>

### `sandbox` (для `<iframe>`)

```html
<iframe src="widget.html" sandbox="allow-scripts allow-forms"></iframe>
```

Без значения — максимальные ограничения. Каждый `allow-*` снимает конкретное ограничение.

### `crossorigin`

```html
<img src="https://cdn.example.com/img.jpg" crossorigin="anonymous">
<script src="https://cdn.example.com/app.js" crossorigin="anonymous"></script>
```

Управляет CORS-запросом. Без `crossorigin` ошибки в скриптах с другого домена не показывают stack trace.

### `integrity` (Subresource Integrity)

```html
<script src="https://cdn.example.com/lib.js"
        integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8w"
        crossorigin="anonymous"></script>
```

Браузер сверяет хеш загруженного файла с указанным. Если не совпадает — файл не выполняется. Защита от CDN-компрометации.

## Частые вопросы на собесе

**В чём разница между attribute и property?**
Attribute — начальное значение из HTML-разметки (`getAttribute`). Property — текущее состояние JS-объекта (`element.prop`). Расходятся после изменения пользователем: `input.getAttribute('value')` = начальное, `input.value` = текущее.

**Что будет, если написать `disabled="false"`?**
Элемент останется disabled. Булевые атрибуты работают по наличию, а не по значению. Любое значение (включая `"false"`) = `true`. Для снятия нужно удалить атрибут.

**Зачем нужен `data-*`?**
Для хранения кастомных данных на элементе: конфигурация, идентификаторы, состояние. Доступ через `dataset` (JS) и `[data-*]` (CSS). Значения всегда строки. Частое применение — `data-testid` для e2e тестов.

**Что делает `rel="noopener"` и зачем?**
Запрещает целевому окну (`target="_blank"`) доступ к `window.opener` исходной страницы. Без него вредоносная страница может перенаправить opener на фишинговую страницу (tabnabbing attack).

**Что такое SRI (`integrity`)?**
Subresource Integrity — проверка целостности загруженного файла по хешу. Если CDN скомпрометирован и файл подменён, хеш не совпадёт и браузер откажется его выполнять.
