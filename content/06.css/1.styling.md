# Каскад и стилизация

Каскад — это **алгоритм**, по которому браузер определяет, какое CSS-правило применить к элементу, когда несколько правил конкурируют. Это не просто «последний побеждает» — каскад учитывает origin, layers, специфичность и порядок. Понимание каскада — то, что отличает senior CSS-разработчика от того, кто борется с `!important`.

---

## Как CSS применяется к HTML

```html
<!-- 1. External (внешний файл) — основной способ -->
<link rel="stylesheet" href="styles.css">

<!-- 2. Internal (внутренний, в <style>) -->
<style>
  .card { padding: 16px; }
</style>

<!-- 3. Inline (на элементе) -->
<div style="color: red;">Текст</div>
```

<span class="text-gray">`@import url('other.css')` внутри CSS — подключает другой файл, но **блокирует параллельную загрузку**. Браузер не может начать скачивать импортируемый файл, пока не распарсит текущий. Используй `<link>` вместо `@import`.</span>

## Каскад (Cascade)

Когда к элементу применяется несколько конфликтующих правил, браузер определяет победителя по **алгоритму каскада**. Шаги проверяются последовательно — каждый следующий используется только если предыдущий не дал победителя.

### Порядок приоритета (от низшего к высшему)

```text
1. Origin & Importance
   user-agent (браузер) → user (пользователь) → author (разработчик)
   !important инвертирует: author !important → user !important → user-agent !important
   Transition > всё.  Animation > всё кроме transition.

2. Cascade Layers (@layer)
   Порядок объявления слоёв. Un-layered (без @layer) побеждает все слои.

3. Specificity
   Inline > ID > class/attr/pseudo-class > element/pseudo-element

4. Order of Appearance
   Последнее объявление побеждает (при равной специфичности).
```

<b class="text-on-semi-light">Ключевая идея:</b> Каскад — это водопад. Вода (стили) течёт сверху вниз по уровням. На каждом уровне побеждает более «весомый» стиль. Если вес равен — побеждает последний.

<b class="text-on-semi-light">На фронте:</b> Когда стили из CSS-модулей «не работают» — причина обычно в порядке импорта (order of appearance) или специфичности стилей библиотеки компонентов. `@layer` решает это элегантно.

### `!important`

```css
.card { color: red !important; } /* побеждает обычные стили любой специфичности */
```

<span class="text-gray">`!important` инвертирует порядок origins: author `!important` побеждает author normal, но user `!important` побеждает author `!important`. Это спроектировано для accessibility — пользовательские стили для крупного шрифта или высокого контраста должны быть главнее.</span>

**Когда оправдан:** utility-классы (`!important` на `.hidden { display: none !important }`), переопределение third-party стилей, когда нет контроля над специфичностью.

**Когда нет:** почти всегда. Если нужен `!important` внутри своего кода — проблема в архитектуре стилей.

## Специфичность (Specificity)

Специфичность определяет приоритет правил на одном уровне каскада. Формат: `(A, B, C)`.

| Уровень | Что считается | Примеры |
|---|---|---|
| **A** (id) | Селекторы по `#id` | `#header` → (1,0,0) |
| **B** (class) | Классы, атрибуты, псевдоклассы | `.card`, `[type="text"]`, `:hover` → (0,1,0) |
| **C** (element) | Элементы, псевдоэлементы | `div`, `::before` → (0,0,1) |

### Примеры вычисления

| Селектор | A | B | C | Специфичность |
|---|---|---|---|---|
| `*` | 0 | 0 | 0 | (0,0,0) |
| `p` | 0 | 0 | 1 | (0,0,1) |
| `.card` | 0 | 1 | 0 | (0,1,0) |
| `p.card` | 0 | 1 | 1 | (0,1,1) |
| `#main` | 1 | 0 | 0 | (1,0,0) |
| `#main .card p` | 1 | 1 | 1 | (1,1,1) |
| `#main .card .title:hover` | 1 | 3 | 0 | (1,3,0) |
| `div#main > p.text::first-line` | 1 | 1 | 3 | (1,1,3) |

### Функциональные псевдоклассы и специфичность

| Псевдокласс | Специфичность |
|---|---|
| `:is(.a, #b)` | Берёт **максимальную** из аргументов → (1,0,0) |
| `:where(.a, #b)` | **Всегда 0** → (0,0,0) |
| `:not(.a)` | Берёт специфичность аргумента → (0,1,0) |
| `:has(.a)` | Берёт специфичность аргумента → (0,1,0) |

<b class="text-on-semi-light">Ключевой gotcha:</b> `#id` и `[id="foo"]` выбирают одно и то же, но специфичность разная:

```css
#header { color: red; }      /* (1,0,0) — побеждает */
[id="header"] { color: blue; } /* (0,1,0) */
```

**Плохо — война специфичностей:**

```css
#app #main .container .card .title { font-size: 18px; }
#app #main .container .card .title.active { font-size: 20px; }
/* Каждое новое правило вынуждено повторять длинную цепочку */
```

**Хорошо — плоская специфичность:**

```css
.card-title { font-size: 18px; }
.card-title.is-active { font-size: 20px; }
/* Минимальная, предсказуемая специфичность */
```

## Наследование (Inheritance)

Некоторые CSS-свойства наследуются от родителя к потомкам, другие — нет.

### Наследуемые (inherited)

`color`, `font-family`, `font-size`, `font-weight`, `font-style`, `line-height`, `text-align`, `text-indent`, `text-transform`, `letter-spacing`, `word-spacing`, `white-space`, `visibility`, `cursor`, `list-style`, `direction`

### Ненаследуемые (non-inherited)

`margin`, `padding`, `border`, `width`, `height`, `display`, `position`, `top`/`right`/`bottom`/`left`, `background`, `overflow`, `z-index`, `opacity`, `transform`, `flex`, `grid`

### Ключевые слова

| Значение | Что делает |
|---|---|
| `inherit` | Принудительно наследует значение от родителя |
| `initial` | Сбрасывает к **начальному** значению свойства (по спецификации CSS) |
| `unset` | Если свойство наследуемое → `inherit`, если нет → `initial` |
| `revert` | Откатывает к стилям **предыдущего origin** (обычно к браузерным) |
| `revert-layer` | Откатывает к стилям **предыдущего @layer** |

```css
/* Сброс всех стилей элемента к браузерным */
.reset { all: revert; }

/* Откат к предыдущему слою */
@layer components {
  .btn { color: revert-layer; /* берёт цвет из base-слоя */ }
}
```

## Box Model

Каждый элемент в CSS — это **прямоугольный бокс** с четырьмя областями:

```text
┌──────────────────────────────────────┐
│              margin                  │
│  ┌────────────────────────────────┐  │
│  │           border               │  │
│  │  ┌──────────────────────────┐  │  │
│  │  │        padding           │  │  │
│  │  │  ┌──────────────────┐   │  │  │
│  │  │  │     content      │   │  │  │
│  │  │  │                  │   │  │  │
│  │  │  └──────────────────┘   │  │  │
│  │  └──────────────────────────┘  │  │
│  └────────────────────────────────┘  │
└──────────────────────────────────────┘
```

### `box-sizing`

| Значение | `width` включает |
|---|---|
| `content-box` (default) | Только content. Padding и border **добавляются сверху** |
| `border-box` | Content + padding + border. Предсказуемая ширина |

```css
/* Универсальный reset — стандарт де-факто */
*, *::before, *::after {
  box-sizing: border-box;
}
```

### Margin Collapse (схлопывание margins)

Вертикальные margins соседних блоков **не складываются, а схлопываются** — берётся больший из двух.

<b class="text-on-semi-light">Ключевая идея:</b> Два margin'а встретились — побеждает больший, а не сумма.

<b class="text-on-semi-light">Как запомнить:</b> Два человека стоят друг напротив друга с вытянутыми руками. Расстояние между ними определяется длиной рук того, у кого руки длиннее — а не суммой обеих.

#### Когда СХЛОПЫВАЕТСЯ

- **Соседние siblings:** margin-bottom верхнего + margin-top нижнего
- **Parent + first/last child:** margin-top родителя + margin-top первого ребёнка (если между ними нет padding, border, inline content)
- **Пустой блок:** собственные margin-top и margin-bottom

```css
/* margin-bottom: 20px + margin-top: 30px = итого 30px (не 50px) */
.box-a { margin-bottom: 20px; }
.box-b { margin-top: 30px; }
```

#### Когда НЕ схлопывается

- **Flex/Grid items** — flex и grid контейнеры создают новый formatting context
- **BFC boundary** — элемент с `overflow` != visible, `display: flow-root`
- **Float** и **absolute/fixed** elements
- **Inline-block** elements
- **Padding или border** между parent и child (разделяет margins)

<b class="text-on-semi-light">На фронте:</b> Неожиданные отступы в списках карточек, «проваливающиеся» margin'ы через родителя — классические баги от margin collapse. Flexbox и Grid полностью устраняют проблему — используй `gap` вместо margin.

## Единицы измерения

| Категория | Единицы | Описание |
|---|---|---|
| **Absolute** | `px` | 1px = 1/96 дюйма (на CSS-уровне, не пиксель экрана) |
| **Relative (font)** | `em` | Относительно font-size родителя (или самого элемента для font-size) |
| | `rem` | Относительно font-size `<html>` (root). **Предсказуемый** |
| | `ch` | Ширина символа `0`. Для ограничения ширины текста |
| | `lh` | Высота line-height элемента |
| **Relative (%)** | `%` | От размера родителя (зависит от контекста) |
| **Viewport** | `vw`, `vh` | 1% ширины/высоты viewport |
| | `dvh`, `svh`, `lvh` | Dynamic/Small/Large viewport height (мобильная address bar) |
| | `vmin`, `vmax` | Меньшее/большее из vw и vh |

### CSS-функции для значений

```css
width: calc(100% - 32px);           /* вычисление */
width: min(600px, 100%);            /* меньшее из двух */
width: max(300px, 50%);             /* большее из двух */
font-size: clamp(1rem, 2.5vw, 2rem); /* min, preferred, max */
```

<b class="text-on-semi-light">`rem` vs `em`:</b> `rem` — предсказуемый, привязан к root. Используй для spacing, font-size. `em` — масштабируется с font-size элемента. Используй для padding/margin, которые должны масштабироваться с текстом (например, padding кнопки).

## Цвета

| Формат | Синтаксис | Пример |
|---|---|---|
| **Hex** | `#RRGGBB` / `#RGB` / `#RRGGBBAA` | `#e94560`, `#fff`, `#0008` |
| **RGB** | `rgb(R G B)` / `rgb(R G B / A)` | `rgb(233 69 96)`, `rgb(0 0 0 / 0.5)` |
| **HSL** | `hsl(H S L)` / `hsl(H S L / A)` | `hsl(350 80% 59%)` |
| **Named** | Keyword | `red`, `transparent`, `currentColor` |

<b class="text-on-semi-light">`currentColor`</b> — наследует текущий `color` элемента. Удобен для иконок, borders, SVG:

```css
.icon { border: 2px solid currentColor; } /* border совпадает с цветом текста */
```

## Частые вопросы на собесе

**Как работает каскад?**
Четыре уровня приоритета: 1) Origin & Importance (user-agent < user < author, `!important` инвертирует). 2) Cascade Layers. 3) Specificity. 4) Order of Appearance. Каждый следующий уровень проверяется, только если предыдущий не дал победителя.

**Как рассчитать специфичность?**
Формат `(A,B,C)`: A = количество `#id`, B = количество `.class`, `[attr]`, `:pseudo-class`, C = количество `element`, `::pseudo-element`. Сравнение слева направо. `:where()` = 0 специфичности. `:is()` берёт максимальную из аргументов.

**Когда margin collapse происходит, а когда нет?**
Схлопывается: вертикальные margins соседних блоков, parent + first/last child (без padding/border между ними). Не схлопывается: flex/grid items, BFC (flow-root, overflow), float, inline-block, absolute/fixed.

**`box-sizing: content-box` vs `border-box`?**
`content-box` (default): width = только content, padding и border сверху. `border-box`: width = content + padding + border. `border-box` предсказуемее — `* { box-sizing: border-box }` стандартный reset.

**Чем `inherit` отличается от `unset`?**
`inherit` — всегда наследует от родителя. `unset` — наследует, если свойство наследуемое (color, font), сбрасывает к initial, если нет (margin, display). `revert` откатывает к браузерным стилям.
