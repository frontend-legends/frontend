# Типографика и шрифты

Типографика — основа визуального качества интерфейса. На senior-уровне важно знать не только `font-size` и `line-height`, но и **стратегии загрузки шрифтов** (`font-display`, preload), **variable fonts**, **text overflow** паттерны и оптимизацию CLS при загрузке шрифтов.

---

## Подключение шрифтов

### `@font-face`

```css
@font-face {
  font-family: 'SpaceGrotesk';
  src: url('/fonts/SpaceGrotesk-Variable.woff2') format('woff2');
  font-weight: 300 700;        /* диапазон для variable font */
  font-style: normal;
  font-display: swap;
  unicode-range: U+0000-00FF, U+0400-04FF; /* Latin + Cyrillic */
}
```

| Свойство | Описание |
|---|---|
| `font-family` | Имя шрифта для использования в CSS |
| `src` | URL файла + формат. Можно несколько (fallback chain) |
| `font-weight` | Вес (или диапазон для variable fonts) |
| `font-display` | Стратегия отображения при загрузке |
| `unicode-range` | Подмножество символов (subset) — browser скачает файл только если нужные символы есть на странице |

### Self-hosting vs CDN

| | Self-hosting | Google Fonts / CDN |
|---|---|---|
| **Privacy** | Нет запросов к Google | Google отслеживает |
| **Performance** | Тот же домен, нет extra DNS | Extra DNS + connection |
| **Контроль** | Полный (subset, format, caching) | Ограниченный |
| **GDPR** | Соответствует | Проблемы в ЕС |

<b class="text-on-semi-light">Рекомендация:</b> Self-hosting через Bunny Fonts или вручную. `woff2` — единственный формат, который нужен (98%+ поддержка).

## `font-display`

Определяет, что показывать пользователю, пока шрифт загружается.

### FOIT vs FOUT

- **FOIT** (Flash of Invisible Text) — текст невидим, пока шрифт не загружен. Плохо для UX.
- **FOUT** (Flash of Unstyled Text) — текст виден в fallback-шрифте, потом перерисовывается. Лучше для UX.

| Значение | Block period | Swap period | Поведение |
|---|---|---|---|
| `auto` | Зависит от browser | Зависит от browser | Непредсказуемо |
| `block` | ~3s | Бесконечный | FOIT до 3с, потом swap. **Плохо** |
| `swap` | ~0ms | Бесконечный | Сразу fallback, swap при загрузке. **FOUT** |
| `fallback` | ~100ms | ~3s | Коротко invisible, swap если быстро загружен |
| `optional` | ~100ms | 0 | Если не загружен быстро — fallback навсегда |

<b class="text-on-semi-light">Рекомендации:</b>

- **`swap`** — для основного текста. Пользователь сразу видит контент.
- **`optional`** — для некритичных шрифтов (декоративные заголовки). Если сеть медленная — fallback без CLS.
- **`fallback`** — компромисс: short block, swap только если быстро загружен.

```css
/* Основной шрифт — swap */
@font-face { font-family: 'Inter'; font-display: swap; ... }

/* Декоративный — optional */
@font-face { font-family: 'Playfair'; font-display: optional; ... }
```

## Variable Fonts

Один файл содержит **все вариации** (weights, widths, styles) вместо отдельных файлов для каждого.

```css
@font-face {
  font-family: 'Inter';
  src: url('Inter-Variable.woff2') format('woff2');
  font-weight: 100 900;  /* весь диапазон */
  font-stretch: 75% 125%; /* ширина */
}

/* Использование любого веса */
h1 { font-weight: 750; }  /* не только 400/700, а любое значение */
p  { font-weight: 380; }
```

### Variation axes

| Ось | CSS-свойство | Описание |
|---|---|---|
| `wght` | `font-weight` | Вес (100–900) |
| `wdth` | `font-stretch` | Ширина (75%–125%) |
| `ital` | `font-style: italic` | Курсив (0/1) |
| `slnt` | `font-style: oblique Xdeg` | Наклон (градусы) |
| `opsz` | `font-optical-sizing` | Оптический размер |

```css
/* Низкоуровневый доступ к осям */
.text { font-variation-settings: 'wght' 450, 'wdth' 90; }
```

<b class="text-on-semi-light">На фронте:</b> Один variable font file (~100KB) заменяет 6–8 static файлов (~400KB+). Меньше запросов, меньше трафик. Плюс — анимация weight через transition для hover-эффектов.

## Текстовые свойства

### `line-height`

```css
/* Unitless — рекомендуется */
p { line-height: 1.5; }  /* = 1.5 × font-size текущего элемента */

/* С единицами — наследует ВЫЧИСЛЕННОЕ значение */
p { line-height: 24px; }  /* потомки получат ровно 24px, независимо от их font-size */
```

<b class="text-on-semi-light">Всегда используй unitless `line-height`</b> — наследуется как множитель, а не как фиксированное значение. `line-height: 1.5` адаптируется к font-size каждого потомка.

### `white-space`

| Значение | Пробелы | Переносы строк | Перенос текста |
|---|---|---|---|
| `normal` | Схлопываются | Игнорируются | Переносится |
| `nowrap` | Схлопываются | Игнорируются | **Не переносится** |
| `pre` | Сохраняются | Сохраняются | Не переносится |
| `pre-wrap` | Сохраняются | Сохраняются | Переносится |
| `pre-line` | Схлопываются | Сохраняются | Переносится |
| `break-spaces` | Сохраняются | Сохраняются | Переносится (даже по пробелам) |

### `word-break` vs `overflow-wrap`

```css
/* overflow-wrap: переносит ДЛИННЫЕ слова, если не влезают */
p { overflow-wrap: break-word; } /* бывший word-wrap */

/* word-break: разрывает слова по любому символу */
p { word-break: break-all; }  /* для CJK и URL-ов */
```

<span class="text-gray">`overflow-wrap: break-word` — безопасный вариант. Переносит только если слово не влезает. `word-break: break-all` — агрессивный: разрывает слова даже если не нужно. Для URL'ов и хешей используй `word-break: break-all` или `overflow-wrap: anywhere`.</span>

### `hyphens`

```css
p {
  hyphens: auto;  /* автоматические переносы слов */
  /* Требует lang="ru" на элементе или предке! */
}
```

## Текстовое переполнение

### Однострочное (ellipsis)

```css
.truncate {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
/* "Очень длинный текст, который не влезает..." */
```

### Многострочное (line-clamp)

```css
.clamp-3 {
  display: -webkit-box;
  -webkit-line-clamp: 3;     /* максимум 3 строки */
  -webkit-box-orient: vertical;
  overflow: hidden;
}
```

<span class="text-gray">Стандартный `line-clamp` (без `-webkit-`) пока в процессе стандартизации. `-webkit-line-clamp` работает во всех современных браузерах, включая Firefox.</span>

## `text-wrap: balance` / `pretty`

### `balance`

Выравнивает длины строк — заголовок больше не будет с одним словом на последней строке.

```css
h1, h2, h3 { text-wrap: balance; }
```

```text
Без balance:              С balance:
┌──────────────────┐     ┌──────────────────┐
│ Очень длинный    │     │ Очень длинный    │
│ заголовок для    │     │ заголовок        │
│ статьи           │     │ для статьи       │
└──────────────────┘     └──────────────────┘
```

### `pretty`

Избегает orphans (одно слово на последней строке) в параграфах.

```css
p { text-wrap: pretty; }
```

<span class="text-gray">`balance` — для коротких текстов (заголовки, captions). На длинных текстах тормозит (browser пересчитывает layout). `pretty` — для параграфов, работает только на последней строке.</span>

## Оптимизация загрузки шрифтов

### 1. Preload критичных шрифтов

```html
<link rel="preload" href="/fonts/Inter-Variable.woff2" as="font" type="font/woff2" crossorigin>
```

<span class="text-gray">`crossorigin` обязателен для шрифтов — даже с того же домена. Без него browser загрузит файл дважды.</span>

### 2. Subset через `unicode-range`

```css
/* Browser скачает файл только если кириллица есть на странице */
@font-face {
  font-family: 'Inter';
  src: url('inter-cyrillic.woff2') format('woff2');
  unicode-range: U+0400-04FF; /* Cyrillic */
}

@font-face {
  font-family: 'Inter';
  src: url('inter-latin.woff2') format('woff2');
  unicode-range: U+0000-00FF; /* Latin */
}
```

### 3. `size-adjust` для CLS prevention

```css
/* Метрики fallback-шрифта подогнаны под целевой */
@font-face {
  font-family: 'Inter Fallback';
  src: local('Arial');
  size-adjust: 107%;       /* компенсация разницы в размерах */
  ascent-override: 90%;
  descent-override: 22%;
  line-gap-override: 0%;
}

body { font-family: 'Inter', 'Inter Fallback', sans-serif; }
```

<b class="text-on-semi-light">Зачем:</b> Когда web font загружается и заменяет fallback, текст «прыгает» из-за разницы в метриках → CLS (Cumulative Layout Shift). `size-adjust` и override-свойства минимизируют разницу.

## Частые вопросы на собесе

**Какие значения `font-display` бывают и когда что использовать?**
`swap` — сразу fallback, swap при загрузке (основной текст). `optional` — если не загружен быстро, fallback навсегда (некритичные шрифты). `fallback` — компромисс. `block` — FOIT (избегать).

**Чем FOIT отличается от FOUT?**
FOIT — текст невидим до загрузки шрифта (плохо). FOUT — текст виден в fallback, перерисовывается (лучше). `font-display: swap` = FOUT. `font-display: block` = FOIT.

**Зачем variable fonts?**
Один файл (100KB) вместо 6–8 (400KB+). Любые промежуточные значения weight/width. Меньше HTTP-запросов. Возможность анимировать font-weight.

**Как обрезать текст многострочным ellipsis?**
`display: -webkit-box; -webkit-line-clamp: N; -webkit-box-orient: vertical; overflow: hidden;` Работает во всех browser'ах.

**`white-space: nowrap` vs `pre` vs `pre-wrap`?**
`nowrap` — текст в одну строку (для truncation). `pre` — сохраняет пробелы и переносы, не переносит (для кода). `pre-wrap` — сохраняет, но переносит при необходимости.

**Как предотвратить CLS при загрузке шрифтов?**
1) `font-display: swap`/`optional`. 2) Preload шрифта. 3) `size-adjust` на fallback для совпадения метрик. 4) `unicode-range` для subset loading.
