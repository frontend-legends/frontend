# Раскладка и поток

Layout — самая объёмная тема CSS. Здесь собраны все ключевые концепции: normal flow, BFC, позиционирование, stacking context, Flexbox и Grid. Каждая из них — потенциальный вопрос на senior-собесе, особенно BFC, stacking context и нюансы `flex` shorthand.

---

## Normal Flow

По умолчанию элементы располагаются в **нормальном потоке** (normal flow):

- **Block-level** элементы (`div`, `p`, `section`) → занимают всю ширину родителя, укладываются **вертикально** друг за другом
- **Inline-level** элементы (`span`, `a`, `strong`) → занимают ширину контента, располагаются **горизонтально** в строке (line box), переносятся при нехватке места

```text
Block flow:         Inline flow:
┌──────────────┐    ┌─────────────────────────┐
│   block 1    │    │ [inline1] [inline2] [in │
├──────────────┤    │ line3] [inline4]         │
│   block 2    │    └─────────────────────────┘
├──────────────┤
│   block 3    │
└──────────────┘
```

## Display

CSS `display` определяет **два уровня** поведения: **outer** (как элемент ведёт себя в потоке родителя) и **inner** (как его потомки располагаются внутри).

| Значение | Outer | Inner | Описание |
|---|---|---|---|
| `block` | block | flow | Блок на всю ширину, вертикальный стек потомков |
| `inline` | inline | flow | В строке, ширина по контенту |
| `inline-block` | inline | flow-root | В строке, но принимает width/height/margin/padding |
| `flex` | block | **flex** | Блок, потомки — flex items |
| `inline-flex` | inline | flex | В строке, потомки — flex items |
| `grid` | block | **grid** | Блок, потомки — grid items |
| `inline-grid` | inline | grid | В строке, потомки — grid items |
| `contents` | — | — | Бокс удаляется, потомки «всплывают» к родителю |
| `none` | — | — | Элемент полностью удалён из потока и render tree |

### `none` vs `visibility: hidden` vs `opacity: 0`

| Свойство | В layout | Видим | Кликабелен | a11y (скринридер) | Анимируем |
|---|---|---|---|---|---|
| `display: none` | Нет | Нет | Нет | Скрыт | Нет |
| `visibility: hidden` | **Да** (занимает место) | Нет | Нет | Скрыт | Да |
| `opacity: 0` | **Да** | Нет | **Да** | **Виден** | Да |

<span class="text-gray">`display: contents` убирает бокс элемента, но сохраняет потомков в потоке. Полезно для семантических обёрток, которые не должны влиять на layout (flex/grid item wrapper). Осторожно с `<button>` и `<a>` — теряется accessibility.</span>

## Block Formatting Context (BFC)

<b class="text-on-semi-light">Ключевая идея:</b> BFC — это изолированный контекст форматирования. Элементы внутри BFC не влияют на элементы снаружи, и наоборот.

<b class="text-on-semi-light">Как запомнить:</b> Коробка с крышкой — всё, что внутри, не вываливается наружу: float'ы не торчат, margin'ы не схлопываются через границу.

<b class="text-on-semi-light">На фронте:</b> Без понимания BFC невозможно объяснить, почему float-элемент «выпадает» из родителя, или почему overflow: hidden «чинит» верстку.

### Что создаёт BFC

- `display: flow-root` — **правильный** способ создать BFC
- `overflow` с значением != `visible` и != `clip` (hidden, auto, scroll)
- `display: flex` / `display: grid` — flex/grid контейнер
- `float: left` / `float: right`
- `position: absolute` / `position: fixed`
- `display: inline-block`
- `contain: layout` / `contain: paint`
- Root element (`<html>`)

### Зачем нужен BFC

1. **Contain floats** — float-элемент без BFC «выпадает» из родителя (родитель схлопывается)
2. **Prevent margin collapse** — margin'ы не схлопываются через границу BFC
3. **Isolation** — независимый layout-контекст

**Плохо — хак через overflow:**

```css
/* overflow: hidden создаёт BFC, но обрезает контент */
.container { overflow: hidden; }
```

**Хорошо — display: flow-root:**

```css
/* Явное создание BFC без побочных эффектов */
.container { display: flow-root; }
```

## Позиционирование

| Значение | Поведение | Containing block | Занимает место в потоке |
|---|---|---|---|
| `static` | По умолчанию, normal flow | — | Да |
| `relative` | Смещается относительно своей позиции | Сам элемент | Да (исходное место зарезервировано) |
| `absolute` | Вырывается из потока, позиционируется относительно ближайшего positioned ancestor | Ближайший ancestor с position != static | Нет |
| `fixed` | Вырывается из потока, позиционируется относительно viewport | Viewport (или ancestor с transform!) | Нет |
| `sticky` | Как relative, пока не достигает threshold, потом как fixed | Scroll container | Да |

### `position: sticky` — deep dive

```css
.header {
  position: sticky;
  top: 0;       /* threshold: «прилипает» при top = 0 */
  z-index: 10;  /* чтобы не уходил под контент */
}
```

<b class="text-on-semi-light">Gotchas, из-за которых sticky не работает:</b>

1. **`overflow: hidden/auto/scroll` на предке** — sticky элемент привязывается к scroll container. Если предок имеет overflow — он становится scroll container, и sticky работает внутри него, а не relative to viewport.
2. **Родитель без достаточной высоты** — sticky не может «уехать» дальше boundaries родителя. Если родитель = высоте sticky элемента, эффекта не будет.
3. **`overflow: hidden` на `<body>` или `<html>`** — частая ошибка при фиксированных layout'ах.

### Containing block

`top`, `right`, `bottom`, `left` позиционируют элемент относительно его **containing block**:

- `position: absolute` → ближайший ancestor с `position` != `static`
- `position: fixed` → viewport, **но**: если на предке есть `transform`, `filter`, `will-change`, `contain: paint` — он становится containing block вместо viewport!

<span class="text-gray">Это частая причина бага: fixed-элемент «прилипает» к родителю с transform вместо viewport.</span>

## Stacking Context

<b class="text-on-semi-light">Ключевая идея:</b> Stacking context — иерархический контекст наложения. `z-index` работает **только внутри своего контекста**. Элемент не может перекрыть элемент из другого stacking context, если parent context ниже.

<b class="text-on-semi-light">Как запомнить:</b> Этаж в здании. `z-index` — номер квартиры на этаже. Квартира на 1-м этаже не может быть выше квартиры на 2-м этаже, какой бы номер у неё ни был.

<b class="text-on-semi-light">На фронте:</b> Модалки, dropdown'ы, tooltips — «почему z-index: 9999 не работает?» Потому что элемент в stacking context с более низким приоритетом.

### Что создаёт stacking context

- `position: relative/absolute/fixed/sticky` + `z-index` != `auto`
- `opacity` < 1
- `transform` (любое значение != `none`)
- `filter` (любое значение != `none`)
- `will-change: transform` / `opacity` / etc.
- `isolation: isolate`
- `mix-blend-mode` != `normal`
- `contain: layout` / `paint`
- Flex/grid child с `z-index` != `auto`

### Пример: z-index не работает

```text
<div class="sidebar" style="position: relative; z-index: 1">
  <div class="dropdown" style="position: absolute; z-index: 9999">
    <!-- z-index: 9999, но внутри контекста sidebar (z-index: 1) -->
  </div>
</div>

<div class="modal" style="position: fixed; z-index: 2">
  <!-- modal (z-index: 2) ВЫШЕ dropdown (9999), потому что
       sidebar (z-index: 1) < modal (z-index: 2) на уровне root -->
</div>
```

### `isolation: isolate`

Создаёт новый stacking context **без побочных эффектов** (в отличие от `opacity: 0.999`, `transform: translateZ(0)`).

```css
/* Изоляция компонента — его z-index не влияет на соседей */
.component { isolation: isolate; }
```

**Плохо — война z-index:**

```css
.dropdown { z-index: 999; }
.modal { z-index: 9999; }
.tooltip { z-index: 99999; }
/* Бесконечная эскалация */
```

**Хорошо — контролируемые контексты:**

```css
.layout { isolation: isolate; }
.dropdown { z-index: 10; }
.modal { z-index: 20; }   /* Внутри своего stacking context */
```

## Flexbox

Одномерная система раскладки: элементы располагаются вдоль **главной оси** (main axis).

### Оси

```text
flex-direction: row (default)

main axis →
┌──────┬──────┬──────┐
│ item │ item │ item │  ← cross axis ↓
└──────┴──────┴──────┘

flex-direction: column

cross axis →
┌──────┐
│ item │  ← main axis ↓
├──────┤
│ item │
├──────┤
│ item │
└──────┘
```

### Container properties

| Свойство | Значения | Описание |
|---|---|---|
| `flex-direction` | `row`, `row-reverse`, `column`, `column-reverse` | Направление main axis |
| `flex-wrap` | `nowrap`, `wrap`, `wrap-reverse` | Перенос на новую строку |
| `justify-content` | `flex-start`, `center`, `flex-end`, `space-between`, `space-around`, `space-evenly` | Выравнивание по main axis |
| `align-items` | `stretch`, `flex-start`, `center`, `flex-end`, `baseline` | Выравнивание по cross axis |
| `align-content` | Как `justify-content` | Выравнивание строк при `wrap` |
| `gap` | `<length>` | Отступ между items |

### Item properties

| Свойство | Default | Описание |
|---|---|---|
| `flex-grow` | `0` | Как сильно растёт (забирает свободное пространство) |
| `flex-shrink` | `1` | Как сильно сжимается (отдаёт пространство) |
| `flex-basis` | `auto` | Базовый размер по main axis (до grow/shrink) |
| `align-self` | `auto` | Переопределяет `align-items` для одного item |
| `order` | `0` | Визуальный порядок (не влияет на tab order — a11y!) |

### `flex` shorthand — senior must-know

| Значение | Эквивалент | Поведение |
|---|---|---|
| `flex: 1` | `flex: 1 1 0%` | Растёт, сжимается, **все items равного размера** |
| `flex: auto` | `flex: 1 1 auto` | Растёт, сжимается, **размер зависит от контента** |
| `flex: none` | `flex: 0 0 auto` | Не растёт, не сжимается. Фиксированный размер |
| `flex: 0 1 auto` | (default) | Не растёт, сжимается при нехватке места |

<b class="text-on-semi-light">Ключевой gotcha: `min-width: auto`</b>

По умолчанию flex items имеют `min-width: auto` — элемент не может быть уже своего контента. Из-за этого длинный текст не обрезается, а растягивает контейнер.

```css
/* Решение: сбросить min-width */
.flex-item { min-width: 0; }
/* Или overflow: hidden (создаёт BFC, устанавливает min-width: 0 фактически) */
.flex-item { overflow: hidden; }
```

## Grid

Двумерная система раскладки: элементы располагаются по **строкам и столбцам** одновременно.

### Container properties

```css
.grid {
  display: grid;
  grid-template-columns: 200px 1fr 1fr;     /* 3 столбца */
  grid-template-rows: auto 1fr auto;         /* 3 строки */
  gap: 16px;                                 /* отступы между ячейками */
}
```

| Свойство | Описание |
|---|---|
| `grid-template-columns` / `rows` | Определяет tracks (дорожки) |
| `grid-template-areas` | Именованные области (декларативный layout) |
| `gap` / `row-gap` / `column-gap` | Отступы между tracks |
| `justify-items` | Выравнивание items по inline axis (горизонтальная) |
| `align-items` | Выравнивание items по block axis (вертикальная) |
| `grid-auto-flow` | `row`, `column`, `dense` — как авто-размещать items |
| `grid-auto-rows` / `columns` | Размер для неявных (автосозданных) tracks |

### Ключевые функции

```css
/* repeat() — повторение */
grid-template-columns: repeat(3, 1fr);           /* 3 равных столбца */

/* minmax() — мин и макс размер track */
grid-template-columns: repeat(3, minmax(200px, 1fr));

/* auto-fill vs auto-fit */
grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
```

### `auto-fill` vs `auto-fit`

| | Что делает | Когда items не заполняют ряд |
|---|---|---|
| `auto-fill` | Создаёт максимум tracks | Пустые tracks **остаются** (занимают место) |
| `auto-fit` | Создаёт максимум tracks | Пустые tracks **схлопываются** (items растягиваются) |

<span class="text-gray">`auto-fit` — обычно то, что нужно. Items растягиваются на всю доступную ширину. `auto-fill` полезен, когда нужно зарезервировать место для будущих items.</span>

### Named areas

```css
.layout {
  display: grid;
  grid-template-areas:
    "header  header  header"
    "sidebar content content"
    "footer  footer  footer";
  grid-template-columns: 250px 1fr 1fr;
  grid-template-rows: auto 1fr auto;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.content { grid-area: content; }
.footer  { grid-area: footer; }
```

### Subgrid

Potомок наследует grid-tracks родителя — выравнивание через уровни вложенности.

```css
.parent {
  display: grid;
  grid-template-columns: 1fr 2fr 1fr;
}
.child {
  grid-column: 1 / -1;  /* span all columns */
  display: grid;
  grid-template-columns: subgrid;  /* наследует 3 колонки родителя */
}
```

## Flexbox vs Grid

| Критерий | Flexbox | Grid |
|---|---|---|
| **Размерность** | 1D (строка ИЛИ столбец) | 2D (строки И столбцы) |
| **Подход** | Content-driven (размер определяется контентом) | Layout-driven (размер определяется контейнером) |
| **Кто управляет** | Items влияют на layout | Container диктует layout |
| **Применение** | Nav bars, toolbars, центрирование, card content | Page layouts, card grids, dashboards, формы |
| **Выравнивание** | По одной оси | По обеим осям |
| **Wrap** | Перенос без контроля высоты строк | Полный контроль строк и столбцов |

<b class="text-on-semi-light">Правило:</b> Если layout одномерный (элементы в ряд или в столбец) — Flexbox. Если двумерный (нужен контроль по строкам И столбцам одновременно) — Grid. Часто используются вместе: Grid для page layout, Flexbox для content внутри grid items.

## Частые вопросы на собесе

**Что такое BFC и зачем?**
Block Formatting Context — изолированный контекст. Элементы внутри BFC не влияют на внешние. Нужен для: contain floats, prevent margin collapse, isolation. Создаётся через `display: flow-root` (лучший способ), `overflow` != visible, flex/grid containers.

**Почему z-index не работает?**
`z-index` работает только внутри своего stacking context. Если элемент A в stacking context с z-index: 1, а элемент B в stacking context с z-index: 2 — ничто внутри A не перекроет B, даже с z-index: 999999.

**Что значит `flex: 1`?**
`flex: 1 1 0%` — grow: 1 (растёт), shrink: 1 (сжимается), basis: 0% (стартовый размер 0). Все items с `flex: 1` будут **одинакового размера**, деля пространство поровну.

**`auto-fill` vs `auto-fit`?**
Оба создают максимальное количество tracks. Разница при недостатке items: `auto-fill` оставляет пустые tracks, `auto-fit` схлопывает их — items растягиваются на всю ширину.

**Почему `position: sticky` не работает?**
Три частые причины: 1) `overflow: hidden/auto/scroll` на предке (создаёт свой scroll container). 2) Родитель не имеет достаточной высоты. 3) `overflow` на body/html.

**`display: none` vs `visibility: hidden` vs `opacity: 0`?**
`none` — удаляет из layout и render tree. `visibility: hidden` — скрывает, но место занято. `opacity: 0` — прозрачный, но кликабельный и видимый для скринридеров.
