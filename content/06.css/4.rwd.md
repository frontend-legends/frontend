# Адаптивный дизайн (RWD)

Responsive Web Design — подход, при котором один сайт адаптируется к **любому viewport**: от мобильного до десктопа. На senior-уровне важны не только media queries, но и **container queries** (стили зависят от контейнера, а не viewport), **fluid typography** (размеры текста масштабируются плавно) и gotchas **viewport units** на мобильных.

---

## Mobile-first vs Desktop-first

| Подход | Media queries | Философия |
|---|---|---|
| **Mobile-first** | `@media (min-width: ...)` | Базовые стили для мобильных, расширяем для больших экранов |
| **Desktop-first** | `@media (max-width: ...)` | Базовые стили для десктопа, упрощаем для маленьких экранов |

<b class="text-on-semi-light">Почему mobile-first — стандарт:</b>

- **Progressive enhancement** — начинаем с минимального, добавляем сложность
- **Производительность** — мобильные получают только нужный CSS, без лишних переопределений
- **Приоритизация** — заставляет определить, что действительно важно

```css
/* Mobile-first: базовые стили → добавляем для больших экранов */
.grid { display: flex; flex-direction: column; }

@media (min-width: 768px) {
  .grid { flex-direction: row; }
}

@media (min-width: 1024px) {
  .grid { max-width: 1200px; margin: 0 auto; }
}
```

<span class="text-gray">Breakpoints не должны привязываться к конкретным устройствам (iPhone 15 = 393px). Используй **content-driven breakpoints** — ставь breakpoint там, где layout ломается.</span>

## Media Queries

### Синтаксис

```css
/* Классический */
@media (min-width: 768px) { ... }
@media (max-width: 1023px) { ... }

/* Range syntax (современный) — читабельнее */
@media (width >= 768px) { ... }
@media (768px <= width < 1024px) { ... }

/* Комбинация */
@media (width >= 768px) and (orientation: landscape) { ... }
```

### Полезные media features

| Feature | Что проверяет | Пример |
|---|---|---|
| `prefers-color-scheme` | Тёмная/светлая тема OS | `prefers-color-scheme: dark` |
| `prefers-reduced-motion` | Пользователь просит убрать анимации | `prefers-reduced-motion: reduce` |
| `prefers-contrast` | Повышенный контраст | `prefers-contrast: more` |
| `hover` | Есть ли hover (мышь) | `hover: hover` (мышь), `hover: none` (тач) |
| `pointer` | Тип указателя | `pointer: fine` (мышь), `pointer: coarse` (палец) |
| `orientation` | Ориентация viewport | `orientation: portrait` / `landscape` |

```css
/* Тёмная тема */
@media (prefers-color-scheme: dark) {
  :root { --bg: #1a1a2e; --text: #eee; }
}

/* Отключение анимаций для a11y */
@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; }
}

/* Hover-эффекты только на устройствах с мышью */
@media (hover: hover) {
  .card:hover { transform: translateY(-4px); }
}
```

<b class="text-on-semi-light">На фронте:</b> `prefers-reduced-motion` — не просто «приятный бонус», а **a11y-обязательство**. У людей с вестибулярными нарушениями анимации могут вызвать тошноту и головокружение.

## Container Queries

<b class="text-on-semi-light">Ключевая идея:</b> Container queries позволяют стилизовать элемент на основе **размера его контейнера**, а не viewport. Компонент адаптируется к тому пространству, в которое помещён.

<b class="text-on-semi-light">Как запомнить:</b> Media query = размер окна. Container query = размер комнаты, в которой стоит мебель. Одна и та же мебель (компонент) выглядит по-разному в разных комнатах.

<b class="text-on-semi-light">На фронте:</b> Переиспользуемые компоненты: карточка в sidebar — компактная, карточка на полную ширину — развёрнутая. Без JS, без дублирования стилей.

### Синтаксис

```css
/* 1. Объявляем контейнер */
.sidebar {
  container-type: inline-size;  /* отслеживаем ширину */
  container-name: sidebar;      /* имя (опционально) */
}

/* 2. Стили зависят от контейнера */
@container (min-width: 400px) {
  .card { display: grid; grid-template-columns: 120px 1fr; }
}

/* С именем контейнера */
@container sidebar (min-width: 300px) {
  .card { flex-direction: row; }
}
```

### Container query units

| Unit | Относительно | Аналог viewport |
|---|---|---|
| `cqw` | 1% ширины контейнера | `vw` |
| `cqh` | 1% высоты контейнера | `vh` |
| `cqi` | 1% inline size контейнера | `vi` |
| `cqb` | 1% block size контейнера | `vb` |

```css
.card-title { font-size: clamp(1rem, 4cqi, 2rem); }
```

### Media queries vs Container queries

| Критерий | Media Queries | Container Queries |
|---|---|---|
| **Реагирует на** | Размер viewport | Размер контейнера |
| **Уровень** | Page-level | Component-level |
| **Переиспользование** | Компонент «знает» о viewport | Компонент адаптируется к месту |
| **Применение** | Page layout, global styles | Компоненты, виджеты |

## Viewport Units

### Классические

| Unit | Описание |
|---|---|
| `vw` | 1% ширины viewport |
| `vh` | 1% высоты viewport |
| `vmin` | Меньшее из `vw` и `vh` |
| `vmax` | Большее из `vw` и `vh` |

### Проблема `100vh` на мобильных

На мобильных устройствах `100vh` **не равен видимой высоте** экрана. Адресная строка браузера (и виртуальная клавиатура) меняют видимую область, но `vh` остаётся привязан к **максимальной** высоте viewport.

```text
┌─────────────────┐
│  Address bar     │ ← скрывается при скролле
├─────────────────┤
│                 │
│  100vh = вся    │ ← включает область address bar
│  высота         │
│                 │
└─────────────────┘
```

### Современные единицы

| Unit | Описание | Когда использовать |
|---|---|---|
| `dvh` | Dynamic — учитывает текущее состояние (address bar видна/скрыта) | Полноэкранные секции. **Рекомендация по умолчанию** |
| `svh` | Small — минимальная видимая область (address bar видна) | Когда контент должен влезть при видимой address bar |
| `lvh` | Large — максимальная видимая область (address bar скрыта) | Фоновые элементы, совпадает со старым `vh` |

```css
/* Полноэкранная секция, работает на мобильных */
.hero { min-height: 100dvh; }

/* Fallback для старых браузеров */
.hero { min-height: 100vh; min-height: 100dvh; }
```

## Fluid Typography

Размер текста, который **плавно масштабируется** между минимумом и максимумом — без media queries.

```css
/* clamp(min, preferred, max) */
h1 { font-size: clamp(1.5rem, 1rem + 3vw, 3rem); }
p  { font-size: clamp(1rem, 0.875rem + 0.5vw, 1.25rem); }

/* Fluid spacing */
section { padding: clamp(1rem, 3vw, 4rem); }
```

<b class="text-on-semi-light">Как работает:</b> При маленьком viewport `1rem + 3vw` < `1.5rem` → используется `1.5rem` (min). При большом viewport `1rem + 3vw` > `3rem` → используется `3rem` (max). Между ними — плавный рост.

<span class="text-gray">Одна строка `clamp()` заменяет 2–3 media query для font-size. Подробнее про единицы → [Каскад и стилизация](/content/css/styling).</span>

## Адаптивные паттерны

### Sidebar layout (Grid)

```css
.layout {
  display: grid;
  grid-template-columns: minmax(200px, 300px) 1fr;
  gap: 24px;
}

@media (width < 768px) {
  .layout { grid-template-columns: 1fr; }
}
```

### Card grid (auto-fit + minmax)

```css
.cards {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(280px, 1fr));
  gap: 16px;
}
/* Автоматически: 1 колонка на мобильных, 2–3 на десктопе */
```

### Stack-to-horizontal (Flexbox)

```css
.toolbar {
  display: flex;
  flex-wrap: wrap;
  gap: 8px;
}
/* На узком экране items переносятся на новую строку */
```

## Частые вопросы на собесе

**Container queries vs media queries?**
Media queries реагируют на viewport, container queries — на размер контейнера. Container queries дают component-level адаптивность: один компонент адаптируется к любому месту (sidebar, main, modal) без JS.

**Почему `100vh` не работает на мобильных?**
`vh` привязан к максимальной высоте viewport (с убранной address bar). Когда address bar видна, `100vh` больше видимой области. Решение: `100dvh` (dynamic viewport height) — учитывает текущее состояние.

**Как работает `clamp()` для typography?**
`clamp(min, preferred, max)` — font-size плавно растёт с viewport: ниже min — используется min, выше max — max, между ними — fluid значение. Заменяет media queries для font-size.

**Зачем `prefers-reduced-motion`?**
A11y-обязательство. Люди с вестибулярными нарушениями (vertigo, motion sickness) испытывают дискомфорт от анимаций. `@media (prefers-reduced-motion: reduce)` отключает или минимизирует движение.

**Mobile-first — почему?**
Progressive enhancement: базовые стили для мобильных, добавляем сложность через `min-width`. Мобильные получают минимум CSS. Заставляет приоритизировать контент. `min-width` media queries чище, чем `max-width` overrides.
