# Анимации и переходы

Анимации в CSS — мощный инструмент, но на senior-уровне вопросы не про `@keyframes` синтаксис, а про **производительность**: почему анимировать можно только `transform` и `opacity`, что делает `will-change`, и как уважать `prefers-reduced-motion`.

---

## Transitions

Transition — плавный переход между двумя состояниями свойства при его изменении (hover, focus, class toggle).

```css
.btn {
  background: var(--primary);
  transform: scale(1);
  transition: background 0.3s ease, transform 0.2s ease;
}

.btn:hover {
  background: var(--primary-dark);
  transform: scale(1.05);
}
```

### Свойства

| Свойство | Описание | Пример |
|---|---|---|
| `transition-property` | Какое свойство анимировать | `background`, `transform`, `all` |
| `transition-duration` | Длительность | `0.3s`, `300ms` |
| `transition-timing-function` | Кривая скорости | `ease`, `linear`, `cubic-bezier()` |
| `transition-delay` | Задержка перед началом | `0.1s` |

**Shorthand:** `transition: property duration timing-function delay`

```css
/* Множественные transitions */
transition: transform 0.3s ease, opacity 0.3s ease 0.1s;
```

### Что можно анимировать

Не все CSS-свойства анимируемы. Animatable properties — те, для которых browser может вычислить промежуточные значения:

- **Анимируемые:** `transform`, `opacity`, `color`, `background-color`, `width`, `height`, `margin`, `padding`, `border-*`, `box-shadow`, `font-size`, `top`/`left`/`right`/`bottom`
- **Не анимируемые:** `display`, `font-family`, `position`, `float`, `grid-template-*`

<span class="text-gray">`display` нельзя анимировать напрямую, но с новым `transition-behavior: allow-discrete` и `@starting-style` можно создать transitions для `display: none ↔ block`.</span>

## `@keyframes` и `animation`

Keyframes — полный контроль над многошаговой анимацией.

```css
@keyframes fadeSlideIn {
  0% {
    opacity: 0;
    transform: translateY(20px);
  }
  100% {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: fadeSlideIn 0.5s ease forwards;
}
```

### Свойства animation

| Свойство | Значения | Описание |
|---|---|---|
| `animation-name` | Имя `@keyframes` | `fadeSlideIn` |
| `animation-duration` | Длительность | `0.5s` |
| `animation-timing-function` | Кривая скорости | `ease`, `linear` |
| `animation-delay` | Задержка | `0.2s` |
| `animation-iteration-count` | Количество повторов | `1`, `3`, `infinite` |
| `animation-direction` | Направление | `normal`, `reverse`, `alternate` |
| `animation-fill-mode` | Состояние до/после | `none`, `forwards`, `backwards`, `both` |
| `animation-play-state` | Пауза | `running`, `paused` |

**Shorthand:** `animation: name duration timing delay count direction fill-mode`

```css
/* Пульсация — бесконечная, туда-обратно */
@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.05); }
}

.notification { animation: pulse 2s ease-in-out infinite; }
```

### `animation-fill-mode`

| Значение | До анимации | После анимации |
|---|---|---|
| `none` | Исходные стили | Исходные стили |
| `forwards` | Исходные стили | **Стили последнего keyframe** |
| `backwards` | **Стили первого keyframe** | Исходные стили |
| `both` | Стили первого keyframe | Стили последнего keyframe |

<span class="text-gray">`forwards` — самый частый: элемент остаётся в конечном состоянии после анимации (например, fade-in → остаётся видимым).</span>

## Timing Functions

| Функция | Поведение | Применение |
|---|---|---|
| `ease` | Быстрый старт, медленный конец (default) | Универсальная |
| `ease-in` | Медленный старт | Элемент уходит (exit) |
| `ease-out` | Медленный конец | Элемент появляется (enter) |
| `ease-in-out` | Медленный старт и конец | Циклические анимации |
| `linear` | Постоянная скорость | Progress bars, вращение |
| `cubic-bezier(x1,y1,x2,y2)` | Кастомная кривая | Для natural motion |
| `steps(n, start\|end)` | Дискретные шаги | Спрайтовая анимация, печатающий текст |

```css
/* Custom cubic-bezier для "упругого" эффекта */
.card { transition: transform 0.4s cubic-bezier(0.34, 1.56, 0.64, 1); }

/* Steps для печатающего текста */
.typewriter {
  width: 0;
  overflow: hidden;
  animation: typing 3s steps(30) forwards;
}
@keyframes typing { to { width: 100%; } }
```

<b class="text-on-semi-light">Правило UX:</b> `ease-out` для входа элемента (появление), `ease-in` для выхода (исчезновение). Элемент «прилетает быстро и тормозит», «разгоняется и улетает».

## Производительность анимаций

Это **самая важная** тема по анимациям на senior-собесе.

### Три уровня рендеринга

```text
1. Layout (reflow)    — пересчёт позиций и размеров    → ДОРОГО
2. Paint (repaint)    — перерисовка пикселей            → СРЕДНЕ
3. Composite          — перекомпозиция GPU-слоёв        → ДЁШЕВО
```

### Какие свойства что вызывают

| Уровень | Свойства | Стоимость |
|---|---|---|
| **Layout** (reflow) | `width`, `height`, `margin`, `padding`, `top`, `left`, `font-size`, `display` | Высокая |
| **Paint** (repaint) | `color`, `background`, `box-shadow`, `border`, `visibility` | Средняя |
| **Composite** only | `transform`, `opacity` | **Низкая (GPU)** |

<b class="text-on-semi-light">Правило:</b> Анимируй **только** `transform` и `opacity`. Они обрабатываются на GPU, минуя layout и paint. Всё остальное вызывает reflow или repaint на каждый кадр.

**Плохо — анимация `left` (reflow каждый кадр):**

```css
.box {
  position: absolute;
  left: 0;
  transition: left 0.3s ease;
}
.box:hover { left: 100px; }
```

**Хорошо — анимация `transform` (composite only):**

```css
.box {
  transform: translateX(0);
  transition: transform 0.3s ease;
}
.box:hover { transform: translateX(100px); }
```

### `will-change`

Подсказка browser'у: «этот элемент скоро будет анимироваться». Browser заранее выносит элемент на отдельный GPU-слой.

```css
.card:hover { will-change: transform; }
.card.animating { transform: scale(1.1); }
```

<b class="text-on-semi-light">Осторожно с `will-change`:</b>

- Создаёт **новый stacking context** и compositing layer
- Каждый layer потребляет **GPU-память**
- **Не** ставь `will-change` на все элементы — только на те, которые будут анимироваться
- Добавляй перед анимацией, убирай после (через JS или при hover)
- `will-change: auto` — сброс

```css
/* НЕ делай так */
* { will-change: transform; } /* каждый элемент на отдельном GPU-слое */

/* Делай так */
.card { transition: transform 0.3s; }
.card:hover { will-change: transform; transform: translateY(-4px); }
```

<span class="text-gray">Подробнее про reflow vs repaint и CRP → [Структура и рендеринг](/content/html/structure).</span>

## `prefers-reduced-motion`

A11y-обязательство: люди с вестибулярными нарушениями (vertigo, motion sickness) могут включить настройку «уменьшить движение» в OS. CSS должен это уважать.

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

<span class="text-gray">Не полностью убирай анимации — `0.01ms` вместо `0s`, чтобы `animationend`/`transitionend` события всё равно сработали (JavaScript-логика, зависящая от этих событий, не сломается).</span>

## Scroll-driven Animations

Анимации, привязанные к **позиции скролла**, а не ко времени. Без JavaScript.

```css
/* Прогресс-бар, привязанный к скроллу страницы */
.progress-bar {
  position: fixed;
  top: 0;
  left: 0;
  height: 3px;
  background: var(--primary);
  transform-origin: left;
  animation: grow-bar linear;
  animation-timeline: scroll();  /* привязка к скроллу */
}

@keyframes grow-bar {
  from { transform: scaleX(0); }
  to { transform: scaleX(1); }
}
```

```css
/* Fade-in при появлении элемента в viewport */
.card {
  animation: fade-in linear both;
  animation-timeline: view();      /* привязка к видимости элемента */
  animation-range: entry 0% entry 100%; /* от начала появления до полного входа */
}

@keyframes fade-in {
  from { opacity: 0; transform: translateY(50px); }
  to { opacity: 1; transform: translateY(0); }
}
```

<b class="text-on-semi-light">На фронте:</b> Заменяет Intersection Observer + JS для scroll-triggered анимаций. Чистый CSS, работает на composite thread, не блокирует main thread.

## View Transitions API

Анимация переходов между страницами (MPA) или состояниями (SPA) — нативный morphing.

```css
/* Определяем элементы для перехода */
.card { view-transition-name: card-hero; }

/* Стилизация перехода */
::view-transition-old(card-hero) { animation: fade-out 0.3s ease; }
::view-transition-new(card-hero) { animation: fade-in 0.3s ease; }
```

```js
// SPA: запуск view transition
document.startViewTransition(() => {
  updateDOM(); // обновляем содержимое
});
```

<span class="text-gray">View Transitions — emerging API. В Nuxt/Next.js уже есть экспериментальная поддержка. Для SPA — стабильно в Chrome. Для MPA (cross-document) — пока ограниченно.</span>

## Частые вопросы на собесе

**Почему анимировать нужно только `transform` и `opacity`?**
`transform` и `opacity` обрабатываются на GPU-уровне (composite), минуя layout и paint. Анимация `width`, `top`, `margin` вызывает reflow на каждый кадр — 60 reflow/сек = тормоза.

**Что делает `will-change` и когда использовать?**
Подсказка browser'у вынести элемент на GPU-слой заранее. Ускоряет старт анимации. Но: создаёт stacking context, потребляет GPU-память. Не ставить на всё — только перед анимацией, убирать после.

**Transition vs animation — когда что?**
Transition — для перехода между двумя состояниями (hover, focus, class toggle). Animation — для многошаговых, циклических или автоматических анимаций (loader, pulse, entry animation).

**Что такое `cubic-bezier()`?**
Кастомная timing function: 4 числа определяют кривую скорости. `cubic-bezier(0, 0, 1, 1)` = linear. `cubic-bezier(0.34, 1.56, 0.64, 1)` = «упругий» эффект. Визуализатор: cubic-bezier.com.

**Как уважать `prefers-reduced-motion`?**
`@media (prefers-reduced-motion: reduce)` — сбрасывать duration анимаций до ~0. Не `0s`, а `0.01ms` — чтобы animationend/transitionend события сработали. Это a11y-обязательство, не опция.
