# Селекторы и at-rules

Селекторы определяют, **к каким элементам** применяются стили. At-rules управляют **поведением CSS** на мета-уровне: адаптивность, слои каскада, шрифты, анимации. На senior-уровне ключевые темы — функциональные псевдоклассы (`:has()`, `:is()`, `:where()`), `@layer` для управления каскадом и `@property` для типизированных custom properties.

---

## Типы селекторов

### Базовые

| Селектор | Синтаксис | Пример | Специфичность |
|---|---|---|---|
| Элемент | `element` | `div`, `p`, `a` | (0,0,1) |
| Класс | `.class` | `.card`, `.is-active` | (0,1,0) |
| ID | `#id` | `#header` | (1,0,0) |
| Универсальный | `*` | `*` | (0,0,0) |
| Атрибут | `[attr]` | `[disabled]`, `[type="text"]` | (0,1,0) |

### Комбинаторы

| Комбинатор | Синтаксис | Что выбирает |
|---|---|---|
| Descendant | `A B` | B — потомок A (на любом уровне) |
| Child | `A > B` | B — прямой потомок A |
| Adjacent sibling | `A + B` | B — сосед сразу после A |
| General sibling | `A ~ B` | B — любой сосед после A |

```css
/* Descendant: любой <p> внутри .card */
.card p { color: gray; }

/* Child: только прямые <li> в <ul> */
ul > li { list-style: disc; }

/* Adjacent sibling: <p> сразу после <h2> */
h2 + p { margin-top: 0; }

/* General sibling: все <p> после <h2> на том же уровне */
h2 ~ p { color: gray; }
```

### Селекторы атрибутов

| Синтаксис | Что выбирает | Пример |
|---|---|---|
| `[attr]` | Элемент с атрибутом | `[disabled]` |
| `[attr="val"]` | Точное совпадение | `[type="email"]` |
| `[attr^="val"]` | Начинается с | `[href^="https"]` |
| `[attr$="val"]` | Заканчивается на | `[href$=".pdf"]` |
| `[attr*="val"]` | Содержит подстроку | `[class*="card"]` |
| `[attr~="val"]` | Содержит слово (через пробел) | `[class~="active"]` |
| `[attr\|="val"]` | Равно или начинается с `val-` | `[lang\|="en"]` |

```css
/* Внешние ссылки — иконка после текста */
a[href^="http"]:not([href*="mysite.com"])::after {
  content: " ↗";
}
/* PDF-ссылки */
a[href$=".pdf"]::before { content: "📄 "; }
```

## Псевдоклассы

### Структурные

| Псевдокласс | Что выбирает |
|---|---|
| `:first-child` | Первый потомок родителя |
| `:last-child` | Последний потомок |
| `:nth-child(an+b)` | По формуле (n = 0, 1, 2...) |
| `:nth-of-type(an+b)` | По формуле, но только среди того же типа |
| `:only-child` | Единственный потомок |
| `:empty` | Элемент без потомков (включая текст) |

**Формула `an+b`:**

```css
:nth-child(2n)      /* чётные: 2, 4, 6... = even */
:nth-child(2n+1)    /* нечётные: 1, 3, 5... = odd */
:nth-child(3n)      /* каждый третий: 3, 6, 9... */
:nth-child(-n+3)    /* первые три: 1, 2, 3 */
:nth-child(n+4)     /* начиная с 4-го */
:nth-child(3n+2)    /* 2, 5, 8, 11... */
```

### State (состояние)

| Псевдокласс | Когда применяется |
|---|---|
| `:hover` | Курсор над элементом |
| `:active` | Момент нажатия (mousedown) |
| `:focus` | Элемент в фокусе (любой способ) |
| `:focus-visible` | Фокус **с клавиатуры** (не от клика мышью) |
| `:focus-within` | Фокус на элементе **или любом потомке** |
| `:visited` | Посещённая ссылка (ограничен по безопасности — только цвет) |
| `:target` | Элемент, чей `id` совпадает с fragment URL (`#section`) |

<b class="text-on-semi-light">`:focus-visible` vs `:focus`:</b> `:focus` срабатывает при любом фокусе (клик мыши тоже). `:focus-visible` — только при клавиатурном фокусе. Это позволяет показывать outline только пользователям клавиатуры, не мешая мышиным.

```css
/* Убираем outline для мыши, оставляем для клавиатуры */
button:focus { outline: none; }
button:focus-visible { outline: 2px solid var(--primary); }
```

### Form (формы)

`:valid`, `:invalid`, `:required`, `:optional`, `:disabled`, `:enabled`, `:checked`, `:indeterminate`, `:placeholder-shown`, `:user-invalid` (после взаимодействия), `:read-only`, `:read-write`

```css
/* Стилизация валидного/невалидного поля после взаимодействия */
input:user-invalid { border-color: red; }
input:valid { border-color: green; }
```

### Функциональные (senior focus)

#### `:is()` — группировка без дублирования

```css
/* Без :is() — повторение */
header a:hover, footer a:hover, nav a:hover { color: red; }

/* С :is() — чисто */
:is(header, footer, nav) a:hover { color: red; }
```

<span class="text-gray">Специфичность `:is()` = максимальная из аргументов. `:is(#id, .class)` → (1,0,0).</span>

#### `:where()` — как `:is()`, но нулевая специфичность

```css
/* Reset-стили с нулевой специфичностью — легко переопределяются */
:where(h1, h2, h3, h4) { margin-top: 0; font-weight: normal; }

/* Любой класс побеждает :where() */
.section-title { font-weight: bold; } /* победит */
```

<b class="text-on-semi-light">На фронте:</b> `:where()` идеален для CSS-ресетов и базовых стилей: задаёшь defaults с нулевой специфичностью, которые любой авторский стиль легко переопределяет.

#### `:not()` — исключение

```css
/* Все ссылки кроме класса .logo */
a:not(.logo) { text-decoration: underline; }

/* Все inputs кроме disabled */
input:not(:disabled) { cursor: pointer; }
```

#### `:has()` — «родительский селектор»

Революционный селектор — позволяет стилизовать элемент **на основе его потомков или соседей**. Ранее это было возможно только через JavaScript.

```css
/* Карточка с изображением получает grid-layout */
.card:has(img) {
  display: grid;
  grid-template-rows: 200px 1fr;
}

/* Форма с ошибкой — кнопка полупрозрачная */
form:has(:invalid) .submit-btn {
  opacity: 0.5;
  pointer-events: none;
}

/* Body с открытым dialog — blur фона */
body:has(dialog[open]) > main {
  filter: blur(4px);
}

/* h2, за которым идёт p — уменьшаем нижний отступ */
h2:has(+ p) { margin-bottom: 0.5rem; }

/* Список с 4+ элементами — двухколоночный */
ul:has(li:nth-child(4)) { columns: 2; }
```

**Плохо — JS-классы для условного стиля:**

```js
// Вручную переключаем класс
if (form.querySelector(':invalid')) {
  submitBtn.classList.add('disabled');
}
```

**Хорошо — чистый CSS с `:has()`:**

```css
form:has(:invalid) .submit-btn { opacity: 0.5; pointer-events: none; }
```

## Псевдоэлементы

| Псевдоэлемент | Что создаёт/выбирает |
|---|---|
| `::before` | Виртуальный элемент перед содержимым (требует `content`) |
| `::after` | Виртуальный элемент после содержимого (требует `content`) |
| `::marker` | Маркер списка (`<li>`, `<summary>`) |
| `::placeholder` | Placeholder текст в `<input>` / `<textarea>` |
| `::selection` | Выделенный пользователем текст |
| `::backdrop` | Фон за `<dialog>`, fullscreen элементом |
| `::first-line` | Первая строка текста |
| `::first-letter` | Первая буква текста |

```css
/* Обязательные поля — звёздочка после label */
label.required::after {
  content: " *";
  color: red;
}

/* Кастомный маркер списка */
li::marker { content: "→ "; color: var(--primary); }

/* Frosted glass backdrop для модалки */
dialog::backdrop { background: rgba(0, 0, 0, 0.5); backdrop-filter: blur(4px); }
```

<span class="text-gray">Один `:` тоже работает (`:before`), но `::` — правильная нотация для псевдоэлементов (CSS3). Двоеточие отличает псевдоэлементы от псевдоклассов.</span>

### `@layer` — управление каскадом

Cascade Layers позволяют **явно задать порядок приоритета** групп стилей, независимо от порядка объявления и специфичности внутри слоёв.

```css
/* 1. Объявляем порядок слоёв */
@layer reset, base, components, utilities;

/* 2. Стили в каждом слое */
@layer reset {
  *, *::before, *::after { margin: 0; box-sizing: border-box; }
}

@layer base {
  body { font-family: 'Inter', sans-serif; line-height: 1.6; }
  a { color: var(--primary); }
}

@layer components {
  .card { padding: 16px; border-radius: 8px; }
  .btn { padding: 8px 16px; }
}

@layer utilities {
  .hidden { display: none !important; }
  .sr-only { position: absolute; width: 1px; height: 1px; overflow: hidden; }
}
```

<b class="text-on-semi-light">Ключевые правила:</b>

- Последний объявленный слой имеет **наивысший приоритет** (utilities побеждает components)
- **Un-layered стили** (вне @layer) побеждают **все** слои
- Внутри слоя — обычный каскад (специфичность, порядок)
- Third-party CSS можно изолировать: `@import url('lib.css') layer(vendor);`

<b class="text-on-semi-light">На фронте:</b> `@layer` решает проблему «стили библиотеки перебивают мои» без `!important`. Помещаешь библиотечные стили в слой с низким приоритетом — свои стили автоматически побеждают.

### `@property` — типизированные custom properties

```css
@property --hue {
  syntax: '<angle>';       /* тип значения */
  inherits: false;         /* наследуется ли */
  initial-value: 0deg;     /* значение по умолчанию */
}

.box {
  background: hsl(var(--hue) 80% 60%);
  transition: --hue 0.5s;  /* теперь browser знает тип и может анимировать! */
}
.box:hover { --hue: 180deg; }
```

<span class="text-gray">Без `@property` browser не знает, что `--hue` — это angle, и не может плавно анимировать изменение. `@property` решает эту проблему, объявляя тип значения.</span>

### `@supports` — feature detection

```css
/* Применить только если browser поддерживает container queries */
@supports (container-type: inline-size) {
  .sidebar { container-type: inline-size; }
}

/* Negation */
@supports not (backdrop-filter: blur(10px)) {
  .overlay { background: rgba(0, 0, 0, 0.8); }
}
```

### `@scope` — scoped стили

```css
/* Стили только внутри .card, но не внутри .card .nested */
@scope (.card) to (.nested) {
  p { color: gray; }  /* только p между .card и .nested */
}
```

<span class="text-gray">`@scope` — «donut scope» (стили с дыркой). Пока ограниченная поддержка, но полезен для компонентных стилей без Shadow DOM.</span>

## CSS-функции

| Функция | Назначение | Пример |
|---|---|---|
| `calc()` | Вычисления с разными единицами | `width: calc(100% - 2rem)` |
| `min()` | Меньшее из значений | `width: min(600px, 100%)` |
| `max()` | Большее из значений | `width: max(300px, 50%)` |
| `clamp()` | min, preferred, max | `font-size: clamp(1rem, 2.5vw, 2rem)` |
| `var()` | CSS custom property с fallback | `color: var(--primary, #333)` |
| `env()` | Environment variables (safe areas) | `padding-bottom: env(safe-area-inset-bottom)` |
| `color-mix()` | Смешивание цветов | `color-mix(in oklch, red 70%, blue)` |
| `light-dark()` | Значение для light/dark theme | `color: light-dark(#333, #eee)` |

```css
/* env() — отступы для notch/island на iOS */
body {
  padding-top: env(safe-area-inset-top);
  padding-bottom: env(safe-area-inset-bottom);
}
```

## Частые вопросы на собесе

**В чём разница `:is()` и `:where()`?**
Одинаковое поведение — группировка селекторов. Разница в специфичности: `:is()` берёт максимальную из аргументов, `:where()` — всегда 0. `:where()` для ресетов и defaults, `:is()` для обычных стилей.

**Как `:has()` работает и зачем?**
`:has()` стилизует элемент на основе его потомков или соседей — «parent selector». `form:has(:invalid)` выбирает форму с невалидным полем. Заменяет JS-переключение классов для условных стилей.

**Как работает `@layer`?**
Объявляешь порядок слоёв: `@layer reset, base, components, utilities`. Последний слой побеждает. Un-layered стили побеждают все слои. Позволяет контролировать каскад без `!important` и войн специфичности.

**Что даёт `@property`?**
Типизирует custom properties: объявляешь syntax (`<color>`, `<angle>`, `<length>`), inherit и initial-value. Главный бенефит — browser может **анимировать** typed custom properties, что невозможно без `@property`.

**Зачем `::before` и `::after` нужен `content`?**
Без `content` псевдоэлемент не существует в render tree. Даже `content: ""` (пустая строка) необходим для отображения. Используется для декоративных элементов, иконок, маркеров.
