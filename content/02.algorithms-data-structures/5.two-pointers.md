### Valid Palindrome

[`Leetcode - 125. Valid Palindrome`](https://leetcode.com/problems/valid-palindrome/description/) — задача, в которой требуется определить, является ли строка палиндромом, учитывая только буквенно-цифровые символы и игнорируя регистр.

**Основные характеристики:**

1. **Входные данные:** строка, которая может содержать буквы, цифры и другие символы.
2. **Цель:** проверить, читается ли строка одинаково слева направо и справа налево (без учёта регистра и неалфавитных символов).
3. **Сложность:** `O(n)` — один проход по строке.

**Двухуказательный метод (оптимальное):**

Идея — сначала очистить строку от неалфавитных символов и привести её к нижнему регистру. Затем проверить с двух концов.

```ts
function isPalindrome(s: string): boolean {
  const str = s.toLowerCase().replace(/[^a-z0-9]/g, "");

  let left = 0;
  let right = str.length - 1;

  while (left < right) {
    if (str[left] !== str[right]) {
      return false;
    }

    left++;
    right--;
  }

  return true;
}
```

**Сравнение с перевёрнутой строкой (простое):**

Идея — преобразовать строку, а затем сравнить её с её реверсом.

```ts
function isPalindrome(s: string): boolean {
  const str = s.toLowerCase().replace(/[^a-z0-9]/g, "");
  return str === str.split("").reverse().join("");
}
```

- Это решение проще, но менее эффективно: создаётся новая строка и массив.
- Сложность по времени `O(n)`, по памяти — `O(n)`.

**Без дополнительной строки (итерация по исходной строке):**

Вместо предварительной фильтрации можно сразу использовать два указателя и пропускать ненужные символы «на лету».

```ts
function isPalindrome(s: string): boolean {
  let left = 0;
  let right = s.length - 1;

  while (left < right) {
    while (left < right && !/[a-z0-9]/i.test(s[left])) left++;
    while (left < right && !/[a-z0-9]/i.test(s[right])) right--;

    if (s[left].toLowerCase() !== s[right].toLowerCase()) {
      return false;
    }

    left++;
    right--;
  }

  return true;
}
```

- Это решение не создаёт дополнительных строк.
- Сложность по времени `O(n)`, по памяти `O(1)`.

**Сравнение решений:**

- Двухуказательный метод — баланс простоты и эффективности: создаём новую строку, но проверка идёт оптимально.
- Сравнение с перевёрнутой строкой — самое простое для понимания, но использует больше памяти.
- Без дополнительной строки — самое оптимальное по памяти, но код чуть сложнее.

### Is Subsequence

[`Leetcode - 392. Is Subsequence`](https://leetcode.com/problems/is-subsequence/description/) — задача, в которой нужно проверить, является ли строка `s` подпоследовательностью строки `t`.

**Основные характеристики:**

1. **Входные данные:** две строки `s` и `t`.
2. **Цель:** проверить, можно ли получить строку `s` из `t`, удаляя некоторые (или ни одного) символы без изменения порядка.
3. **Сложность:** `O(|t|)` — достаточно одного прохода по строке `t`.

**Решение через два указателя (простое и оптимальное):**

```ts
function isSubsequence(s: string, t: string): boolean {
  let i = 0;
  let j = 0;

  while (i < t.length && j < s.length) {
    if (t[i] === s[j]) {
      j++;
    }
    i++;
  }

  return j === s.length;
}
```

- Идея: проходим по строке `t` и двигаем указатель `j` по `s`, когда символы совпадают.
- Сложность: `O(|t|)`, память `O(1)`.

**Решение через метод `indexOf`:**

```ts
function isSubsequence(s: string, t: string): boolean {
  let index = -1;

  for (const char of s) {
    index = t.indexOf(char, index + 1);
    if (index === -1) return false;
  }

  return true;
}
```

- Идея: ищем каждый символ из `s` в строке `t`, начиная с позиции после найденного.
- Сложность: в худшем случае `O(|s|·|t|)`, но на практике для небольших строк работает нормально.

**Решение через динамическое программирование (подходит для большого количества запросов):**

```ts
function preprocess(t: string): number[][] {
  const dp: number[][] = Array.from({ length: t.length + 1 }, () =>
    Array(26).fill(-1)
  );

  for (let i = t.length - 1; i >= 0; i--) {
    for (let c = 0; c < 26; c++) {
      dp[i][c] = dp[i + 1][c];
    }
    dp[i][t.charCodeAt(i) - 97] = i + 1;
  }

  return dp;
}

function isSubsequenceDP(s: string, t: string): boolean {
  const dp = preprocess(t);
  let pos = 0;

  for (const char of s) {
    pos = dp[pos][char.charCodeAt(0) - 97];
    if (pos === -1) return false;
  }

  return true;
}
```

- Идея: заранее для каждой позиции строки `t` храним, где встретится следующий символ.
- Сложность: предобработка `O(|t|·|Σ|)`, каждый запрос `O(|s|)`.
- Удобно, если нужно проверять много строк `s` на одной строке `t`.

**Сравнение решений:**

- Два указателя — простое и оптимальное решение для одного запроса.
- `indexOf` — короткий и читаемый вариант, но может быть медленнее.
- ДП — лучший вариант, если нужно много проверок на одной строке `t`.

### Two Sum II - Input Array Is Sorted

[`Leetcode - 167. Two Sum II - Input Array Is Sorted`](https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой дан отсортированный массив чисел и целевое значение, и нужно найти индексы двух чисел, сумма которых равна этому значению.

**Основные характеристики:**

1. **Входные данные:** отсортированный массив `numbers` и целевое число `target`.
2. **Цель:** найти два индекса (нумерация с 1), таких что `numbers[i] + numbers[j] = target`.
3. **Сложность:** оптимально решается за `O(n)` с использованием двух указателей.

**Решение через два указателя (оптимальное):**

```ts
function twoSum(numbers: number[], target: number): number[] {
  let left = 0;
  let right = numbers.length - 1;

  while (left < right) {
    const sum = numbers[left] + numbers[right];

    if (sum === target) {
      return [left + 1, right + 1];
    } else if (sum < target) {
      left++;
    } else {
      right--;
    }
  }

  return [-1, -1];
}
```

- Идея: используем отсортированность массива. Двигаем левый указатель вправо, если сумма меньше цели, и правый влево — если сумма больше.
- Сложность: `O(n)`, память `O(1)`.

**Решение через бинарный поиск:**

```ts
function twoSum(numbers: number[], target: number): number[] {
  for (let i = 0; i < numbers.length; i++) {
    let left = i + 1;
    let right = numbers.length - 1;
    const complement = target - numbers[i];

    while (left <= right) {
      const mid = Math.floor((left + right) / 2);
      if (numbers[mid] === complement) {
        return [i + 1, mid + 1];
      } else if (numbers[mid] < complement) {
        left = mid + 1;
      } else {
        right = mid - 1;
      }
    }
  }

  return [-1, -1];
}
```

- Идея: для каждого числа ищем его пару бинарным поиском.
- Сложность: `O(n log n)`.
- Память: `O(1)`.

**Решение через HashMap (универсальное, но лишнее для отсортированного массива):**

```ts
function twoSum(numbers: number[], target: number): number[] {
  const map = new Map<number, number>();

  for (let i = 0; i < numbers.length; i++) {
    const complement = target - numbers[i];

    if (map.has(complement)) {
      return [map.get(complement)! + 1, i + 1];
    }

    map.set(numbers[i], i);
  }

  return [-1, -1];
}
```

- Идея: сохраняем ранее встреченные числа и проверяем, есть ли в мапе дополнение.
- Сложность: `O(n)`.
- Память: `O(n)`.

**Сравнение решений:**

- **Два указателя** — оптимальное решение для отсортированного массива (`O(n)`, `O(1)` память).
- **Бинарный поиск** — медленнее (`O(n log n)`), но всё ещё корректно.
- **HashMap** — универсальное решение, хорошо для несортированных массивов, но избыточно для отсортированных.

### Container With Most Water

[`Leetcode - 11. Container With Most Water`](https://leetcode.com/problems/container-with-most-water/description/) — задача, в которой дан массив высот, и нужно найти две линии, которые вместе с осью X образуют контейнер, содержащий наибольшее количество воды.

**Основные характеристики:**

1. **Входные данные:** массив целых чисел, где `height[i]` — высота линии на позиции `i`.
2. **Цель:** найти максимальную площадь воды, которую можно удержать между двумя линиями.
3. **Сложность:** оптимальное решение работает за `O(n)`.

**Решение через полный перебор (brute force):**

```ts
function maxArea(height: number[]): number {
  let max = 0;

  for (let i = 0; i < height.length; i++) {
    for (let j = i + 1; j < height.length; j++) {
      const area = (j - i) * Math.min(height[i], height[j]);
      max = Math.max(max, area);
    }
  }

  return max;
}
```

- Идея: проверить все пары линий и вычислить площадь.
- Сложность: `O(n^2)` — слишком медленно для больших входных данных.

**Оптимальное решение (двухуказательный метод):**

```ts
function maxArea(height: number[]): number {
  let left = 0;
  let right = height.length - 1;
  let max = 0;

  while (left < right) {
    const xDiff = right - left;
    const yDiff = Math.min(height[left], height[right]);

    max = Math.max(max, xDiff * yDiff);

    if (height[left] < height[right]) {
      left++;
    } else {
      right--;
    }
  }

  return max;
}
```

- Идея: начинаем с крайних линий и постепенно двигаем указатель меньшей высоты внутрь.
- Сложность: `O(n)`, память `O(1)`.

**Вариант с бинарным поиском улучшений (редко применяется):**

```ts
function maxArea(height: number[]): number {
  let left = 0;
  let right = height.length - 1;
  let max = 0;

  while (left < right) {
    const area = (right - left) * Math.min(height[left], height[right]);
    max = Math.max(max, area);

    if (height[left] < height[right]) {
      const prev = height[left];
      while (left < right && height[left] <= prev) left++; // пропуск меньших значений
    } else {
      const prev = height[right];
      while (left < right && height[right] <= prev) right--; // пропуск меньших значений
    }
  }

  return max;
}
```

- Идея: дополнительно пропускаем заведомо невыгодные позиции.
- Иногда быстрее на практике, но асимптотика та же `O(n)`.

**Сравнение решений:**

- **Brute force:** легко реализовать, но работает медленно — `O(n^2)`.
- **Два указателя:** стандартное оптимальное решение — `O(n)`.
- **Два указателя с пропусками:** немного оптимизирует практическое время работы, но сложнее для понимания.

### 3Sum

[`Leetcode - 15. 3Sum`](https://leetcode.com/problems/3sum/description/) — задача, в которой нужно найти все уникальные тройки чисел в массиве, сумма которых равна нулю.

**Основные характеристики:**

1. **Входные данные:** массив целых чисел (может содержать отрицательные и положительные значения).
2. **Цель:** найти все уникальные тройки `(a, b, c)`, такие что `a + b + c = 0`.
3. **Сложность:** в оптимальных решениях достигается `O(n²)`.

**Решение через brute force (три цикла):**

```ts
function threeSumBrute(nums: number[]): number[][] {
  const res: number[][] = [];
  const n = nums.length;

  for (let i = 0; i < n; i++) {
    for (let j = i + 1; j < n; j++) {
      for (let k = j + 1; k < n; k++) {
        if (nums[i] + nums[j] + nums[k] === 0) {
          const triplet = [nums[i], nums[j], nums[k]].sort((a, b) => a - b);
          if (
            !res.some(
              (r) =>
                r[0] === triplet[0] &&
                r[1] === triplet[1] &&
                r[2] === triplet[2]
            )
          ) {
            res.push(triplet);
          }
        }
      }
    }
  }

  return res;
}
```

- Простая реализация, но очень медленная: `O(n³)`.
- Для больших массивов непригодна.

**Решение через сортировку + два указателя (оптимальное):**

```ts
function threeSum(nums: number[]): number[][] {
  if (nums.length < 3) return [];

  const res: number[][] = [];
  nums.sort((a, b) => a - b);

  for (let i = 0; i < nums.length - 2; i++) {
    if (i > 0 && nums[i] === nums[i - 1]) continue;

    let left = i + 1;
    let right = nums.length - 1;

    while (left < right) {
      const sum = nums[i] + nums[left] + nums[right];

      if (sum === 0) {
        res.push([nums[i], nums[left], nums[right]]);
        left++;
        right--;

        while (left < right && nums[left] === nums[left - 1]) left++;
        while (left < right && nums[right] === nums[right + 1]) right--;
      } else if (sum < 0) {
        left++;
      } else {
        right--;
      }
    }
  }

  return res;
}
```

- Основная идея: фиксируем один элемент и ищем два других через два указателя.
- Время: `O(n²)`, что является оптимальным.
- Убираем дубликаты с помощью пропусков одинаковых чисел.

**Решение через HashSet (альтернативное):**

```ts
function threeSumHash(nums: number[]): number[][] {
  const res: number[][] = [];
  const seen = new Set<string>();

  nums.sort((a, b) => a - b);

  for (let i = 0; i < nums.length - 2; i++) {
    const target = -nums[i];
    const set = new Set<number>();

    for (let j = i + 1; j < nums.length; j++) {
      const complement = target - nums[j];
      if (set.has(complement)) {
        const triplet = [nums[i], complement, nums[j]];
        const key = triplet.join(",");
        if (!seen.has(key)) {
          res.push(triplet);
          seen.add(key);
        }
      }
      set.add(nums[j]);
    }
  }

  return res;
}
```

- Использует `Set` для поиска пар.
- Тоже работает за `O(n²)`, но с большим оверхедом по памяти.
- Хорош для понимания идеи поиска пар через HashSet.

**Сравнение решений:**

- **Brute force**: легко реализовать, но слишком медленно (`O(n³)`).
- **Два указателя**: оптимальное решение (`O(n²)`, `O(1)` памяти).
- **HashSet**: тоже `O(n²)`, но требует больше памяти, иногда проще для восприятия.

На практике почти всегда используется **решение с двумя указателями**.
