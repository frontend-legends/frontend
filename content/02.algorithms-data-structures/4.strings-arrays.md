### Merge Sorted Arrays

[`Leetcode - 88. Merge Sorted Arrays`](https://leetcode.com/problems/merge-sorted-array/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой даны два отсортированных массива, и нужно объединить их в один отсортированный массив.

**Основные характеристики:**

1. **Входные данные:** два отсортированных массива.
2. **Цель:** объединить их в один отсортированный массив на месте (без использования дополнительной памяти).
3. **Сложность:** `O(m + n)` — проходим по обоим массивам один раз.

**Решение через сортировку (простое):**

```ts
function merge(nums1: number[], m: number, nums2: number[], n: number): void {
  if (m + n === m) {
    return;
  }

  let j = 0;

  for (let i = 0; i < nums1.length; i++) {
    const el1 = nums1[i];

    if (j >= nums2.length) break;

    if (el1 === 0) {
      nums1[i] = nums2[j];

      j++;
    }
  }

  nums1.sort((a, b) => a - b);
}
```

**Оптимальное решение (двухуказательный метод):**

```ts
function merge(nums1: number[], m: number, nums2: number[], n: number): void {
  // последний индекс "настоящих" элементов nums1
  let i = m - 1;
  // последний индекс nums2
  let j = n - 1;
  // последний индекс результирующего массива
  let k = m + n - 1;

  while (j >= 0) {
    if (i >= 0 && nums1[i] > nums2[j]) {
      nums1[k] = nums1[i];
      i--;
    } else {
      nums1[k] = nums2[j];
      j--;
    }
    k--;
  }
}
```

**Сравнение решений:**

- Простое решение легче понять и быстро накидать, но работает медленнее `(O((m+n)·log(m+n)))`.

- Оптимальное решение использует идею двух указателей и работает за `O(m+n)`, что важно для больших массивов.

### Remove Element

[`Leetcode - 27. Remove Element`](https://leetcode.com/problems/remove-element/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно удалить все вхождения заданного элемента из массива «на месте» и вернуть новую длину массива.

**Основные характеристики:**

1. **Входные данные:** массив чисел `nums` и число `val`, которое нужно удалить.
2. **Цель:** удалить все элементы, равные `val`, изменив массив на месте (без использования дополнительного массива).
3. **Сложность:** `O(n)` — проходим по массиву один раз.

**Наивное решение (сортировка после удаления):**

```ts
function removeElement(nums: number[], val: number): number {
  let removedItemsCount = 0;

  for (let i = 0; i < nums.length; i++) {
    const el = nums[i];

    if (el === val) {
      nums[i] = 0;
      removedItemsCount++;
    }
  }

  nums.sort((a, b) => b - a);

  const arrLen = nums.length - removedItemsCount;

  return arrLen;
}
```

- Минус: мы используем сортировку после замены, что даёт сложность `O(n log n)` вместо линейной.
- Плюс: код простой и легко понять.

**Оптимальное решение (двухуказательный метод):**

```ts
function removeElement(nums: number[], val: number): number {
  let k = 0; // индекс для записи "правильных" элементов

  for (let i = 0; i < nums.length; i++) {
    if (nums[i] !== val) {
      nums[k] = nums[i];
      k++;
    }
  }

  return k;
}
```

- В этом решении мы переписываем массив, сохраняя только нужные элементы.
- Итоговый массив в пределах `[0, k)` содержит все элементы, отличные от `val`.
- Сложность: `O(n)` — один проход по массиву.

**Сравнение решений:**

- Наивный подход проще для быстрого старта, но медленнее и неэффективен.
- Оптимальное решение использует технику двух указателей и позволяет добиться линейной сложности без сортировки.

### Remove Duplicates from Sorted Array

[`Leetcode - 26. Remove Duplicates from Sorted Array`](https://leetcode.com/problems/remove-duplicates-from-sorted-array/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно удалить дубликаты из отсортированного массива на месте и вернуть новую длину массива.

**Основные характеристики:**

1. **Входные данные:** отсортированный массив `nums`.
2. **Цель:** удалить все дубликаты так, чтобы каждый элемент встречался только один раз, сохранив порядок.
3. **Сложность:** `O(n)` — один проход по массиву.

**Решение (двухуказательный метод):**

```ts
function removeDuplicates(nums: number[]): number {
  if (nums.length === 0) return 0;
  if (nums.length === 1) return 1;

  let k = 1;

  for (let i = 1; i < nums.length; i++) {
    if (nums[i - 1] !== nums[i]) {
      nums[k] = nums[i];
      k++;
    }
  }

  nums.length = k;

  return k;
}
```

**Как это работает:**

- Начинаем с индекса `k = 1` — туда будем записывать уникальные элементы.
- Проходим по массиву с `i = 1` и сравниваем текущий элемент с предыдущим.
- Если элементы разные, значит найден новый уникальный элемент. Мы записываем его на позицию `k` и увеличиваем `k`.
- В итоге все уникальные элементы оказываются в диапазоне `[0, k)`.

**Пример:**

```
nums = [1, 1, 2, 2, 3]
После работы алгоритма: [1, 2, 3]
Возвращаемое значение: 3
```

**Сравнение с наивным решением:**

- Наивный подход мог бы использовать `Set` или фильтрацию (`filter`), но это бы потребовало дополнительной памяти.
- Оптимальное решение работает на месте, не создавая новых массивов, и проходит входной массив только один раз.

### Remove Duplicates from Sorted Array II

[`Leetcode - 80. Remove Duplicates from Sorted Array II`](https://leetcode.com/problems/remove-duplicates-from-sorted-array-ii/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно удалить дубликаты из отсортированного массива так, чтобы каждый элемент встречался **не более двух раз**.

**Основные характеристики:**

1. **Входные данные:** отсортированный массив `nums`.
2. **Цель:** удалить дубликаты на месте так, чтобы каждый элемент встречался не чаще двух раз, сохранив порядок.
3. **Сложность:** `O(n)` — один проход по массиву.

**Решение (двухуказательный метод):**

```ts
function removeDuplicates(nums: number[]): number {
  let k = 0;

  for (let i = 0; i < nums.length; i++) {
    if (k < 2 || nums[i] !== nums[k - 2]) {
      nums[k] = nums[i];
      k++;
    }
  }

  return k;
}
```

**Как это работает:**

- Мы используем указатель `k`, чтобы записывать допустимые элементы.
- Условие `k < 2` позволяет первые два элемента всегда добавлять в массив.
- Для всех остальных элементов проверяем, отличается ли текущий `nums[i]` от элемента на позиции `k - 2`. Если отличается — это значит, что в результирующем массиве ещё нет двух одинаковых подряд, и мы можем добавить его.
- В итоге в диапазоне `[0, k)` массив содержит все элементы без лишних повторов (максимум по 2 каждого).

**Пример:**

```
nums = [1, 1, 1, 2, 2, 3]
После работы алгоритма: [1, 1, 2, 2, 3]
Возвращаемое значение: 5
```

**Сравнение с предыдущей задачей (Remove Duplicates from Sorted Array):**

- В задаче `Remove Duplicates from Sorted Array` каждый элемент должен встречаться только один раз.
- В этой задаче разрешено по два одинаковых элемента, поэтому условие проверки чуть сложнее — мы сравниваем текущий элемент с тем, что находится на позиции `k - 2`.
- Оба решения работают за `O(n)` времени и на месте, не используя дополнительной памяти.

### Majority Element

[`Leetcode - 169. Majority Element`](https://leetcode.com/problems/majority-element/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно найти элемент, встречающийся больше чем `n/2` раз в массиве.

**Основные характеристики:**

1. **Входные данные:** массив `nums` длиной `n`.
2. **Цель:** вернуть элемент, который встречается чаще всего (гарантируется, что такой элемент существует).
3. **Сложность:** оптимальное решение работает за `O(n)` по времени и `O(1)` по памяти.

**Решение через хеш-таблицу (наивное):**

```ts
function majorityElement(nums: number[]): number {
  let obj = {};
  let maxCount = 0;
  let maxNum: number | null = null;

  for (let i = 0; i < nums.length; i++) {
    const el = nums[i];

    obj[el] = (obj[el] || 0) + 1;

    if (obj[el] > maxCount) {
      maxCount = obj[el];
      maxNum = el;
    }
  }

  return maxNum;
}
```

**Оптимальное решение (алгоритм [Boyer-Moore Voting Algorithm](https://neerc.ifmo.ru/wiki/index.php?title=%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC_%D0%91%D0%BE%D0%B9%D0%B5%D1%80%D0%B0-%D0%9C%D1%83%D1%80%D0%B0)):**

```ts
function majorityElement(nums: number[]): number {
  let candidate = nums[0];
  let count = 1;

  for (let i = 1; i < nums.length; i++) {
    if (count === 0) {
      candidate = nums[i];
    }

    count += nums[i] === candidate ? 1 : -1;
  }

  return candidate;
}
```

**Как это работает:**

- Идея в том, что если вычёркивать разные элементы парами, то мажоритарный элемент всё равно останется.
- Алгоритм поддерживает «кандидата» и счётчик. Когда счётчик обнуляется, кандидат меняется.
- В конце гарантированно остаётся majority element.

**Пример:**

```
nums = [2, 2, 1, 1, 1, 2, 2]
Результат: 2
```

**Сравнение решений:**

- Хеш-таблица: легко реализовать, но требует дополнительной памяти `O(n)`.
- Бойер–Мур: оптимален, работает за `O(n)` и не требует дополнительной памяти.

### Rotate Array

[`Leetcode - 189. Rotate Array`](https://leetcode.com/problems/rotate-array/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно сдвинуть элементы массива вправо на `k` шагов.

**Основные характеристики:**

1. **Входные данные:** массив `nums` и число `k`.
2. **Цель:** изменить массив на месте так, чтобы каждый элемент оказался сдвинут вправо на `k` позиций.
3. **Сложность:** оптимальное решение достигается за `O(n)` по времени и `O(1)` по памяти.

**Решение через копию массива (наивное):**

```ts
function rotate(nums: number[], k: number): void {
  const copy = [...nums];
  const n = nums.length;

  for (let i = 0; i < copy.length; i++) {
    const el = copy[i];
    let j = (i + k) % n;
    nums[j] = el;
  }
}
```

**Оптимальное решение (три разворота):**

```ts
function rotate(nums: number[], k: number): void {
  const n = nums.length;
  k = k % n;

  reverse(nums, 0, n - 1);
  reverse(nums, 0, k - 1);
  reverse(nums, k, n - 1);
}

function reverse(nums: number[], start: number, end: number): void {
  while (start < end) {
    [nums[start], nums[end]] = [nums[end], nums[start]];
    start++;
    end--;
  }
}
```

**Как это работает:**

- Сначала переворачиваем весь массив.
- Потом переворачиваем первые `k` элементов.
- Наконец, переворачиваем оставшуюся часть массива.
- В итоге массив оказывается сдвинутым на `k` позиций.

**Пример:**

```
nums = [1,2,3,4,5,6,7], k = 3
После алгоритма: [5,6,7,1,2,3,4]
```

**Сравнение решений:**

- Копия массива: проще, но требует `O(n)` дополнительной памяти.
- Метод с разворотами: работает на месте, занимает `O(1)` памяти, что и делает его оптимальным.

### Best Time to Buy and Sell Stock

[`Leetcode - 121. Best Time to Buy and Sell Stock`](https://leetcode.com/problems/best-time-to-buy-and-sell-stock/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно найти максимальную прибыль от одной покупки и одной продажи акции.

**Основные характеристики:**

1. **Входные данные:** массив `prices`, где `prices[i]` — цена акции в `i`-й день.
2. **Цель:** выбрать два дня `i < j`, чтобы максимизировать `prices[j] - prices[i]`.
3. **Сложность:** `O(n)` — один проход по массиву.

**Решение (один проход):**

```ts
function maxProfit(prices: number[]): number {
  let profit = 0;
  let min = prices[0];

  for (let i = 1; i < prices.length; i++) {
    if (prices[i] < min) {
      min = prices[i];
    } else {
      const tempProfit = prices[i] - min;

      if (tempProfit > profit) {
        profit = tempProfit;
      }
    }
  }

  return profit;
}
```

**Как это работает:**

- Мы отслеживаем минимальную цену на данный момент (`min`).
- Для каждой новой цены считаем потенциальную прибыль `prices[i] - min`.
- Если прибыль больше текущего максимума — обновляем `profit`.
- Если встречаем меньшую цену — обновляем `min`.

**Пример:**

```
prices = [7, 1, 5, 3, 6, 4]
Минимум сначала = 7
День 2: цена = 1, обновляем min = 1
День 3: цена = 5, profit = 4
День 4: цена = 3, profit = 4
День 5: цена = 6, profit = 5
День 6: цена = 4, profit = 5
Результат: 5
```

**Сравнение с наивным решением:**

- Наивный подход — перебрать все пары `(i, j)` и искать максимум (`O(n²)`).
- Оптимальное решение проходит массив один раз и использует только текущий минимум и максимальную прибыль (`O(n)`).

### Best Time to Buy and Sell Stock II

[`Leetcode - 122. Best Time to Buy and Sell Stock II`](https://leetcode.com/problems/best-time-to-buy-and-sell-stock-ii/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно найти максимальную прибыль от неограниченного количества сделок (покупка + продажа), при условии, что одновременно нельзя держать более одной акции.

**Основные характеристики:**

1. **Входные данные:** массив `prices`, где `prices[i]` — цена акции в `i`-й день.
2. **Цель:** максимизировать прибыль, делая покупки и продажи многократно.
3. **Сложность:** оптимальное решение работает за `O(n)`.

**Наивное решение (рекурсия / перебор):**

- Для каждого дня пробовать купить или не покупать, потом рекурсивно рассматривать все варианты.
- Очень медленно: экспоненциальная сложность.

**Жадное решение (простое суммирование всех подъёмов):**

```ts
function maxProfit(prices: number[]): number {
  let profit = 0;

  for (let i = 1; i < prices.length; i++) {
    if (prices[i] > prices[i - 1]) {
      profit += prices[i] - prices[i - 1];
    }
  }

  return profit;
}
```

**Как это работает:**

- Любой рост цены можно интерпретировать как прибыль: купили на `i-1`, продали на `i`.
- Суммируем все положительные разницы.

**Пример:**

```
prices = [7,1,5,3,6,4]
День 2: падение, ничего не делаем
День 3: рост 5 - 1 = 4 → profit = 4
День 4: падение
День 5: рост 6 - 3 = 3 → profit = 7
День 6: падение
Результат: 7
```

**DP решение (через состояния hold/sold):**

```ts
function maxProfit(prices: number[]): number {
  let hold = -prices[0];
  let sold = 0;

  for (let i = 1; i < prices.length; i++) {
    hold = Math.max(hold, sold - prices[i]);
    sold = Math.max(sold, hold + prices[i]);
  }

  return sold;
}
```

**Как это работает:**

- `hold` — максимальная прибыль, если в данный момент держим акцию.
- `sold` — максимальная прибыль, если сейчас акций нет.
- На каждом шаге обновляем состояния:
  - Либо продолжаем держать (`hold`), либо покупаем новую акцию.
  - Либо остаёмся без акций (`sold`), либо продаём акцию.

**Сравнение решений:**

- **Наивное** — перебор всех вариантов, экспоненциальное время.
- **Жадное** — быстрое и простое, `O(n)`, интуитивно складывает все подъёмы.
- **DP (hold/sold)** — более общий и гибкий способ, тоже `O(n)`, но полезен для расширенных условий (например, комиссии или ограничение на количество сделок).

### Jump Game

[`Leetcode - 55. Jump Game`](https://leetcode.com/problems/jump-game/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно определить, можно ли допрыгнуть от начала массива до последнего индекса.

**Основные характеристики:**

1. **Входные данные:** массив `nums`, где `nums[i]` — максимальная длина прыжка из позиции `i`.
2. **Цель:** проверить, существует ли путь от индекса `0` до последнего индекса массива.
3. **Сложность:** `O(n)` — один проход по массиву.

**Решение (жадный алгоритм):**

```ts
function canJump(nums: number[]): boolean {
  const n = nums.length;

  let max = 0;

  for (let i = 0; i < n; i++) {
    if (i > max) return false;

    max = Math.max(max, i + nums[i]);

    if (max >= nums.length - 1) return true;
  }

  return true;
}
```

**Как это работает:**

- Переменная `max` хранит самую дальнюю позицию, до которой мы можем допрыгнуть.
- На каждой итерации проверяем: если текущий индекс `i` больше `max`, значит мы «застряли» и нельзя пройти дальше.
- Обновляем `max = Math.max(max, i + nums[i])` — то есть прыгаем настолько далеко, насколько возможно из текущей позиции.
- Если в какой-то момент `max >= последний индекс`, возвращаем `true`.

**Пример:**

```
nums = [2, 3, 1, 1, 4]

i = 0, max = 0 → max = max(0, 0+2) = 2
i = 1, max = 2 → max = max(2, 1+3) = 4 (>= последний индекс)
→ результат: true
```

```
nums = [3, 2, 1, 0, 4]

i = 0, max = 0 → max = 3
i = 1, max = 3 → max = 3
i = 2, max = 3 → max = 3
i = 3, max = 3 → max = 3
i = 4 → но i = 4 > max (3) → false
```

**Сравнение с наивным решением:**

- Наивный подход: рекурсивный перебор всех возможных прыжков (экспоненциальная сложность `O(2^n)`).
- Жадное решение: один проход по массиву, обновляем максимальную достижимую позицию (`O(n)`), работает эффективно даже для больших массивов.

### Jump Game II

[`Leetcode - 45. Jump Game II`](https://leetcode.com/problems/jump-game-ii/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно найти минимальное количество прыжков, чтобы достичь конца массива.

**Основные характеристики:**

1. **Входные данные:** массив `nums`, где `nums[i]` — это максимальная длина прыжка из позиции `i`.
2. **Цель:** найти минимальное количество прыжков для достижения последнего индекса.
3. **Сложность:** оптимальное решение работает за `O(n)`.

**Решение (жадный алгоритм):**

```ts
function jump(nums: number[]): number {
  const n = nums.length;

  let jumps = 0;
  let counter = 0;
  let biggestJump = 0;

  for (let i = 0; i < n - 1; i++) {
    const jumpAvailable = nums[i];

    biggestJump = Math.max(biggestJump, i + jumpAvailable);

    if (counter === i) {
      jumps++;
      counter = biggestJump;
    }
  }

  return jumps;
}
```

**Как это работает:**

- Мы поддерживаем диапазон текущего прыжка (`counter`).
- Для каждой позиции считаем, насколько далеко можно прыгнуть (`biggestJump`).
- Если дошли до конца текущего диапазона (`i == counter`), нужно сделать новый прыжок:
  - увеличиваем `jumps`;
  - обновляем диапазон до `biggestJump`.

- В конце массив гарантированно достижим, поэтому возвращаем `jumps`.

**Пример:**

```
nums = [2, 3, 1, 1, 4]

Итерации:
i=0: biggestJump = max(0, 0+2) = 2
       counter=0 → прыжок=1, counter=2

i=1: biggestJump = max(2, 1+3) = 4

i=2: biggestJump = max(4, 2+1) = 4
       i == counter=2 → прыжок=2, counter=4

Достигли конца массива.
Результат: 2
```

**Сравнение с наивным решением:**

- Наивный подход использует BFS/DFS для перебора всех возможных путей, что даёт экспоненциальное время.
- Можно улучшить до `O(n²)`, проверяя все возможные прыжки из каждой позиции.
- Жадный алгоритм оптимален и работает за `O(n)`, так как мы делаем ровно столько прыжков, сколько необходимо.

### H-Index

[`Leetcode - 274. H-Index`](https://leetcode.com/problems/h-index/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно вычислить **индекс Хирша (h-index)** исследователя на основе количества цитирований его статей.

**Основные характеристики:**

1. **Входные данные:** массив `citations`, где `citations[i]` — количество цитирований `i`-й статьи.
2. **Цель:** найти максимальное число `h`, такое что у исследователя есть как минимум `h` статей с числом цитирований ≥ `h`.
3. **Сложность:** зависит от реализации. Наивное решение может быть `O(n²)`, оптимальные решения достигают `O(n log n)` или `O(n)`.

**Решение (наивное, перебор):**

```ts
function hIndex(citations: number[]): number {
  const n = citations.length;

  for (let i = n; i > 0; i--) {
    const appropriateArticles = citations.filter(
      (article: number) => article >= i
    ).length;

    if (appropriateArticles >= i) {
      return i;
    }
  }

  return 0;
}
```

- Перебираем `i` от `n` до `1`.
- Для каждого `i` считаем, сколько статей имеют `≥ i` цитирований.
- Если таких статей хотя бы `i`, то это и есть `h-index`.
- Сложность: `O(n²)` из-за фильтрации на каждом шаге.

**Решение через сортировку (оптимальное, `O(n log n)`):**

```ts
function hIndex(citations: number[]): number {
  citations.sort((a, b) => b - a); // сортируем по убыванию

  let h = 0;

  for (let i = 0; i < citations.length; i++) {
    if (citations[i] >= i + 1) {
      h = i + 1;
    }
  }

  return h;
}
```

- Сортируем статьи по убыванию.
- Проходим массив: пока `citations[i] ≥ i + 1`, увеличиваем `h`.
- Итоговое значение `h` — и есть H-Index.
- Сложность: `O(n log n)`.

**Решение через counting sort (линейное, `O(n)`):**

```ts
function hIndex(citations: number[]): number {
  const n = citations.length;
  const count = new Array(n + 1).fill(0);

  for (let c of citations) {
    if (c >= n) {
      count[n]++;
    } else {
      count[c]++;
    }
  }

  let total = 0;

  for (let i = n; i >= 0; i--) {
    total += count[i];
    if (total >= i) {
      return i;
    }
  }

  return 0;
}
```

- Подсчитываем, сколько статей имеют `0`, `1`, …, `n` цитирований (все большие значения приравниваем к `n`).
- Идём с конца (`i = n → 0`), суммируем количество статей.
- Когда суммарное число статей ≥ `i`, это и есть H-Index.
- Сложность: `O(n)`, память `O(n)`.

**Пример:**

```
citations = [3, 0, 6, 1, 5]

Сортировка: [6, 5, 3, 1, 0]

i = 0 → citations[0] = 6 ≥ 1 → h = 1
i = 1 → citations[1] = 5 ≥ 2 → h = 2
i = 2 → citations[2] = 3 ≥ 3 → h = 3
i = 3 → citations[3] = 1 < 4 → стоп

Результат: 3
```

**Сравнение решений:**

- Наивное решение (`O(n²)`) слишком медленное.
- Решение через сортировку (`O(n log n)`) оптимально для большинства случаев.
- Решение через [Counting Sort](/content/algorithms-data-structures/sorting#counting-sort) (`O(n)`) самое быстрое, но требует дополнительной памяти `O(n)`.

### Insert Delete GetRandom O(1)

[`Leetcode - 380. Insert Delete GetRandom O(1)`](https://leetcode.com/problems/insert-delete-getrandom-o1/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно реализовать структуру данных, поддерживающую вставку, удаление и случайный доступ за `O(1)`.

**Основные характеристики:**

1. **Операции:**
   - `insert(val)` — добавить элемент, если его нет.
   - `remove(val)` — удалить элемент, если он есть.
   - `getRandom()` — вернуть случайный элемент.

2. **Цель:** все операции должны выполняться за `O(1)` в среднем.
3. **Подход:** используем массив и хеш-таблицу (`Map`) для индексов.

**Реализация:**

```ts
class RandomizedSet {
  private arr: number[];
  private map: Map<number, number>;

  constructor() {
    this.arr = [];
    this.map = new Map();
  }

  insert(val: number): boolean {
    if (this.map.has(val)) return false;

    this.arr.push(val);
    this.map.set(val, this.arr.length - 1);
    return true;
  }

  remove(val: number): boolean {
    if (!this.map.has(val)) return false;

    const index = this.map.get(val)!;
    const last = this.arr[this.arr.length - 1];

    this.arr[index] = last;
    this.map.set(last, index);

    this.arr.pop();
    this.map.delete(val);

    return true;
  }

  getRandom(): number {
    const index = Math.floor(Math.random() * this.arr.length);
    return this.arr[index];
  }
}
```

**Как это работает:**

- Для **insert**: просто добавляем элемент в конец массива и сохраняем его индекс в `map`.
- Для **remove**: чтобы удалить за `O(1)`, меняем удаляемый элемент с последним, обновляем индекс в `map`, удаляем последний элемент.
- Для **getRandom**: берём случайный индекс от `0` до `n - 1` и возвращаем элемент из массива.

**Пример:**

```ts
const set = new RandomizedSet();
set.insert(1); // true
set.insert(2); // true
set.remove(1); // true
set.getRandom(); // вернёт 2 (так как остался один элемент)
```

**Сравнение с наивным подходом:**

- Наивный способ (только массив):
  - `insert` — `O(1)`
  - `remove` — `O(n)` (нужно искать элемент)
  - `getRandom` — `O(1)`

- Оптимальное решение (массив + хеш-таблица): все три операции за `O(1)`.

### Product of Array Except Self

[`Leetcode - 238. Product of Array Except Self`](https://leetcode.com/problems/product-of-array-except-self/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно для каждого элемента массива вернуть произведение всех остальных элементов, не используя операцию деления.

**Основные характеристики:**

1. **Входные данные:** массив `nums` длиной `n`.
2. **Цель:** вернуть массив `res`, где `res[i] = product(nums[j], j ≠ i)`.
3. **Сложность:** оптимальное решение достигается за `O(n)` по времени и `O(1)` по памяти (если не учитывать выходной массив).

**Наивное решение (двойной цикл):**

```ts
function productExceptSelf(nums: number[]): number[] {
  const n = nums.length;
  const res: number[] = [];

  for (let i = 0; i < n; i++) {
    let prod = 1;
    for (let j = 0; j < n; j++) {
      if (i !== j) prod *= nums[j];
    }
    res[i] = prod;
  }

  return res;
}
```

- Для каждого элемента пересчитываем произведение всех остальных.
- Сложность: `O(n²)` по времени, `O(1)` по памяти.
- Слишком медленно для больших массивов.

**Оптимальное решение (префиксы + постфиксы):**

```ts
function productExceptSelf(nums: number[]): number[] {
  const n = nums.length;
  const res: number[] = [];

  let prefix = 1;
  for (let i = 0; i < n; i++) {
    res[i] = prefix;
    prefix *= nums[i];
  }

  let postfix = 1;
  for (let i = n - 1; i >= 0; i--) {
    res[i] *= postfix;
    postfix *= nums[i];
  }

  return res;
}
```

**Как это работает:**

- На первом проходе записываем произведение всех чисел слева от `i`.
- На втором проходе умножаем его на произведение всех чисел справа от `i`.
- Итог: `res[i] = product(left of i) * product(right of i)`.

**Пример:**

```
nums = [1, 2, 3, 4]

Префиксы: [1, 1, 2, 6]
Постфиксы: [24, 12, 4, 1]
Результат: [24, 12, 8, 6]
```

**Сложность:** `O(n)` по времени и `O(1)` по памяти.

**Решение с дополнительными массивами:**

```ts
function productExceptSelf(nums: number[]): number[] {
  const n = nums.length;
  const prefix: number[] = Array(n).fill(1);
  const postfix: number[] = Array(n).fill(1);
  const res: number[] = [];

  for (let i = 1; i < n; i++) {
    prefix[i] = prefix[i - 1] * nums[i - 1];
  }

  for (let i = n - 2; i >= 0; i--) {
    postfix[i] = postfix[i + 1] * nums[i + 1];
  }

  for (let i = 0; i < n; i++) {
    res[i] = prefix[i] * postfix[i];
  }

  return res;
}
```

- Использует два массива для префиксов и постфиксов.
- Память: `O(n)`.
- Логика чуть проще для понимания, но менее оптимально.

**Вариант с делением (ограниченный):**

```ts
function productExceptSelf(nums: number[]): number[] {
  const n = nums.length;
  const res: number[] = [];
  let totalProduct = 1;
  let zeroCount = 0;

  for (let num of nums) {
    if (num === 0) zeroCount++;
    else totalProduct *= num;
  }

  for (let i = 0; i < n; i++) {
    if (zeroCount > 1) res[i] = 0;
    else if (zeroCount === 1) res[i] = nums[i] === 0 ? totalProduct : 0;
    else res[i] = totalProduct / nums[i];
  }

  return res;
}
```

- Работает только если разрешено использовать деление.
- При наличии нулей требует дополнительной обработки.
- Сложность: `O(n)` по времени и `O(1)` по памяти.

**Сравнение решений:**

- Наивное: простое, но `O(n²)`.
- С доп. массивами: `O(n)` и `O(n)` памяти, удобно для обучения.
- Оптимальное (префикс + постфикс): `O(n)` и `O(1)` памяти, лучшее решение.
- С делением: проще, но ненадёжное при `0` в массиве.

### Gas Station

[`Leetcode - 134. Gas Station`](https://leetcode.com/problems/gas-station/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно определить, с какой заправки можно начать движение, чтобы объехать весь круг и вернуться обратно, если это возможно.

**Основные характеристики:**

1. **Входные данные:** два массива одинаковой длины — `gas[i]` (количество топлива на i-й заправке) и `cost[i]` (стоимость топлива, чтобы доехать до следующей заправки).
2. **Цель:** вернуть индекс заправки, с которой можно начать путь и проехать весь круг. Если это невозможно — вернуть `-1`.
3. **Сложность:** оптимальное решение достигается за `O(n)` по времени и `O(1)` по памяти.

**Наивное решение (brute force):**

Пробуем стартовать с каждой заправки и проверяем, можно ли проехать полный круг.

```ts
function canCompleteCircuit(gas: number[], cost: number[]): number {
  const n = gas.length;

  for (let start = 0; start < n; start++) {
    let tank = 0;
    let count = 0;

    while (count < n) {
      const idx = (start + count) % n;
      tank += gas[idx] - cost[idx];
      if (tank < 0) break;
      count++;
    }

    if (count === n) return start;
  }

  return -1;
}
```

- Время: `O(n^2)`
- Память: `O(1)`

**Оптимальное решение (жадный алгоритм):**

```ts
function canCompleteCircuit(gas: number[], cost: number[]): number {
  let tankFueled = 0;
  let totalTank = 0;

  const totalGas = gas.reduce((acc, val) => acc + val, 0);
  const totalCost = cost.reduce((acc, val) => acc + val, 0);

  // Если топлива меньше, чем нужно — решения нет
  if (totalGas < totalCost) return -1;

  let start = 0;

  for (let i = 0; i < gas.length; i++) {
    const diff = gas[i] - cost[i];
    totalTank += diff;
    tankFueled += diff;

    // Если бак опустел, старт переносим на следующую заправку
    if (tankFueled < 0) {
      start = i + 1;
      tankFueled = 0;
    }
  }

  return totalTank >= 0 ? start : -1;
}
```

- Время: `O(n)`
- Память: `O(1)`

**Как это работает:**

- Если суммарное `gas < cost`, то решения нет.
- Проходим по кругу, накапливая баланс топлива.
- Когда баланс падает ниже 0, это значит, что начать путь с текущей стартовой точки невозможно.
- Переносим старт на следующую заправку.
- Гарантия: если решение существует, то найденный `start` будет единственным.

**Пример:**

```

cost = [3,4,5,1,2]

```

**Пояснение:**

- Старт с `0`: топлива не хватает.
- Старт с `3`: накапливается достаточно топлива, чтобы проехать весь круг.

**Сравнение решений:**

- Brute force: простая логика, но медленно (`O(n^2)`).

### Candy

[`Leetcode - 135. Candy`](https://leetcode.com/problems/candy/description/?envType=study-plan-v2&envId=top-interview-150) — задача, в которой нужно раздать детям конфеты в соответствии с их рейтингами.

**Основные характеристики:**

1. **Входные данные:** массив `ratings`, где `ratings[i]` — рейтинг i-го ребёнка.
2. **Цель:** выдать каждому ребёнку хотя бы одну конфету. Ребёнок с более высоким рейтингом должен получать больше конфет, чем его соседи.
3. **Сложность:** оптимальное решение достигается за `O(n)` по времени и `O(n)` по памяти.

**Решение через два прохода (оптимальное):**

```ts
function candy(ratings: number[]): number {
  const n = ratings.length;

  const candies = new Array(n).fill(1);

  // слева направо
  for (let i = 1; i < n; i++) {
    if (ratings[i] > ratings[i - 1]) {
      candies[i] = candies[i - 1] + 1;
    }
  }

  // справа налево
  for (let i = n - 2; i >= 0; i--) {
    if (ratings[i] > ratings[i + 1] && candies[i] <= candies[i + 1]) {
      candies[i] = candies[i + 1] + 1;
    }
  }

  return candies.reduce((acc, val) => acc + val, 0);
}
```

**Как это работает:**

- Изначально каждому ребёнку даём по 1 конфете.
- Проход слева направо: если рейтинг текущего ребёнка выше левого, то он получает на одну конфету больше.
- Проход справа налево: если рейтинг текущего ребёнка выше правого, но у него не больше конфет, то увеличиваем до `candies[i+1] + 1`.
- В итоге условия задачи соблюдены, а сумма — минимальна.

**Пример:**

```
ratings = [1,0,2]
Выдача: 2 + 1 + 2 = 5
```

**Наивное решение (итеративное улучшение):**

```ts
function candy(ratings: number[]): number {
  const n = ratings.length;
  const candies = new Array(n).fill(1);

  let changed = true;
  while (changed) {
    changed = false;
    for (let i = 0; i < n; i++) {
      if (
        i > 0 &&
        ratings[i] > ratings[i - 1] &&
        candies[i] <= candies[i - 1]
      ) {
        candies[i] = candies[i - 1] + 1;
        changed = true;
      }
      if (
        i < n - 1 &&
        ratings[i] > ratings[i + 1] &&
        candies[i] <= candies[i + 1]
      ) {
        candies[i] = candies[i + 1] + 1;
        changed = true;
      }
    }
  }

  return candies.reduce((acc, val) => acc + val, 0);
}
```

**Как это работает:**

- Начинаем с 1 конфеты у каждого.
- Пока есть несоблюдённые условия, корректируем конфеты, проходя по массиву.
- Алгоритм корректный, но может потребовать до `O(n^2)` времени.

**Сравнение решений:**

- Наивный итеративный метод: простой для понимания, но медленный (`O(n^2)`).
- Два прохода: оптимальный метод, работает за `O(n)` и является стандартным решением.

### Trapping Rain Water

[`Leetcode - 42. Trapping Rain Water`](https://leetcode.com/problems/trapping-rain-water/description/) — задача, в которой нужно посчитать, сколько воды может удержаться между столбиками после дождя.

**Основные характеристики:**

1. **Входные данные:** массив `height`, где `height[i]` — высота столбика на позиции `i`.
2. **Цель:** вычислить суммарное количество удержанной воды.
3. **Сложность:** оптимальное решение достигается за `O(n)` по времени и `O(1)` по памяти.

**Решение через префиксные и суффиксные максимумы (наивное, но понятное):**

```ts
function trap(height: number[]): number {
  const n = height.length;
  if (n <= 2) return 0;

  let water = 0;

  const leftHigh = new Array(n).fill(0);
  const rightHigh = new Array(n).fill(0);

  leftHigh[0] = height[0];
  for (let i = 1; i < n; i++) {
    leftHigh[i] = Math.max(leftHigh[i - 1], height[i]);
  }

  rightHigh[n - 1] = height[n - 1];
  for (let i = n - 2; i >= 0; i--) {
    rightHigh[i] = Math.max(rightHigh[i + 1], height[i]);
  }

  for (let i = 0; i < n; i++) {
    water += Math.min(leftHigh[i], rightHigh[i]) - height[i];
  }

  return water;
}
```

**Как это работает:**

- Для каждой позиции находим максимальную высоту слева и справа.
- Вода над этой позицией = `min(leftMax, rightMax) - height[i]`.
- Складываем результаты.

**Сложность:** `O(n)` по времени, `O(n)` по памяти.

**Оптимальное решение (два указателя):**

```ts
function trap(height: number[]): number {
  const n = height.length;
  if (n <= 2) return 0;

  let left = 0;
  let right = n - 1;
  let leftHigh = 0;
  let rightHigh = 0;
  let water = 0;

  while (left < right) {
    if (height[left] < height[right]) {
      if (height[left] >= leftHigh) {
        leftHigh = height[left];
      } else {
        water += leftHigh - height[left];
      }
      left++;
    } else {
      if (height[right] >= rightHigh) {
        rightHigh = height[right];
      } else {
        water += rightHigh - height[right];
      }
      right--;
    }
  }

  return water;
}
```

**Как это работает:**

- Двигаем два указателя (`left` и `right`).
- Всегда двигаем тот, где высота меньше, потому что ограничивающая стена определяется минимальной из двух сторон.
- Обновляем максимум слева/справа и считаем, сколько воды удержится.

**Сложность:** `O(n)` по времени, `O(1)` по памяти.

**Сравнение решений:**

- Префикс/суффикс: проще для понимания, но требует `O(n)` памяти.
- Два указателя: оптимальное решение, работает на месте и требует `O(1)` памяти.

**Пример:**

```
height = [0,1,0,2,1,0,1,3,2,1,2,1]

Ответ: 6
```

Объяснение: вода заполняет ямы между столбиками, максимум 6 единиц воды.

### Roman to Integer

[`Leetcode - 13. Roman to Integer`](https://leetcode.com/problems/roman-to-integer/description/) — задача, в которой нужно преобразовать строку с римским числом в обычное целое число.

**Основные характеристики:**

1. **Входные данные:** строка `s`, представляющая число в римской системе.
2. **Цель:** вернуть целое число, эквивалентное римской записи.
3. **Сложность:** оптимальное решение достигается за `O(n)` по времени и `O(1)` по памяти.

**Решение с проверкой специальных комбинаций (IV, IX, XL, XC, CD, CM):**

```ts
function romanToInt(s: string): number {
  const SYMBOLS = {
    I: 1,
    V: 5,
    X: 10,
    L: 50,
    C: 100,
    D: 500,
    M: 1000,
  };

  const SPECIALS = {
    IV: 4,
    IX: 9,
    XL: 40,
    XC: 90,
    CD: 400,
    CM: 900,
  };

  let result = 0;
  const arr = s.split("");

  for (let i = 0; i < arr.length; i++) {
    const el = arr[i];

    if (i !== arr.length - 1) {
      const nextEl = arr[i + 1];
      const doubleEl = el + nextEl;

      if (SPECIALS[doubleEl] !== undefined) {
        result += SPECIALS[doubleEl];
        i++; // пропускаем второй символ
        continue;
      }
    }

    result += SYMBOLS[el];
  }

  return result;
}
```

**Как это работает:**

- Проверяем каждую букву.
- Если вместе с соседней она образует «специальное» число (например, `IV`), добавляем его значение и пропускаем второй символ.
- Если нет — просто добавляем значение текущей буквы.

**Пример:**

```
s = "MCMXCIV"
Результат = 1994
```

**Оптимальное решение через сравнение текущего и следующего значения:**

```ts
function romanToInt(s: string): number {
  const SYMBOLS = {
    I: 1,
    V: 5,
    X: 10,
    L: 50,
    C: 100,
    D: 500,
    M: 1000,
  };

  let result = 0;

  for (let i = 0; i < s.length; i++) {
    const curr = SYMBOLS[s[i]];
    const next = i < s.length - 1 ? SYMBOLS[s[i + 1]] : 0;

    if (curr < next) {
      result -= curr; // вычитаем, если меньший символ стоит перед большим
    } else {
      result += curr;
    }
  }

  return result;
}
```

**Как это работает:**

- Если текущий символ меньше следующего (`I` перед `V`), то вычитаем.
- Иначе прибавляем.
- Такой подход автоматически учитывает все «специальные» комбинации без отдельной проверки.

**Пример:**

```
s = "MCMXCIV"
Пошагово:
M = 1000 (+1000)
C перед M → вычитаем 100 → 900
M = 1000 (+1000)
X перед C → вычитаем 10 → 90
C = 100 (+100)
I перед V → вычитаем 1 → 4
V = 5 (+5)
Итого = 1994
```

**Сравнение решений:**

- Решение со **специальными комбинациями** более «явное», легко понять новичкам.
- Решение через **сравнение с соседом** — более элегантное и минималистичное, оптимально для собеседований.

### Integer to Roman

[`Leetcode - 12. Integer to Roman`](https://leetcode.com/problems/integer-to-roman/description/) — задача, в которой нужно преобразовать целое число в римскую запись.

**Основные характеристики:**

1. **Входные данные:** число `num` (1 ≤ num ≤ 3999).
2. **Цель:** вернуть строку с римским представлением числа.
3. **Сложность:** оптимальное решение достигается за `O(1)` по времени (число всегда ограничено 3999) и `O(1)` по памяти.

**Жадное решение (основное):**

```ts
function intToRoman(num: number): string {
  const VALUES = [1000, 900, 500, 400, 100, 90, 50, 40, 10, 9, 5, 4, 1];
  const SYMBOLS = [
    "M",
    "CM",
    "D",
    "CD",
    "C",
    "XC",
    "L",
    "XL",
    "X",
    "IX",
    "V",
    "IV",
    "I",
  ];

  let res = "";
  let val = num;

  for (let i = 0; i < VALUES.length; i++) {
    while (val >= VALUES[i]) {
      val -= VALUES[i];
      res += SYMBOLS[i];
    }
  }

  return res;
}
```

**Как это работает:**

- Задаём массивы значений и соответствующих римских символов.
- Идём от большего к меньшему: пока текущее число `val` ≥ текущего значения, вычитаем его и добавляем символ.
- В итоге строим строку жадно, начиная с самых больших чисел.

**Пример:**

```
num = 1994
Алгоритм: 1000 (M) → 900 (CM) → 90 (XC) → 4 (IV)
Результат: "MCMXCIV"
```

**Вариант с таблицей и пошаговой конкатенацией:**

```ts
function intToRoman(num: number): string {
  const THOUSANDS = ["", "M", "MM", "MMM"];
  const HUNDREDS = ["", "C", "CC", "CCC", "CD", "D", "DC", "DCC", "DCCC", "CM"];
  const TENS = ["", "X", "XX", "XXX", "XL", "L", "LX", "LXX", "LXXX", "XC"];
  const ONES = ["", "I", "II", "III", "IV", "V", "VI", "VII", "VIII", "IX"];

  return (
    THOUSANDS[Math.floor(num / 1000)] +
    HUNDREDS[Math.floor((num % 1000) / 100)] +
    TENS[Math.floor((num % 100) / 10)] +
    ONES[num % 10]
  );
}
```

**Как это работает:**

- Используем готовые таблицы для каждой позиции (тысячи, сотни, десятки, единицы).
- Разбиваем число по разрядам и сразу конкатенируем нужные символы.
- Решение работает быстрее на практике, так как избегает циклов.

**Пример:**

```
num = 3549
= "MMM" + "D" + "XL" + "IX"
= "MMMDXLIX"
```

**Сравнение решений:**

- **Жадное решение:** универсальное, легко масштабируется, работает через цикл.
- **Табличное решение:** быстрее и компактнее, но требует зашить готовые комбинации.

Оба решения имеют `O(1)` сложность, так как диапазон входных данных фиксирован.

### Length of Last Word

[`Leetcode - 58. Length of Last Word`](https://leetcode.com/problems/length-of-last-word/) — задача, в которой нужно найти длину последнего слова в строке.

**Основные характеристики:**

1. **Входные данные:** строка `s`, содержащая буквы и пробелы.
2. **Цель:** вернуть длину последнего слова (последовательность символов без пробелов).
3. **Сложность:** оптимальное решение работает за `O(n)` по времени и `O(1)` по памяти.

**Решение через реверс строки (наивное):**

```ts
function lengthOfLastWord(s: string): number {
  const rev = s.split("").reverse().join("").trim();

  for (let i = 0; i < rev.length; i++) {
    const ch = rev[i];
    if (ch === " ") return i;
  }

  return rev.length;
}
```

- Переворачиваем строку.
- Убираем пробелы в начале (бывшие пробелы в конце).
- Ищем первый пробел — его индекс равен длине последнего слова.
- Если пробела нет, возвращаем длину строки.

**Оптимальное решение (итерация с конца):**

```ts
function lengthOfLastWord(s: string): number {
  let i = s.length - 1;

  // Пропускаем пробелы в конце
  while (i >= 0 && s[i] === " ") i--;

  let length = 0;
  // Считаем символы до пробела или начала строки
  while (i >= 0 && s[i] !== " ") {
    length++;
    i--;
  }

  return length;
}
```

- Начинаем с конца.
- Пропускаем все пробелы.
- Считаем длину последнего слова.

**Пример:**

```
s = "Hello World" → 5
s = "   fly me   to   the moon  " → 4
s = "luffy is still joyboy" → 6
```

**Сравнение решений:**

- Реверс: понятный, но требует `O(n)` памяти.
- Итерация с конца: эффективный вариант с `O(1)` памяти.

### Longest Common Prefix

[`Leetcode - 14. Longest Common Prefix`](https://leetcode.com/problems/longest-common-prefix/description/) — задача, в которой нужно найти наибольший общий префикс среди массива строк.

**Основные характеристики:**

1. Входные данные: массив строк `strs`.
2. Цель: вернуть самую длинную строку, которая является префиксом всех строк в массиве.
3. Сложность: оптимальное решение достигается за `O(n * m)` по времени, где `n` — количество строк, а `m` — длина минимальной строки.

**Решение через сортировку и сравнение крайних:**

```ts
function longestCommonPrefix(strs: string[]): string {
  if (!strs.length) return "";

  const arr = strs.sort();

  const first = arr[0];
  const last = arr[arr.length - 1];

  let i = 0;
  while (i < first.length && first[i] === last[i]) {
    i++;
  }

  return first.slice(0, i);
}
```

**Как это работает:**

- Сортируем массив строк.
- Берем первую и последнюю строку после сортировки.
- Сравниваем символы по позиции, пока они совпадают.
- Результат — общий префикс первой и последней строки, который будет префиксом всего массива.

**Пример:**

```
strs = ["flower","flow","flight"]
После сортировки: ["flight","flow","flower"]
Префикс первой и последней: "fl"
```

**Решение методом вертикального сканирования:**

```ts
function longestCommonPrefix(strs: string[]): string {
  if (!strs.length) return "";

  for (let i = 0; i < strs[0].length; i++) {
    const char = strs[0][i];

    for (let j = 1; j < strs.length; j++) {
      if (i >= strs[j].length || strs[j][i] !== char) {
        return strs[0].slice(0, i);
      }
    }
  }

  return strs[0];
}
```

**Как это работает:**

- Берем первый символ первой строки.
- Проверяем этот символ на всех остальных строках.
- Если символы совпадают, продолжаем с следующим символом.
- Как только есть несоответствие или конец строки, возвращаем префикс.

**Сравнение решений:**

- Сортировка и сравнение крайних: проще, легко понять, время `O(n log n + m)`.
- Вертикальное сканирование: оптимально по времени для больших массивов и длинных строк, время `O(n * m)`.

**Решение через split, reverse, join:**

```ts
function reverseWords(s: string): string {
  return s.trim().split(/\s+/).reverse().join(" ");
}
```

**Как это работает:**

- Убираем пробелы с начала и конца строки с помощью `trim()`.
- Разбиваем строку на массив слов по одному или нескольким пробелам с помощью регулярного выражения `/\s+/`.
- Переворачиваем массив слов.
- Соединяем массив обратно в строку через один пробел.

**Пример:**

```
s = "  the sky  is blue  "
Результат: "blue is sky the"
```

**Решение с использованием двух указателей (на месте, без split):**

```ts
function reverseWordsInPlace(s: string): string {
  let words: string[] = [];
  let i = 0;
  const n = s.length;

  while (i < n) {
    while (i < n && s[i] === " ") i++;
    if (i >= n) break;
    let j = i;
    while (j < n && s[j] !== " ") j++;
    words.push(s.slice(i, j));
    i = j;
  }

  return words.reverse().join(" ");
}
```

**Как это работает:**

- Проходим по строке, накапливаем слова в массив.
- Пропускаем лишние пробелы.
- После сбора всех слов переворачиваем массив и соединяем слова через один пробел.

**Сравнение решений:**

- `split + reverse + join`: проще и лаконично, требует `O(n)` памяти для массива слов.
- Двухуказательный метод: работает без регулярных выражений, более контролируемый сбор слов, тоже `O(n)` памяти, но чуть медленнее по коду.

[`Leetcode - 6. Zigzag Conversion`](https://leetcode.com/problems/zigzag-conversion/description/) — задача, в которой нужно записать строку в форме "зигзаг" по заданному числу рядов и вернуть её построчно.

1. **Входные данные:** строка `s` и число рядов `numRows`.
2. **Цель:** вернуть строку, прочитанную построчно в формате "зигзаг".
3. **Сложность:** оптимальное решение достигается за `O(n)` по времени и `O(n)` по памяти.

**Решение через прямое построение строк для каждого ряда:**

```ts
function convert(s: string, numRows: number): string {
  if (numRows === 1 || s.length <= numRows) return s;

  const rows = new Array(Math.min(numRows, s.length)).fill("");
  let curRow = 0;
  let goingDown = false;

  for (const char of s) {
    rows[curRow] += char;
    if (curRow === 0 || curRow === numRows - 1) goingDown = !goingDown;
    curRow += goingDown ? 1 : -1;
  }

  return rows.join("");
}
```

**Как это работает:**

- Создаём массив строк для каждого ряда.
- Проходим по символам строки и добавляем их в соответствующий ряд.
- Меняем направление движения (вниз/вверх), когда достигаем первого или последнего ряда.
- Объединяем все ряды в итоговую строку.

**Пример:**

```
s = "PAYPALISHIRING", numRows = 3
Ряды после построения:
P   A   H   N
A P L S I I G
Y   I   R
Итог: "PAHNAPLSIIGYIR"
```

**Альтернативное решение через индексную формулу (оптимизация без дополнительного массива для рядов):**

```ts
function convert(s: string, numRows: number): string {
  if (numRows === 1 || s.length <= numRows) return s;

  const step = 2 * numRows - 2;
  let res = "";

  for (let row = 0; row < numRows; row++) {
    for (let i = row; i < s.length; i += step) {
      res += s[i];
      const j = i + step - 2 * row;
      if (row !== 0 && row !== numRows - 1 && j < s.length) {
        res += s[j];
      }
    }
  }

  return res;
}
```

**Как это работает:**

- Вычисляем шаг `2 * numRows - 2`, который определяет цикл "зигзага".
- Для каждого ряда собираем символы по формуле, учитывая обратные диагональные элементы.
- Работает за `O(n)` по времени и `O(1)` по дополнительной памяти (кроме строки-результата).

**Сравнение решений:**

- **Прямое построение рядов:** проще для понимания, легко реализуется, требует `O(n)` памяти для хранения рядов.
- **Индексная формула:** использует меньше дополнительной памяти, но чуть сложнее для понимания и отладки.

### Find the Index of the First Occurrence in a String

[`Leetcode - 28. Find the Index of the First Occurrence in a String`](https://leetcode.com/problems/implement-strstr/) — задача, в которой нужно найти первую позицию подстроки `needle` в строке `haystack`.

**Основные характеристики:**

1. Входные данные: строки `haystack` и `needle`.
2. Цель: вернуть индекс первого вхождения `needle` в `haystack` или `-1`, если подстрока не найдена.
3. Сложность: оптимальное решение достигается за `O(n*m)` по времени, где `n` — длина `haystack`, а `m` — длина `needle`.

**Решение через встроенный метод:**

```ts
function strStr(haystack: string, needle: string): number {
  return haystack.indexOf(needle);
}
```

**Решение через наивное сравнение:**

```ts
function strStr(haystack: string, needle: string): number {
  const n = haystack.length;
  const m = needle.length;
  if (m === 0) return 0;

  for (let i = 0; i <= n - m; i++) {
    let j = 0;
    while (j < m && haystack[i + j] === needle[j]) j++;
    if (j === m) return i;
  }

  return -1;
}
```

**Решение с использованием алгоритма [Кнута–Морриса–Пратта (KMP)](https://neerc.ifmo.ru/wiki/index.php?title=%D0%90%D0%BB%D0%B3%D0%BE%D1%80%D0%B8%D1%82%D0%BC_%D0%9A%D0%BD%D1%83%D1%82%D0%B0-%D0%9C%D0%BE%D1%80%D1%80%D0%B8%D1%81%D0%B0-%D0%9F%D1%80%D0%B0%D1%82%D1%82%D0%B0):**

```ts
function strStr(haystack: string, needle: string): number {
  if (needle.length === 0) return 0;

  const lps = buildLPS(needle);
  let i = 0,
    j = 0;

  while (i < haystack.length) {
    if (haystack[i] === needle[j]) {
      i++;
      j++;
      if (j === needle.length) return i - j;
    } else if (j > 0) {
      j = lps[j - 1];
    } else {
      i++;
    }
  }

  return -1;
}

function buildLPS(pattern: string): number[] {
  const lps = Array(pattern.length).fill(0);
  let len = 0,
    i = 1;

  while (i < pattern.length) {
    if (pattern[i] === pattern[len]) {
      len++;
      lps[i] = len;
      i++;
    } else if (len > 0) {
      len = lps[len - 1];
    } else {
      lps[i] = 0;
      i++;
    }
  }

  return lps;
}
```

**Как это работает:**

- Встроенный метод: использует готовый `indexOf` JavaScript.
- Наивное сравнение: проверяет каждый возможный стартовый индекс и сравнивает символы.
- KMP: строит префикс-функцию для оптимизации поиска, избегая повторного сравнения.

**Пример:**

```
haystack = "hello", needle = "ll"
Результат: 2
```

**Сравнение решений:**

- Встроенный метод: проще всего использовать, `O(n*m)` в худшем случае.
- Наивное сравнение: ручная реализация, `O(n*m)`.
- KMP: оптимально для больших строк и повторяющихся паттернов, `O(n+m)` по времени и `O(m)` по памяти.

### Text Justification

[`Leetcode - 68. Text Justification`](https://leetcode.com/problems/text-justification/) — задача, в которой нужно оформить текст в строки с заданной шириной, распределяя пробелы между словами.

**Основные характеристики:**

1. **Входные данные:** массив слов `words` и число `maxWidth`.
2. **Цель:** вернуть массив строк, каждая строка длиной `maxWidth`, с равномерно распределёнными пробелами.
3. **Сложность:** оптимальное решение работает за `O(n)` по времени и памяти, где `n` — количество слов.

**Решение с пошаговым распределением пробелов:**

```ts
function fullJustify(words: string[], maxWidth: number): string[] {
  const res: string[] = [];
  let currWords: string[] = [];
  let currLen = 0;

  for (let i = 0; i < words.length; i++) {
    const word = words[i];

    if (currLen + word.length + currWords.length > maxWidth) {
      const gaps = currWords.length - 1;
      const spaces = maxWidth - currLen;

      if (gaps === 0) {
        res.push(currWords[0] + " ".repeat(spaces));
      } else {
        const spacePerGap = Math.floor(spaces / gaps);
        let extra = spaces % gaps;
        let line = "";

        for (let j = 0; j < currWords.length; j++) {
          line += currWords[j];
          if (j < gaps) {
            line += " ".repeat(spacePerGap + (extra > 0 ? 1 : 0));
            if (extra > 0) extra--;
          }
        }

        res.push(line);
      }

      currWords = [];
      currLen = 0;
    }

    currWords.push(word);
    currLen += word.length;
  }

  const lastLine = currWords.join(" ");
  res.push(lastLine + " ".repeat(maxWidth - lastLine.length));

  return res;
}
```

**Как это работает:**

- Идём по словам и собираем их в текущую строку, пока не превышен `maxWidth`.
- Когда строка полная, распределяем пробелы:
  - Если одно слово — добавляем все пробелы в конец.
  - Если несколько слов — распределяем пробелы равномерно и оставшиеся добавляем слева направо.

- Последнюю строку объединяем обычными пробелами и добиваем до `maxWidth`.

**Пример:**

```
words = ["This","is","an","example","of","text","justification."]
maxWidth = 16
Результат:
[
 "This    is    an",
 "example  of text",
 "justification.  "
]
```

**Альтернативное решение (с использованием reduce и динамической сборки):**

```ts
function fullJustify(words: string[], maxWidth: number): string[] {
  const res: string[] = [];
  let line: string[] = [];
  let lineLen = 0;

  words.forEach((word) => {
    if (lineLen + word.length + line.length > maxWidth) {
      const spaces = maxWidth - lineLen;
      const gaps = line.length - 1 || 1;
      res.push(
        line
          .map(
            (w, i) => w + " ".repeat(i < gaps ? Math.ceil(spaces / gaps) : 0)
          )
          .join("")
          .slice(0, maxWidth)
      );
      line = [];
      lineLen = 0;
    }
    line.push(word);
    lineLen += word.length;
  });

  const lastLine = line.join(" ");
  res.push(lastLine + " ".repeat(maxWidth - lastLine.length));

  return res;
}
```

**Сравнение решений:**

- Первый метод: более явный, легко понять, оптимально распределяет пробелы.
- Второй метод: компактный, использует функциональный подход, но чуть сложнее читать и отлаживать.
